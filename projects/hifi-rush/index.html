PBR (Cook-Torrance BRDF)

GGX Distribution + Schlick-GGX Geometry + Fresnel-Schlick 조합의 물리 기반 렌더링을 구현했습니다.

Shader_Defines.hlsli - PBR 핵심 함수 HLSL 
float3 fresnelSchlick(float cosTheta, float3 F0) {
    return F0 + (1.0 - F0) * pow(1.0 - cosTheta, 5.0);
}
float DistributionGGX(float3 N, float3 H, float roughness) {
    float a2 = pow(roughness, 4.0);
    float NdotH2 = pow(max(dot(N, H), 0.0), 2.0);
    float denom = 3.14159 * pow(NdotH2 * (a2 - 1.0) + 1.0, 2.0);
    return a2 / denom;
}
float GeometrySmith(float3 N, float3 V, float3 L, float roughness) {
    float k = pow(roughness + 1.0, 2.0) / 8.0;
    float ggx1 = max(dot(N,L),0.0) / (max(dot(N,L),0.0) * (1.0-k) + k);
    float ggx2 = max(dot(N,V),0.0) / (max(dot(N,V),0.0) * (1.0-k) + k);
    return ggx1 * ggx2;
}

03 셀 셰이딩 & NPR

Hi-Fi Rush의 만화 스타일 그래픽을 재현하기 위한 NPR 파이프라인입니다. 라이팅 결과를 4단계로 양자화하여 셀 셰이딩을 적용하고, 모델 타입에 따라 다른 외곽선 기법을 사용합니다.

모델 타입외곽선 기법방식
Animated (스켈레탈)정점 확장 (Vertex Extrusion)법선 방향으로 정점을 밀어낸 뒤 전면 컬링
Non-Animated (정적)라플라시안 필터3x3 법선 버퍼 에지 검출

// 라플라시안 외곽선 (Non-Animated 모델)
float3 OutLine = (float3)0;

float gray = 1.f - dot(grayScale, OutLine) / clamp(dist/10.f, 1.f, 100.f);

Shader_VtxAnimMesh.hlsl - 정점 확장 외곽선 HLSL 
VS_OUT VS_OUTLINE(VS_IN In) {
    // 4개 가중치 본 블렌딩
    BoneMatrix = g_BoneMatrix[In.vBlendIndex0.x] * In.vBlendWeight0.x
               + g_BoneMatrix[In.vBlendIndex0.y] * In.vBlendWeight0.y
               + g_BoneMatrix[In.vBlendIndex0.z] * In.vBlendWeight0.z
               + g_BoneMatrix[In.vBlendIndex0.w] * In.vBlendWeight0.w;
    // 법선 방향으로 정점을 밀어 외곽선 생성

    Out.vPosition = mul(Out.vWorldPos, mul(g_ViewMatrix, g_ProjMatrix));
}

Union Effect

여러 이펙트를 하나의 그룹으로 묶어 키프레임 기반으로 동기화하는 시스템입니다. 보스전의 복합 이펙트(충돌 판정 + 사운드 + 시각 효과)를 하나의 타임라인에서 관리할 수 있어, 이펙트 제작 생산성을 크게 향상시켰습니다.

PhysX 5.3 통합

09 트러블슈팅

스마트 포인터 전환

기존에 사용하던 프레임워크가 레퍼런스 카운트를 수동 관리하는 방식으로, 스마트 포인터와 비슷한 기능을 수행하고 있었습니다. 하지만 협업 중에는 관리가 여의치 않았고, 누군가 실수하면 찾아 올라가기도 힘들고 의도치 않은 곳에서 메모리 누수가 한번씩 발생했습니다.

이를 해결하기 위해 프레임워크의 모든 변수를 스마트 포인터(shared_ptr/weak_ptr)로 변환하는 작업을 진행했습니다. 동시에 다른 팀원들도 스마트 포인터를 쉽게 사용할 수 있도록 매크로를 작성하여 제공했습니다.

스마트 포인터 변경 후 기존의 메모리 관련 문제가 10건도 발생하지 않아, 작업 효율을 크게 올릴 수 있었습니다.
이펙트 제작 생산성 문제

제가 주로 가장 많이 작업한 파트가 이펙트 파트인데, 이펙트에 미묘하게 자잘한 효과도 많이 들어가고, 충돌 판정이나 사운드 같은 것들을 개별적으로 넣기 힘든 부분들이 많았습니다.

이 문제를 해결하기 위해 직접 설계한 Union Effect 구조를 만들었습니다. 여러 이펙트를 하나의 타임라인으로 묶어 키프레임 기반으로 충돌, 사운드, 시각 효과를 동기화하는 방식입니다.

Union Effect로 이펙트의 생산성을 정말 많이 올릴 수 있었습니다. 복합 이펙트의 타이밍 관리가 일관되게 처리됩니다.

핵심 성과

  • 14-Pass Deferred 파이프라인 - 셀 셰이딩 + PBR + 그림자 + Glow + SSR + 안개
  • 200+ 이펙트 클래스 - 상태 머신 라이프사이클 + Union Effect 키프레임 동기화
  • 80+ 상태 보스 AI - 4단계 페이즈, 독립 기계팔 FSM, 리듬 패리 시스템
  • 로딩 시간 93% 단축 - 이진 직렬화 + 텍스처 캐싱 + 멀티스레딩
  • PhysX 5.3 통합 - CCT, 그룹 충돌, Sweep 테스트, 방향 기반 히트 리포트
