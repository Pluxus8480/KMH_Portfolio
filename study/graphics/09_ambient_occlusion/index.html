Ambient Occlusion (SSAO, HBAO+, GTAO)

Part 1: Ambient Occlusion이란?

왜 모서리나 구석이 더 어둡게 보이나요?

현실에서 주변 환경광(Ambient Light)은 모든 방향에서 오지만, 좁은 공간이나 모서리는 빛이 덜 도달합니다.

Ambient Occlusion은 이 "막힌 정도"를 계산해 더 사실적인 음영을 만듭니다.

Ambient Occlusion 개념:

AO 없음:                         AO 적용:
┌──────────────────────┐        ┌──────────────────────┐
│                      │        │                      │
│    ┌─────┐           │        │    ┌─────┐           │
│    │     │           │        │  ▓▓│     │           │
│    │     │           │        │  ▓▓│     │           │
│    └─────┘           │        │  ▓▓└─────┘▓▓         │
│                      │        │    ▓▓▓▓▓▓▓▓          │
│══════════════════════│        │▓▓══════════════════▓▓│
└──────────────────────┘        └──────────────────────┘

▓ = 어두운 영역 (빛이 차단됨)

구석, 틈새, 접촉점 = 주변 지오메트리가 빛을 차단

🎯 비유로 이해하기

AO는 "먼지가 쌓이는 곳"을 어둡게 만드는 것과 같습니다.

구석, 틈새, 접촉점 - 청소하기 힘든 곳이 더 어둡습니다.

Part 2: SSAO (Screen Space Ambient Occlusion)

SSAO는 어떻게 동작하나요?

SSAO는 2007년 Crytek이 발표한 기법입니다. 각 픽셀 주변에 랜덤 샘플을 배치하고, 깊이 버퍼와 비교해 "막힌 비율"을 계산합니다.

SSAO 샘플링:

            법선 방향
               ↑
               │
        ●  ●   │   ●  ●
          ●    │    ●
        ●      │      ●      ● = 샘플 포인트
    ───────────●───────────   ● = 현재 픽셀
          (현재 픽셀)

반구 내 랜덤 샘플 배치
→ 각 샘플의 깊이 vs 실제 깊이 비교
→ 샘플이 표면 뒤에 있으면 "차폐됨"

차폐율 = 차폐된 샘플 수 / 전체 샘플 수

SSAO 반구 샘플링

SSAO 구현 (HLSL)
// 커널 샘플 (미리 계산된 반구 내 랜덤 방향)
static const float3 kernel[16] = {
    float3(0.2, 0.1, 0.3), float3(-0.3, 0.2, 0.1),
    // ... 더 많은 샘플
};

float SSAO_PS(float2 uv : TEXCOORD) : SV_Target
{
    // 현재 픽셀 정보
    float depth = depthBuffer.Sample(pointSamp, uv).r;
    float3 position = ReconstructViewPosition(uv, depth);
    float3 normal = normalBuffer.Sample(pointSamp, uv).rgb * 2.0 - 1.0;

    // 랜덤 회전 벡터 (노이즈 텍스처에서)
    float3 randomVec = noiseTexture.Sample(wrapSamp, uv * noiseScale).rgb;

    // TBN 행렬 (법선 기준 반구 방향)
    float3 tangent = normalize(randomVec - normal * dot(randomVec, normal));
    float3 bitangent = cross(normal, tangent);
    float3x3 TBN = float3x3(tangent, bitangent, normal);

    float occlusion = 0.0;
    float radius = 0.5;  // 샘플링 반경

    for (int i = 0; i < 16; i++)
    {
        // 샘플 위치 계산
        float3 sampleDir = mul(kernel[i], TBN);
        float3 samplePos = position + sampleDir * radius;

        // 뷰 공간 → 클립 → UV
        float4 offset = mul(float4(samplePos, 1), projMatrix);
        float2 sampleUV = offset.xy / offset.w * 0.5 + 0.5;

        // 깊이 비교
        float sampleDepth = depthBuffer.Sample(pointSamp, sampleUV).r;
        float3 sampleViewPos = ReconstructViewPosition(sampleUV, sampleDepth);

        // 샘플이 표면 뒤에 있는지 체크
        float rangeCheck = smoothstep(0.0, 1.0,
            radius / abs(position.z - sampleViewPos.z));

        if (sampleViewPos.z >= samplePos.z + 0.025)
            occlusion += rangeCheck;
    }

    occlusion = 1.0 - (occlusion / 16.0);
    return pow(occlusion, 2.0);  // 강도 조절
}

Part 3: SSAO 블러와 노이즈 제거

SSAO 결과가 노이즈가 심한데요?

샘플 수가 적으면 노이즈가 발생합니다. 4x4 노이즈 패턴과 Edge-Aware Blur로 해결합니다.

Edge-Preserving Blur
// 깊이/법선 경계를 보존하는 블러
float BlurSSAO(float2 uv)
{
    float centerAO = ssaoBuffer.Sample(pointSamp, uv).r;
    float centerDepth = depthBuffer.Sample(pointSamp, uv).r;
    float3 centerNormal = normalBuffer.Sample(pointSamp, uv).rgb;

    float result = 0.0;
    float totalWeight = 0.0;

    // 4x4 블러 커널
    for (int x = -2; x <= 2; x++)
    {
        for (int y = -2; y <= 2; y++)
        {
            float2 offset = float2(x, y) * texelSize;
            float2 sampleUV = uv + offset;

            float sampleAO = ssaoBuffer.Sample(pointSamp, sampleUV).r;
            float sampleDepth = depthBuffer.Sample(pointSamp, sampleUV).r;
            float3 sampleNormal = normalBuffer.Sample(pointSamp, sampleUV).rgb;

            // 깊이 차이 가중치
            float depthWeight = exp(-abs(centerDepth - sampleDepth) * 100.0);

            // 법선 차이 가중치
            float normalWeight = max(dot(centerNormal, sampleNormal), 0.0);

            float weight = depthWeight * normalWeight;
            result += sampleAO * weight;
            totalWeight += weight;
        }
    }

    return result / totalWeight;
}
SSAO 파이프라인:

┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│  Depth +     │ → │    SSAO      │ → │   Blur       │
│  Normal      │    │  (노이즈 有)  │    │ (Edge-Aware) │
└──────────────┘    └──────────────┘    └──────────────┘
                                              │
                                              ▼
                                   ┌──────────────┐
                                   │ Final AO     │
                                   │ (깨끗한 결과) │
                                   └──────────────┘

Part 4: HBAO (Horizon-Based Ambient Occlusion)

HBAO는 SSAO와 어떻게 다른가요?

HBAO는 지평선 각도를 기반으로 차폐를 계산합니다. 물리적으로 더 정확하고, 넓은 영역의 AO를 잘 표현합니다.

HBAO 원리:

SSAO: 반구 내 랜덤 샘플링
┌─────────────────────────────┐
│     ●    ●                  │
│  ●     ●    ●               │
│    ●  P  ●                  │  P = 현재 픽셀
│  ●     ●                    │  불규칙한 샘플 분포
└─────────────────────────────┘

HBAO: 방사형 방향별 최대 지평선 각도
┌─────────────────────────────┐
│         ↑                   │
│       ↗ │ ↖                 │
│     ←   P   →               │  각 방향으로 레이 마칭
│       ↙ │ ↘                 │  가장 높은 지평선 찾기
│         ↓                   │
└─────────────────────────────┘

θ = horizon angle (지평선 각도)
AO = sin(θ) 기반 계산

SSAO vs HBAO 접근 방식

HBAO 핵심 개념
// HBAO 간략 구현
float HBAO(float2 uv)
{
    float3 position = ReconstructViewPosition(uv);
    float3 normal = GetNormal(uv);

    float ao = 0.0;
    const int numDirections = 8;
    const int numSteps = 4;

    for (int dir = 0; dir < numDirections; dir++)
    {
        // 각 방향 (45도 간격)
        float angle = dir * (2.0 * PI / numDirections);
        float2 direction = float2(cos(angle), sin(angle));

        float maxHorizon = -1.0;  // 최대 지평선 각도의 sin

        for (int step = 1; step <= numSteps; step++)
        {
            float2 sampleUV = uv + direction * step * stepSize;
            float3 samplePos = ReconstructViewPosition(sampleUV);

            // 현재 픽셀에서 샘플까지의 벡터
            float3 horizonVec = samplePos - position;
            float horizonLen = length(horizonVec);
            horizonVec /= horizonLen;

            // 지평선 각도 (법선과의 내적)
            float horizonAngle = dot(normal, horizonVec);

            // 거리 기반 감쇠
            float attenuation = 1.0 - saturate(horizonLen / aoRadius);

            maxHorizon = max(maxHorizon, horizonAngle * attenuation);
        }

        // 지평선 각도에서 AO 계산
        // 높은 지평선 = 더 많이 차폐됨
        ao += saturate(maxHorizon);
    }

    return 1.0 - (ao / numDirections);
}
기법 장점 단점
SSAO 간단, 빠름 물리적 정확도 낮음
HBAO 물리 기반, 품질 좋음 SSAO보다 느림

Part 5: GTAO (Ground Truth Ambient Occlusion)

가장 최신 기법은 무엇인가요?

GTAO는 2016년 Activision이 발표한 기법입니다. 레이트레이싱 AO에 가까운 품질을 Screen Space에서 달성합니다.

GTAO 핵심 개념
// GTAO는 적분을 해석적으로 계산
// Ground Truth: ∫ V(ω) * cos(θ) dω

// 슬라이스 기반 접근
float GTAO(float2 uv)
{
    float3 viewPos = ReconstructViewPosition(uv);
    float3 viewNormal = GetViewNormal(uv);

    float ao = 0.0;
    const int slices = 2;  // 수평/수직 슬라이스

    for (int slice = 0; slice < slices; slice++)
    {
        // 슬라이스 방향
        float phi = (slice + 0.5) * PI / slices;
        float3 sliceDir = float3(cos(phi), sin(phi), 0);

        // 법선을 슬라이스 평면에 투영
        float3 projNormal = viewNormal - sliceDir * dot(viewNormal, sliceDir);
        float projNormalLen = length(projNormal);

        // 슬라이스 내에서 지평선 검색
        float2 horizons = SearchHorizons(uv, sliceDir, viewPos);

        // 적분 계산 (해석적 공식)
        float h1 = horizons.x;  // 한쪽 방향 지평선
        float h2 = horizons.y;  // 반대쪽 방향 지평선

        // Ground Truth 적분 근사
        float n = acos(projNormal.z / projNormalLen);
        ao += IntegrateArc(h1, h2, n) * projNormalLen;
    }

    return ao / slices;
}

// 해석적 적분
float IntegrateArc(float h1, float h2, float n)
{
    float sinN = sin(n);
    float cosN = cos(n);

    // 적분 공식 (논문 참조)
    return 0.25 * (
        -cos(2.0 * h1 - n) + cosN + 2.0 * h1 * sinN
        -cos(2.0 * h2 - n) + cosN + 2.0 * h2 * sinN
    );
}
🎯 GTAO의 장점

물리적 정확성: 레이트레이싱 결과와 유사

적은 샘플: 해석적 계산으로 샘플 수 감소

시간적 안정성: 노이즈 적음

Part 6: AO 적용과 최적화

AO를 어떻게 최종 이미지에 적용하나요?

AO는 보통 Ambient Light에만 곱합니다. 직접광에는 적용하지 않습니다 (그림자가 따로 있으므로).

AO 적용
// 라이팅에 AO 적용
float3 ApplyLighting(float3 albedo, float3 normal,
                      float3 lightDir, float ao)
{
    // Ambient (AO 적용)
    float3 ambient = ambientColor * albedo * ao;

    // Diffuse (AO 적용 안함 - 직접광)
    float NdotL = max(dot(normal, lightDir), 0);
    float3 diffuse = lightColor * albedo * NdotL;

    // Specular (AO 적용 안함)
    float3 specular = CalculateSpecular(normal, lightDir);

    return ambient + diffuse + specular;
}

// 또는 Indirect Light 전체에 적용
float3 FinalColor = directLight + indirectLight * ao;
최적화 기법 설명
절반 해상도 1/2 해상도에서 계산 후 업샘플
Interleaved Rendering 체커보드 패턴으로 분할 계산
Temporal Filter 이전 프레임 결과 재사용
Compute Shader GPU 병렬 처리 활용
