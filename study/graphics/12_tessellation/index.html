Tessellation과 Displacement Mapping

Part 1: Tessellation이란?

왜 GPU에서 폴리곤을 생성해야 하나요?

저폴리곤 메시를 GPU에서 동적으로 분할해 고해상도로 만듭니다. 메모리 대역폭을 절약하면서 디테일을 높입니다.

가까이 있는 오브젝트는 많이 분할하고, 멀리 있으면 적게 분할 → 적응적 LOD

삼각형 N개
GPU에서 동적 생성
메모리: 저폴리만 전송
LOD: 거리별 분할 조절

Tessellation 분할 개념

🎯 비유로 이해하기

Tessellation은 종이접기와 같습니다.

큰 종이(저폴리)를 보내면, GPU가 많은 접기(분할)를 해서 복잡한 형태를 만듭니다.

Part 3: Hull Shader

Hull Shader는 무엇을 하나요?

Tessellation Factor를 결정합니다. 거리, 화면 크기, 곡률 등을 기반으로 분할 수준을 정합니다.

Hull Shader (HLSL)
// 입력: 패치의 컨트롤 포인트들
struct VS_OUTPUT
{
    float3 position : POSITION;
    float3 normal   : NORMAL;
    float2 uv       : TEXCOORD0;
};

// 출력: 컨트롤 포인트 (Domain Shader로)
struct HS_OUTPUT
{
    float3 position : POSITION;
    float3 normal   : NORMAL;
    float2 uv       : TEXCOORD0;
};

// Patch Constant Function (패치당 1번 호출)
struct HS_CONSTANT_OUTPUT
{
    float edges[3]  : SV_TessFactor;      // 엣지별 분할 수
    float inside    : SV_InsideTessFactor; // 내부 분할 수
};

// 거리 기반 Tessellation Factor 계산
float CalculateTessFactor(float3 position)
{
    float distance = length(cameraPos - position);

    // 가까우면 많이 분할, 멀면 적게
    float factor = maxTessellation / (1.0 + distance * 0.1);
    return clamp(factor, 1, maxTessellation);
}

HS_CONSTANT_OUTPUT PatchConstantFunction(
    InputPatch patch,
    uint patchID : SV_PrimitiveID)
{
    HS_CONSTANT_OUTPUT output;

    // 각 엣지의 중심점에서 Factor 계산
    float3 edgeCenter0 = (patch[1].position + patch[2].position) * 0.5;
    float3 edgeCenter1 = (patch[2].position + patch[0].position) * 0.5;
    float3 edgeCenter2 = (patch[0].position + patch[1].position) * 0.5;

    output.edges[0] = CalculateTessFactor(edgeCenter0);
    output.edges[1] = CalculateTessFactor(edgeCenter1);
    output.edges[2] = CalculateTessFactor(edgeCenter2);

    // 내부 Factor (평균)
    output.inside = (output.edges[0] + output.edges[1] + output.edges[2]) / 3.0;

    return output;
}

// 컨트롤 포인트 함수 (컨트롤 포인트당 1번 호출)
[domain("tri")]                    // 삼각형 패치
[partitioning("fractional_odd")]   // 분할 방식
[outputtopology("triangle_cw")]    // 출력 토폴로지
[outputcontrolpoints(3)]           // 출력 컨트롤 포인트 수
[patchconstantfunc("PatchConstantFunction")]
HS_OUTPUT HS_Main(
    InputPatch patch,
    uint id : SV_OutputControlPointID)
{
    HS_OUTPUT output;

    output.position = patch[id].position;
    output.normal = patch[id].normal;
    output.uv = patch[id].uv;

    return output;
}

Part 4: Domain Shader

Domain Shader는 무엇을 하나요?

Tessellator가 생성한 각 정점의 최종 위치를 계산합니다. 무게 중심 좌표(Barycentric)를 사용해 보간합니다.

Barycentric 좌표
삼각형 꼭짓점: P0, P1, P2
내부 점 P = u·P0 + v·P1 + w·P2
(u + v + w = 1)
↓
Tessellator
각 점의 uvw 좌표 생성
→
Domain Shader
uvw로 패치 속성 보간

Barycentric 좌표계

Domain Shader (HLSL)
struct DS_OUTPUT
{
    float4 position : SV_Position;
    float3 worldPos : TEXCOORD0;
    float3 normal   : TEXCOORD1;
    float2 uv       : TEXCOORD2;
};

[domain("tri")]
DS_OUTPUT DS_Main(
    HS_CONSTANT_OUTPUT patchConstants,
    float3 bary : SV_DomainLocation,  // uvw 좌표
    const OutputPatch patch)
{
    DS_OUTPUT output;

    // Barycentric 보간
    float3 position = patch[0].position * bary.x +
                      patch[1].position * bary.y +
                      patch[2].position * bary.z;

    float3 normal = patch[0].normal * bary.x +
                    patch[1].normal * bary.y +
                    patch[2].normal * bary.z;

    float2 uv = patch[0].uv * bary.x +
                patch[1].uv * bary.y +
                patch[2].uv * bary.z;

    // Displacement Mapping!
    float height = displacementMap.SampleLevel(linearSamp, uv, 0).r;
    position += normalize(normal) * height * displacementScale;

    // 변환
    output.worldPos = position;
    output.position = mul(float4(position, 1), viewProjMatrix);
    output.normal = normalize(normal);
    output.uv = uv;

    return output;
}

Part 5: Displacement Mapping

Normal Map과 Displacement Map의 차이는?

Normal Map은 법선만 바꿔 라이팅을 속입니다 (실제 형태 변화 없음).

Displacement Map은 실제 정점 위치를 이동시킵니다 (실루엣, 자기 그림자 변화).

원본
프로파일: 평평함
실제 형태 변화 없음
그림자 변화 없음
Normal Map
프로파일: 평평함
형태 그대로, 라이팅만 변화
그림자 변화 없음
Displacement
프로파일: 울퉁불퉁
실제 형태 변화!
실루엣 + 그림자 변화

표면 디테일 기법 비교

기법 실루엣 자기 그림자 비용
Normal Map ❌ 변화 없음 ❌ 낮음
Parallax Occlusion ❌ ⚠️ 가능 (제한) 중간
Displacement ✅ 변화 ✅ 높음

Part 6: 활용 사례와 최적화

Tessellation은 어디에 사용하나요?

지형, 물, 캐릭터 디테일 등에 사용됩니다. 하지만 성능 비용이 있어 신중히 사용해야 합니다.

🎯 활용 사례

지형: 높이 맵 Displacement, 거리 기반 LOD

물/파도: 동적 파형 생성

캐릭터: 근접 시 디테일 증가

곡면: Bezier/B-Spline 패치

화면 공간 기반 Adaptive Tessellation
// 화면에서 차지하는 픽셀 수 기반 Factor
float ScreenSpaceTessFactor(float3 p0, float3 p1)
{
    // 월드 좌표 → 클립 좌표
    float4 clip0 = mul(float4(p0, 1), viewProjMatrix);
    float4 clip1 = mul(float4(p1, 1), viewProjMatrix);

    // NDC로 변환
    float2 ndc0 = clip0.xy / clip0.w;
    float2 ndc1 = clip1.xy / clip1.w;

    // 화면 공간에서 엣지 길이 (픽셀)
    float2 screenSize = float2(screenWidth, screenHeight);
    float edgeLength = length((ndc0 - ndc1) * screenSize * 0.5);

    // 목표: edgeLength 픽셀당 N개 정점
    float targetPixelsPerEdge = 16.0;
    return max(1, edgeLength / targetPixelsPerEdge);
}

// 절두체 컬링 (보이지 않는 패치 스킵)
bool IsPatchVisible(float3 p0, float3 p1, float3 p2)
{
    // 패치의 바운딩 박스가 절두체 안에 있는지 체크
    float3 minP = min(min(p0, p1), p2);
    float3 maxP = max(max(p0, p1), p2);

    return FrustumCheck(minP, maxP);
}

HS_CONSTANT_OUTPUT PatchConstantFunction(...)
{
    // 컬링 체크
    if (!IsPatchVisible(patch[0].position,
                        patch[1].position,
                        patch[2].position))
    {
        // 보이지 않으면 분할 안 함
        output.edges[0] = output.edges[1] = output.edges[2] = 0;
        output.inside = 0;
        return output;
    }

    // 화면 공간 기반 Factor
    output.edges[0] = ScreenSpaceTessFactor(patch[1].position, patch[2].position);
    // ...
}
최적화 설명
거리 기반 Factor 멀수록 적게 분할
화면 공간 기반 픽셀 밀도에 맞춰 분할
절두체 컬링 보이지 않는 패치 스킵
Back-face 컬링 뒷면 패치 스킵
Max Factor 제한 과도한 분할 방지

📌 핵심 정리

  • Tessellation: GPU에서 동적으로 폴리곤 분할, 메모리 대역폭 절약
  • Hull Shader: Tessellation Factor 결정 (거리/화면 기반)
  • Tessellator: 고정 기능, 정점과 토폴로지 생성
  • Domain Shader: Barycentric 보간, Displacement 적용
  • Displacement: Normal Map과 달리 실제 형태 변화, 실루엣/그림자 영향
