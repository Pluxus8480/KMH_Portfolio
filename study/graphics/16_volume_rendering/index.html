Volume Rendering과 Fog 효과

Part 1: 볼륨 효과의 필요성

왜 안개나 연기를 특별히 처리해야 하나요?

안개, 연기, 빛줄기 등은 표면이 아닌 공간 자체에 존재합니다. 일반 메시로는 표현하기 어렵습니다.

Volumetric Rendering은 빛이 매체(공기, 먼지)를 통과하며 산란/흡수되는 현상을 시뮬레이션합니다.

→
안개/매체
산란 + 흡수 발생
→
카메라
↓

다른 방향의 빛이 시선으로 산란
Out-Scattering
시선 방향 빛이 다른 곳으로 산란
Absorption
빛이 매체에 흡수됨
↓
최종 색상 = 원래 색상 x 투과율 + 산란광

Part 2: 거리 기반 Fog

가장 간단한 안개 구현은?

거리 기반 Fog는 카메라와의 거리에 따라 안개색을 블렌딩합니다. 계산이 간단하지만 사실적이지 않습니다.

거리 기반 Fog
// 선형 Fog
float LinearFog(float distance)
{
    return saturate((fogEnd - distance) / (fogEnd - fogStart));
}

// 지수 Fog
float ExponentialFog(float distance)
{
    return exp(-distance * fogDensity);
}

// 지수 제곱 Fog (더 급격한 변화)
float ExponentialSquaredFog(float distance)
{
    float f = distance * fogDensity;
    return exp(-f * f);
}

// 적용
float4 ApplyFog(float4 sceneColor, float3 worldPos)
{
    float distance = length(worldPos - cameraPos);
    float fogFactor = ExponentialFog(distance);

    // 블렌딩: 멀수록 fogColor에 가까워짐
    return lerp(fogColor, sceneColor, fogFactor);
}
Fog 유형 공식 특징
Linear (end - d) / (end - start) 단순, 급격한 전환
Exponential e^(-d × density) 자연스러운 감쇠
Exp Squared e^(-(d × density)²) 가까이는 맑고 멀리는 짙음

Part 3: Height Fog

골짜기에만 안개가 깔리게 하려면?

Height Fog는 높이에 따라 밀도가 변합니다. 낮은 곳은 짙고 높은 곳은 옅은 안개를 표현합니다.

높은 곳: 맑음
↓ fogHeight
옅은 안개
↓
중간 안개
↓
짙은 안개 (지면)
↓
밀도 = baseDensity x e^(-높이 x falloff)

Height Fog 밀도 분포

Height Fog 구현
// 높이 기반 밀도 계산
float GetHeightFogDensity(float3 worldPos)
{
    float height = worldPos.y - fogBaseHeight;

    // 높이에 따른 지수 감쇠
    float density = fogDensity * exp(-height * fogFalloff);

    return max(density, 0);
}

// Ray Marching을 통한 적분
float CalculateHeightFog(float3 startPos, float3 endPos)
{
    float3 rayDir = endPos - startPos;
    float rayLength = length(rayDir);
    rayDir /= rayLength;

    const int steps = 16;
    float stepSize = rayLength / steps;

    float totalDensity = 0;

    for (int i = 0; i < steps; i++)
    {
        float3 samplePos = startPos + rayDir * (i + 0.5) * stepSize;
        totalDensity += GetHeightFogDensity(samplePos) * stepSize;
    }

    // Beer's Law: 투과율 = e^(-광학 밀도)
    return exp(-totalDensity);
}

// 해석적 적분 (더 빠름)
float CalculateHeightFogAnalytic(float3 startPos, float3 endPos)
{
    float heightStart = startPos.y - fogBaseHeight;
    float heightEnd = endPos.y - fogBaseHeight;

    // 적분 결과 (간략화)
    float integral = fogDensity / fogFalloff *
        (exp(-heightStart * fogFalloff) - exp(-heightEnd * fogFalloff));

    return exp(-integral);
}

Part 4: Volumetric Fog (God Rays)

빛줄기(God Rays)는 어떻게 구현하나요?

Volumetric Fog는 빛이 공기 입자에 의해 산란되는 현상을 시뮬레이션합니다. Ray Marching으로 구현합니다.

태양 (빛줄기)
→
그림자 생성 물체
빛 차단 → 그림자 영역 생성
→
카메라
↓
Ray Marching
카메라에서 각 픽셀로 레이를 쏘고
일정 간격마다 라이트 가시성 체크
→ 가시적인 빛 누적 = 빛줄기(God Rays) 효과

Volumetric Light 원리

Volumetric Fog Ray Marching
// Volumetric Fog (Post-Process)
float4 VolumetricFog_PS(float2 uv : TEXCOORD) : SV_Target
{
    float depth = depthBuffer.Sample(pointSamp, uv).r;
    float3 worldPos = ReconstructWorldPosition(uv, depth);
    float3 rayStart = cameraPos;
    float3 rayEnd = worldPos;

    float3 rayDir = normalize(rayEnd - rayStart);
    float rayLength = length(rayEnd - rayStart);

    const int numSteps = 64;
    float stepSize = rayLength / numSteps;

    float3 scatteredLight = 0;
    float transmittance = 1;

    // Ray Marching
    for (int i = 0; i < numSteps; i++)
    {
        float3 samplePos = rayStart + rayDir * (i + 0.5) * stepSize;

        // 1. 매체 밀도
        float density = GetFogDensity(samplePos);

        // 2. 빛 가시성 (그림자 맵 샘플)
        float lightVisibility = SampleShadowMap(samplePos);

        // 3. In-Scattering (빛이 시선으로 산란)
        float3 inScattering = lightColor * lightVisibility * density;

        // Phase Function (Henyey-Greenstein)
        float3 lightDir = normalize(lightPosition - samplePos);
        float phase = HenyeyGreenstein(dot(rayDir, -lightDir), 0.3);
        inScattering *= phase;

        // 4. 누적
        float extinction = density * extinctionCoeff;
        float stepTransmittance = exp(-extinction * stepSize);

        scatteredLight += inScattering * transmittance * stepSize;
        transmittance *= stepTransmittance;

        // 조기 종료
        if (transmittance < 0.01) break;
    }

    // 원본 씬과 합성
    float3 sceneColor = sceneTexture.Sample(linearSamp, uv).rgb;
    float3 finalColor = sceneColor * transmittance + scatteredLight;

    return float4(finalColor, 1);
}

// Henyey-Greenstein Phase Function
float HenyeyGreenstein(float cosTheta, float g)
{
    float g2 = g * g;
    return (1 - g2) / (4 * PI * pow(1 + g2 - 2 * g * cosTheta, 1.5));
}

Part 5: Froxel 기반 최적화

Ray Marching이 너무 비싼데 최적화 방법은?

Froxel (Frustum + Voxel)은 절두체를 3D 그리드로 나눕니다. 한 번의 Compute Pass로 전체 볼륨을 계산하고 재사용합니다.

카메라 → Froxel 구조 (Frustum + Voxel)
↓
Z0
가까움
Z1
중간
Z2
중간
Z3
멀리
↓
각 Froxel에 미리 Fog 계산 결과 저장
픽셀 셰이더에서 3D 텍스처 샘플링만 하면 됨

Froxel 볼륨 분할

Froxel Volumetric Fog
// Froxel 볼륨 (3D 텍스처)
RWTexture3D froxelVolume : register(u0);  // RGB = 산란광, A = 투과율

[numthreads(8, 8, 1)]
void CS_InjectFog(uint3 id : SV_DispatchThreadID)
{
    // Froxel → 월드 좌표
    float3 worldPos = FroxelToWorld(id);

    // 밀도와 산란 계산
    float density = GetFogDensity(worldPos);
    float3 scattering = CalculateScattering(worldPos);

    froxelVolume[id] = float4(scattering, density);
}

// Temporal Reprojection (시간적 재사용)
[numthreads(8, 8, 8)]
void CS_TemporalAccumulate(uint3 id : SV_DispatchThreadID)
{
    // 현재 Froxel 값
    float4 current = froxelVolume[id];

    // 이전 프레임 위치 계산
    float3 worldPos = FroxelToWorld(id);
    float3 prevUVW = WorldToPrevFroxel(worldPos);

    // 이전 프레임 샘플
    float4 history = prevFroxelVolume.SampleLevel(linearSamp, prevUVW, 0);

    // 블렌딩 (90% 이전 + 10% 현재)
    froxelVolume[id] = lerp(current, history, 0.9);
}

// 레이 마칭 (단순화)
[numthreads(8, 8, 1)]
void CS_RayMarch(uint3 id : SV_DispatchThreadID)
{
    float3 scatteredLight = 0;
    float transmittance = 1;

    // 뒤에서 앞으로 누적
    for (int z = froxelDepth - 1; z >= 0; z--)
    {
        float4 froxel = froxelVolume[uint3(id.xy, z)];
        float3 scattering = froxel.rgb;
        float density = froxel.a;

        float extinction = density * extinctionCoeff * sliceThickness;
        float sliceTransmittance = exp(-extinction);

        scatteredLight += scattering * transmittance;
        transmittance *= sliceTransmittance;
    }

    integratedFog[id.xy] = float4(scatteredLight, transmittance);
}

Part 6: 대기 산란 (Atmospheric Scattering)

하늘의 파란색과 석양의 붉은색은 어떻게 표현하나요?

Rayleigh/Mie 산란으로 대기 효과를 시뮬레이션합니다. 파장에 따라 산란량이 다릅니다.

산란 유형 원인 특징
Rayleigh 공기 분자 짧은 파장(파랑) 더 많이 산란
Mie 먼지, 에어로졸 모든 파장 비슷, 전방 산란 강함
대기 산란 (간략화)
// Rayleigh 산란 계수 (파장 의존)
float3 RayleighCoeff()
{
    // λ^-4 의존성 → 파랑 > 초록 > 빨강
    return float3(5.8e-6, 13.5e-6, 33.1e-6);
}

// Mie 산란 계수
float MieCoeff()
{
    return 21e-6;  // 파장 독립
}

// 대기 산란 계산
float3 AtmosphericScattering(float3 rayDir, float3 sunDir)
{
    float cosTheta = dot(rayDir, sunDir);

    // Rayleigh Phase
    float rayleighPhase = 3.0 / (16.0 * PI) * (1 + cosTheta * cosTheta);

    // Mie Phase (전방 산란 강조)
    float miePhase = HenyeyGreenstein(cosTheta, 0.76);

    // 광학 깊이 (간략화)
    float opticalDepth = CalculateOpticalDepth(rayDir);

    // 산란광
    float3 rayleigh = RayleighCoeff() * rayleighPhase;
    float3 mie = MieCoeff() * miePhase;

    float3 scattering = (rayleigh + mie) * sunIntensity;

    return scattering * (1 - exp(-opticalDepth));
}

📌 핵심 정리

  • Volume Rendering: 공간 자체의 빛 산란/흡수 시뮬레이션
  • Height Fog: 높이에 따른 밀도 변화, 해석적 적분 가능
  • Volumetric Fog: Ray Marching + 그림자 = God Rays
  • Froxel: 3D 그리드로 볼륨 정보 사전 계산, Temporal 재사용
  • Atmospheric: Rayleigh(파랑) + Mie(흰색) 산란
