Instancing과 Batching 최적화

Part 1: Draw Call의 비용

왜 Draw Call을 줄여야 하나요?

각 Draw Call은 CPU와 GPU 사이의 통신 비용이 발생합니다. 상태 변경, 드라이버 검증, 커맨드 버퍼 전송 등이 필요합니다.

오브젝트가 1000개라면 1000번의 Draw Call? CPU 병목이 발생합니다.

CPU 작업 (각 Draw Call마다)
1. 상태 검증 (Validation)
2. 셰이더/텍스처 바인딩 변경
3. 상수 버퍼 업데이트
4. 드라이버 API 호출
5. 커맨드 버퍼에 기록
↓
GPU: 실제 렌더링 (상대적으로 빠름)

Draw Call이 많으면 → CPU가 병목 / Draw Call이 적으면 → GPU 활용 극대화

Draw Call의 CPU 오버헤드

Part 2: Static Batching

Static Batching은 어떻게 동작하나요?

움직이지 않는 오브젝트들의 메시를 하나의 큰 메시로 합칩니다. 빌드 타임 또는 로드 타임에 수행됩니다.

메모리는 증가하지만 Draw Call이 크게 감소합니다.

Static Batching 전:

오브젝트 A
Draw 1
오브젝트 B
Draw 2
오브젝트 C
Draw 3

같은 머티리얼이라도 3번의 Draw Call

↓

Static Batching 후:

Combined Mesh (A + B + C)
Draw 1 — 1번의 Draw Call로 모두 렌더링

Static Batching 개념

Static Batching 구현 (개념)
// 같은 머티리얼을 사용하는 Static Mesh 합치기
void StaticBatch(std::vector& meshes)
{
    std::vector combinedVertices;
    std::vector combinedIndices;

    uint32_t baseVertex = 0;

    for (StaticMesh* mesh : meshes)
    {
        // 월드 변환 적용 (더 이상 움직이지 않으므로)
        Matrix worldMatrix = mesh->GetWorldMatrix();

        for (const Vertex& v : mesh->vertices)
        {
            Vertex transformed = v;
            transformed.position = worldMatrix * v.position;
            transformed.normal = worldMatrix.GetRotation() * v.normal;
            combinedVertices.push_back(transformed);
        }

        for (uint32_t index : mesh->indices)
        {
            combinedIndices.push_back(index + baseVertex);
        }

        baseVertex += mesh->vertices.size();
    }

    // 하나의 버퍼로 생성
    CreateCombinedMesh(combinedVertices, combinedIndices);
}
🎯 Static Batching의 트레이드오프

장점: Draw Call 대폭 감소, 렌더링 성능 향상

단점: 메모리 증가 (각 인스턴스 정점이 복사됨), 오브젝트 개별 이동 불가

Part 3: Dynamic Batching

움직이는 오브젝트도 배칭할 수 있나요?

Dynamic Batching은 매 프레임 CPU에서 메시를 합칩니다. 작은 메시에만 효과적이고, 오버헤드가 발생할 수 있습니다.

항목 Static Batching Dynamic Batching
적용 시점 빌드/로드 타임 매 프레임
오브젝트 이동 불가 가능
CPU 오버헤드 없음 있음 (정점 변환)
메모리 증가 임시 버퍼
적합한 대상 배경, 건물 작은 동적 오브젝트

Part 4: GPU Instancing

같은 메시를 수천 개 그려야 한다면?

GPU Instancing이 정답입니다! 하나의 Draw Call로 동일한 메시를 수천 번 그립니다. 각 인스턴스의 변환 정보만 버퍼로 전달합니다.

Mesh Data (1개)
Vertex Buffer: Position, Normal, UV, Tangent
Index Buffer: 0, 1, 2, ...
Instance Data (N개)
World Matrix [0]
World Matrix [1]
...
World Matrix [N-1]
+ Color, Scale, etc.
↓
DrawIndexedInstanced(N)
1번의 Draw Call로 N개 렌더링

GPU Instancing 데이터 구조

GPU Instancing - C++ 설정
// 인스턴스 데이터 구조체
struct InstanceData
{
    XMFLOAT4X4 worldMatrix;
    XMFLOAT4 color;  // 인스턴스별 색상
};

// 인스턴스 버퍼 생성
void CreateInstanceBuffer(int instanceCount)
{
    std::vector instances(instanceCount);

    // 각 인스턴스 데이터 설정
    for (int i = 0; i < instanceCount; i++)
    {
        instances[i].worldMatrix = CalculateWorldMatrix(i);
        instances[i].color = GetInstanceColor(i);
    }

    D3D11_BUFFER_DESC desc = {};
    desc.ByteWidth = sizeof(InstanceData) * instanceCount;
    desc.Usage = D3D11_USAGE_DYNAMIC;
    desc.BindFlags = D3D11_BIND_VERTEX_BUFFER;
    desc.CPUAccessFlags = D3D11_CPU_ACCESS_WRITE;

    D3D11_SUBRESOURCE_DATA initData = {};
    initData.pSysMem = instances.data();

    device->CreateBuffer(&desc, &initData, &instanceBuffer);
}

// 렌더링
void RenderInstanced()
{
    // 버텍스 버퍼 + 인스턴스 버퍼 바인딩
    ID3D11Buffer* buffers[] = { vertexBuffer, instanceBuffer };
    UINT strides[] = { sizeof(Vertex), sizeof(InstanceData) };
    UINT offsets[] = { 0, 0 };

    context->IASetVertexBuffers(0, 2, buffers, strides, offsets);
    context->IASetIndexBuffer(indexBuffer, DXGI_FORMAT_R32_UINT, 0);

    // 인스턴스 드로우
    context->DrawIndexedInstanced(
        indexCount,      // 인덱스 수
        instanceCount,   // 인스턴스 수
        0, 0, 0
    );
}

// 입력 구조체
struct VSInput
{
    // Per-Vertex 데이터
    float3 position : POSITION;
    float3 normal   : NORMAL;
    float2 uv       : TEXCOORD0;

    // Per-Instance 데이터
    float4x4 worldMatrix : WORLD;      // 인스턴스별 월드 행렬
    float4 instanceColor : INST_COLOR; // 인스턴스별 색상
    uint instanceID : SV_InstanceID;   // 시스템 제공 ID
};

struct PSInput
{
    float4 position : SV_Position;
    float3 worldPos : TEXCOORD0;
    float3 normal   : TEXCOORD1;
    float2 uv       : TEXCOORD2;
    float4 color    : TEXCOORD3;
};

PSInput VS_Instanced(VSInput input)
{
    PSInput output;

    // 인스턴스별 월드 행렬 사용
    float4 worldPos = mul(float4(input.position, 1), input.worldMatrix);
    output.position = mul(worldPos, viewProj);
    output.worldPos = worldPos.xyz;
    output.normal = mul(input.normal, (float3x3)input.worldMatrix);
    output.uv = input.uv;
    output.color = input.instanceColor;

    return output;
}

Part 5: Indirect Drawing

인스턴스 수를 GPU에서 결정하고 싶다면?

Indirect Drawing을 사용합니다. GPU가 컬링 결과에 따라 실제로 그릴 인스턴스 수를 결정합니다. CPU 라운드트립이 없습니다.

Compute Shader (GPU Culling)
For each instance:
  if (FrustumCull(instance) == VISIBLE)
    AppendToVisibleBuffer(instance)
    atomicAdd(drawArgs.instanceCount, 1)
↓
Indirect Arguments Buffer (GPU에서 작성)
IndexCountPerInstance: 36
InstanceCount: ??? (GPU가 결정)
StartIndexLocation: 0
BaseVertexLocation: 0
StartInstanceLocation: 0
↓
DrawIndexedInstancedIndirect()
CPU는 인스턴스 수를 모르고 그냥 호출

GPU-Driven Rendering

Indirect Drawing 구현
// Indirect Arguments 구조체
struct DrawIndexedInstancedIndirectArgs
{
    UINT IndexCountPerInstance;
    UINT InstanceCount;
    UINT StartIndexLocation;
    UINT BaseVertexLocation;
    UINT StartInstanceLocation;
};

// Indirect 버퍼 생성
void CreateIndirectBuffer()
{
    D3D11_BUFFER_DESC desc = {};
    desc.ByteWidth = sizeof(DrawIndexedInstancedIndirectArgs);
    desc.Usage = D3D11_USAGE_DEFAULT;
    desc.BindFlags = D3D11_BIND_UNORDERED_ACCESS;
    desc.MiscFlags = D3D11_RESOURCE_MISC_DRAWINDIRECT_ARGS;

    device->CreateBuffer(&desc, nullptr, &indirectArgsBuffer);
}

// 렌더링
void RenderIndirect()
{
    // 1. Compute Shader로 GPU 컬링
    context->CSSetShader(cullingShader, nullptr, 0);
    context->Dispatch(instanceCount / 64 + 1, 1, 1);

    // 2. Indirect Draw
    context->DrawIndexedInstancedIndirect(
        indirectArgsBuffer,
        0  // offset
    );
}

Part 6: 최적화 가이드라인

어떤 상황에서 어떤 기법을 써야 하나요?

상황에 따라 최적의 기법이 다릅니다:

상황 추천 기법
움직이지 않는 배경 오브젝트 Static Batching
같은 메시, 다른 위치 (풀, 나무) GPU Instancing
작은 동적 오브젝트 Dynamic Batching
대규모 인스턴싱 + GPU 컬링 Indirect Drawing
다양한 메시, 같은 머티리얼 SRP Batcher / Material Batching
🎯 성능 팁

Draw Call 목표: 모바일 100-300, PC 1000-3000

인스턴싱 최소 수: 10개 이상부터 효과적

주의: 배칭으로 합친 메시는 개별 컬링이 안됨

📌 핵심 정리

  • Draw Call은 CPU 오버헤드가 크므로 줄이는 것이 중요
  • Static Batching: 정적 오브젝트를 하나의 메시로 합침
  • Dynamic Batching: 작은 동적 오브젝트를 매 프레임 합침
  • GPU Instancing: 같은 메시를 1 Draw Call로 수천 개 렌더링
  • Indirect Drawing: GPU가 인스턴스 수를 결정, GPU-Driven 렌더링
