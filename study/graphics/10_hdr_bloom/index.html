눈부신 빛의 표현

HDR 렌더링과 Bloom 효과

Part 1: HDR이란?

왜 일반 렌더링으로는 밝은 빛을 표현할 수 없나요?

일반 LDR(Low Dynamic Range)은 색상을 0~1 범위로 제한합니다. 하지만 현실의 밝기는 태양(10만 lux)부터 달빛(0.1 lux)까지 엄청난 범위를 가집니다.

HDR은 1.0을 초과하는 밝기를 저장해, 나중에 효과(Bloom, 노출 조절)에 활용합니다.

현실 세계
0.0001 (달빛). . . . . . . .100,000+ (태양)
LDR (8비트)
0.0 (검정). .1.0 (흰색, 클램프)
HDR (16비트+)
0.0 (검정). . . . . . . .무한한 밝기
HDR → 1.0 초과 값 → Bloom, 노출 조절에 활용

Dynamic Range 차이

🎯 비유로 이해하기

LDR은 "볼륨 최대 10"인 스피커와 같습니다.

HDR은 "볼륨 100까지 있지만, 나중에 10으로 줄여서 틀 수 있는" 스피커입니다. 그 사이에서 조절이 가능합니다.

Part 2: HDR 렌더 타겟

HDR 렌더링을 위한 설정은?

8비트 포맷(R8G8B8A8) 대신 16비트/32비트 부동소수점 포맷을 사용합니다.

포맷 비트/채널 범위 용도
R8G8B8A8_UNORM 8bit 0~1 LDR, UI
R16G16B16A16_FLOAT 16bit ±65504 HDR (권장)
R32G32B32A32_FLOAT 32bit ±3.4×10³⁸ 고정밀 연산
R11G11B10_FLOAT 11/11/10bit 양수만 HDR (메모리 절약)
HDR 렌더 타겟 생성 (DirectX 11)
// HDR 렌더 타겟 텍스처 생성
D3D11_TEXTURE2D_DESC hdrDesc = {};
hdrDesc.Width = screenWidth;
hdrDesc.Height = screenHeight;
hdrDesc.MipLevels = 1;
hdrDesc.ArraySize = 1;
hdrDesc.Format = DXGI_FORMAT_R16G16B16A16_FLOAT;  // HDR 포맷!
hdrDesc.SampleDesc.Count = 1;
hdrDesc.Usage = D3D11_USAGE_DEFAULT;
hdrDesc.BindFlags = D3D11_BIND_RENDER_TARGET | D3D11_BIND_SHADER_RESOURCE;

ID3D11Texture2D* hdrTexture;
device->CreateTexture2D(&hdrDesc, nullptr, &hdrTexture);

// Render Target View
ID3D11RenderTargetView* hdrRTV;
device->CreateRenderTargetView(hdrTexture, nullptr, &hdrRTV);

// Shader Resource View (Post-Processing용)
ID3D11ShaderResourceView* hdrSRV;
device->CreateShaderResourceView(hdrTexture, nullptr, &hdrSRV);

Part 3: Bloom 효과

밝은 부분이 번지는 효과는 어떻게 만드나요?

Bloom은 HDR의 밝은 영역(1.0 초과)을 추출해 블러 처리 후 원본에 합성합니다.

눈이나 카메라 렌즈에서 빛이 산란되는 현상을 모방합니다.

HDR Scene
태양(5.0) / 조명(2.0) / 일반 오브젝트(0.3~0.8)
↓
Threshold
밝은 부분 추출
↓
Downsample + Blur
↓
Upsample + Blur
Original
원본 유지
↓
Composite (합성)
↓
Final Output

Bloom 처리 단계

Bloom - Brightness Threshold
// 1단계: 밝은 픽셀만 추출
float4 BrightnessThreshold_PS(float2 uv : TEXCOORD) : SV_Target
{
    float3 color = hdrTexture.Sample(linearSamp, uv).rgb;

    // 휘도(Luminance) 계산
    float luminance = dot(color, float3(0.2126, 0.7152, 0.0722));

    // Threshold 이상만 추출
    float threshold = 1.0;
    float softThreshold = 0.5;

    // Soft Knee (부드러운 전환)
    float knee = threshold * softThreshold;
    float soft = luminance - threshold + knee;
    soft = clamp(soft, 0, knee * 2);
    soft = soft * soft / (4 * knee + 0.00001);

    float contribution = max(soft, luminance - threshold);
    contribution /= max(luminance, 0.00001);

    return float4(color * contribution, 1);
}
Bloom - Gaussian Blur (분리 가능)
// 가우시안 블러 가중치 (9탭)
static const float weights[5] = {
    0.227027, 0.1945946, 0.1216216, 0.054054, 0.016216
};

// 수평 블러
float4 BlurHorizontal_PS(float2 uv : TEXCOORD) : SV_Target
{
    float3 result = bloomTexture.Sample(linearSamp, uv).rgb * weights[0];

    for (int i = 1; i < 5; i++)
    {
        float2 offset = float2(texelSize.x * i, 0);
        result += bloomTexture.Sample(linearSamp, uv + offset).rgb * weights[i];
        result += bloomTexture.Sample(linearSamp, uv - offset).rgb * weights[i];
    }

    return float4(result, 1);
}

// 수직 블러
float4 BlurVertical_PS(float2 uv : TEXCOORD) : SV_Target
{
    float3 result = blurHTexture.Sample(linearSamp, uv).rgb * weights[0];

    for (int i = 1; i < 5; i++)
    {
        float2 offset = float2(0, texelSize.y * i);
        result += blurHTexture.Sample(linearSamp, uv + offset).rgb * weights[i];
        result += blurHTexture.Sample(linearSamp, uv - offset).rgb * weights[i];
    }

    return float4(result, 1);
}

Part 4: 다운샘플/업샘플 체인

넓은 범위의 Bloom은 어떻게 구현하나요?

MIP 체인을 사용합니다. 다운샘플하면서 블러하고, 업샘플하면서 누적합니다. 적은 샘플로 넓은 블러를 얻습니다.

Progressive Bloom 체인

Bloom MIP Chain 구현
// 다운샘플 + 블러 (Karis Average 포함)
float4 Downsample_PS(float2 uv : TEXCOORD) : SV_Target
{
    // 13탭 다운샘플 (고품질)
    float3 a = sourceTexture.Sample(linearSamp, uv + float2(-2, -2) * texelSize).rgb;
    float3 b = sourceTexture.Sample(linearSamp, uv + float2( 0, -2) * texelSize).rgb;
    float3 c = sourceTexture.Sample(linearSamp, uv + float2( 2, -2) * texelSize).rgb;
    // ... (13개 샘플)

    // Weighted average with Karis average to reduce fireflies
    float3 result = (a + c + g + i) * 0.125 * 0.5;
    result += (b + d + f + h) * 0.25 * 0.5;
    result += e * 0.5;

    return float4(result, 1);
}

// 업샘플 + 이전 레벨 합성
float4 Upsample_PS(float2 uv : TEXCOORD) : SV_Target
{
    // Tent 필터 (3×3) 업샘플
    float3 a = lowerMip.Sample(linearSamp, uv + float2(-1, -1) * texelSize).rgb;
    float3 b = lowerMip.Sample(linearSamp, uv + float2( 0, -1) * texelSize).rgb;
    // ...

    float3 upsampled = (a + c + g + i) / 16.0
                     + (b + d + f + h) / 8.0
                     + e / 4.0;

    // 현재 레벨과 합성
    float3 current = currentMip.Sample(linearSamp, uv).rgb;

    return float4(current + upsampled * bloomIntensity, 1);
}

Part 5: 최종 합성과 Tone Mapping

HDR을 모니터에 출력하려면?

대부분의 모니터는 LDR(0~1)만 표시합니다. Tone Mapping으로 HDR 범위를 압축합니다.

최종 합성 + Tone Mapping
// Bloom 합성 + Tone Mapping
float4 FinalComposite_PS(float2 uv : TEXCOORD) : SV_Target
{
    // HDR 씬
    float3 hdrColor = hdrSceneTexture.Sample(linearSamp, uv).rgb;

    // Bloom 추가
    float3 bloom = bloomTexture.Sample(linearSamp, uv).rgb;
    hdrColor += bloom * bloomStrength;

    // 노출 조절 (Exposure)
    hdrColor *= exposure;

    // Tone Mapping (ACES Filmic)
    float3 ldrColor = ACESToneMapping(hdrColor);

    // 감마 보정
    ldrColor = pow(ldrColor, 1.0 / 2.2);

    return float4(ldrColor, 1);
}

// ACES Tone Mapping
float3 ACESToneMapping(float3 x)
{
    float a = 2.51;
    float b = 0.03;
    float c = 2.43;
    float d = 0.59;
    float e = 0.14;
    return saturate((x * (a * x + b)) / (x * (c * x + d) + e));
}
Tone Mapping 특징
Reinhard 간단, 약간 탁한 느낌
ACES Filmic 영화 같은 룩, 콘트라스트 좋음
Uncharted 2 게임에 적합, 디테일 보존
AgX 최신 기법, 자연스러운 색상

Part 6: 자동 노출 (Auto Exposure)

밝은 곳에서 어두운 곳으로 이동할 때 눈이 적응하는 효과는?

자동 노출은 화면의 평균 휘도를 계산해 노출을 조절합니다. 눈의 적응 현상을 모방합니다.

평균 휘도 계산 (Log Average)
// 1단계: 휘도 계산 + 다운샘플
// 최종적으로 1x1 텍스처로 감소

float4 LuminanceDownsample_PS(float2 uv : TEXCOORD) : SV_Target
{
    float3 color = hdrTexture.Sample(linearSamp, uv).rgb;

    // 휘도 계산
    float luminance = dot(color, float3(0.2126, 0.7152, 0.0722));

    // Log 공간에서 평균 (0 방지)
    float logLum = log(luminance + 0.0001);

    return float4(logLum, luminance, 0, 1);
}

// 2단계: 적응 속도 적용
float AdaptLuminance(float currentLum, float targetLum, float deltaTime)
{
    float adaptSpeed = (targetLum > currentLum) ? 2.0 : 1.0;  // 밝아지면 빠르게
    return currentLum + (targetLum - currentLum) * (1 - exp(-deltaTime * adaptSpeed));
}

// 3단계: 노출 계산
float CalculateExposure(float avgLuminance)
{
    // 목표 휘도 기준 노출 계산
    float keyValue = 0.18;  // 중간 회색
    float exposure = keyValue / avgLuminance;

    // 범위 제한
    return clamp(exposure, minExposure, maxExposure);
}

📌 핵심 정리

  • HDR은 1.0 초과 밝기를 저장, 16비트 부동소수점 포맷 사용
  • Bloom = Threshold → Downsample/Blur → Upsample → Composite
  • MIP 체인으로 넓은 범위 Bloom을 효율적으로 계산
  • Tone Mapping으로 HDR → LDR 변환 (ACES, Reinhard 등)
  • 자동 노출로 동적 밝기 적응, 눈의 반응 시뮬레이션
