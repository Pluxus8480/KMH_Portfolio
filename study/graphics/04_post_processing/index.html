Part 1: Post-Processing이란?

Post-Processing은 언제 실행되나요?

Post-Processing은 3D 렌더링이 끝난 후, 2D 이미지에 적용되는 효과입니다. 전체 화면을 대상으로 픽셀 단위 처리를 합니다.

마치 사진을 찍은 후 포토샵에서 보정하는 것과 같습니다.

↓
Post-Processing Chain
SSAO → SSR → DoF → Motion Blur → Bloom → Color Grading → Tone Mapping → AA → UI
↓
Final Frame Buffer

Part 3: Bloom 구현

Bloom은 어떻게 만드나요?

Bloom은 밝은 영역이 번지는 효과입니다. 3단계로 구현됩니다:

1. Threshold: 밝은 픽셀만 추출

2. Blur: 가우시안 블러 적용

3. Composite: 원본과 합성

HLSL - Bloom (간략화)
// 1. Brightness Threshold Pass
float4 BrightnessPS(float2 uv : TEXCOORD) : SV_Target
{
    float4 color = sceneTexture.Sample(sampler, uv);
    float brightness = dot(color.rgb, float3(0.2126, 0.7152, 0.0722));

    if (brightness > threshold)
        return color;
    else
        return float4(0, 0, 0, 0);
}

// 2. Gaussian Blur (Separable - 가로/세로 분리)
float4 BlurHorizontalPS(float2 uv : TEXCOORD) : SV_Target
{
    float weights[5] = {0.227, 0.194, 0.121, 0.054, 0.016};
    float3 result = brightTexture.Sample(sampler, uv).rgb * weights[0];

    for (int i = 1; i < 5; i++)
    {
        float2 offset = float2(texelSize.x * i, 0);
        result += brightTexture.Sample(sampler, uv + offset).rgb * weights[i];
        result += brightTexture.Sample(sampler, uv - offset).rgb * weights[i];
    }
    return float4(result, 1);
}

// 3. Composite
float4 CompositePS(float2 uv : TEXCOORD) : SV_Target
{
    float3 scene = sceneTexture.Sample(sampler, uv).rgb;
    float3 bloom = bloomTexture.Sample(sampler, uv).rgb;
    return float4(scene + bloom * bloomIntensity, 1);
}

Part 4: Tone Mapping

HDR을 모니터에 표시하려면?

HDR(High Dynamic Range)은 0~∞ 범위지만, 모니터는 0~1만 표시할 수 있습니다. Tone Mapping이 이 변환을 담당합니다.

다양한 Tone Mapping 연산자
// Reinhard (간단)
float3 ReinhardToneMap(float3 hdr)
{
    return hdr / (hdr + 1.0);
}

// ACES (영화 산업 표준)
float3 ACESFilmToneMap(float3 x)
{
    float a = 2.51f;
    float b = 0.03f;
    float c = 2.43f;
    float d = 0.59f;
    float e = 0.14f;
    return saturate((x * (a * x + b)) / (x * (c * x + d) + e));
}

// Uncharted 2 (게임용)
float3 Uncharted2ToneMap(float3 x)
{
    float A = 0.15, B = 0.50, C = 0.10;
    float D = 0.20, E = 0.02, F = 0.30;
    return ((x * (A * x + C * B) + D * E) /
            (x * (A * x + B) + D * F)) - E / F;
}
연산자 특징
Reinhard 간단, 약간 탁함
ACES 영화 같은 룩, 콘트라스트 좋음
Uncharted 2 게임에 적합, 디테일 보존

Part 5: 효율적인 구현

Post-Processing을 효율적으로 하려면?

몇 가지 최적화 기법:

🎯 최적화 팁

다운샘플링: Bloom 등은 1/4 해상도에서 계산 후 업샘플

효과 병합: 여러 효과를 한 패스에 (Color Grading + Tone Mapping)

Compute Shader: 복잡한 효과는 Compute로 (SSAO 등)

LUT: Color Grading은 3D LUT로 빠르게

다운샘플 + 업샘플 체인
// Bloom 다운샘플 체인
// 1920x1080 → 960x540 → 480x270 → 240x135 → 120x67

void BloomDownsample()
{
    for (int i = 0; i < MIP_COUNT; i++)
    {
        // 이전 MIP를 블러하며 다운샘플
        SetRenderTarget(bloomMip[i]);
        DrawFullscreenQuad(bloomMip[i-1], downsampleShader);
    }
}

void BloomUpsample()
{
    for (int i = MIP_COUNT - 1; i > 0; i--)
    {
        // 업샘플하며 합성
        SetRenderTarget(bloomMip[i-1]);
        DrawFullscreenQuad(bloomMip[i], upsampleShader);
    }
}

Part 6: Color Grading

영화 같은 색감은 어떻게 만드나요?

LUT (Look-Up Table)을 사용합니다. 모든 색상 변환을 3D 텍스처로 미리 계산해 둡니다.

입력 색상 (R, G, B)
↓
3D LUT (32³ 또는 64³)
RGB 값을 좌표로 사용하여 변환된 색상을 조회
예: [0.2, 0.5, 0.8] → LUT[5][16][26]
↓
출력 색상 (R', G', B')

아티스트가 Photoshop/DaVinci에서 LUT 제작 → 게임에 적용

LUT 기반 Color Grading

LUT 적용
// 3D LUT 샘플링
float3 ApplyColorGrading(float3 color)
{
    // LUT 크기 (예: 32)
    float lutSize = 32.0;

    // 색상을 LUT 좌표로 변환
    float3 uvw = color * (lutSize - 1.0) / lutSize + 0.5 / lutSize;

    // 3D 텍스처 샘플링
    return lutTexture.Sample(trilinearSampler, uvw).rgb;
}

📌 핵심 정리

  • Post-Processing은 3D 렌더링 후 2D 이미지에 적용되는 효과
  • Bloom = Threshold → Blur → Composite
  • Tone Mapping으로 HDR → LDR 변환 (ACES, Reinhard 등)
  • 다운샘플링으로 비용 절감, 효과 병합으로 패스 감소
  • Color LUT로 아티스트가 제작한 색감 빠르게 적용
