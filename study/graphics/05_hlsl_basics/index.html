HLSL 셰이더 프로그래밍 기초

Part 1: HLSL이란?

HLSL은 무엇인가요?

HLSL(High-Level Shading Language)은 DirectX용 셰이더 언어입니다. C와 비슷한 문법으로 GPU에서 실행될 코드를 작성합니다.

OpenGL은 GLSL, NVIDIA는 Cg를 사용하지만, 핵심 개념은 동일합니다.

🎯 비유로 이해하기

CPU 코드가 "혼자 열심히 일하는 전문가"라면,

GPU 셰이더는 "수천 명이 같은 일을 동시에 하는 공장"입니다.

각 픽셀(또는 정점)이 독립적으로 같은 코드를 실행합니다.

Part 2: 데이터 타입

HLSL에서 사용하는 데이터 타입은?

벡터와 행렬 타입이 기본 내장되어 있습니다:

HLSL 데이터 타입
// 스칼라 타입
float   x = 1.0f;      // 32비트 부동소수점
half    y = 1.0h;      // 16비트 (모바일 최적화)
int     i = 42;        // 정수
uint    u = 10u;       // 부호 없는 정수
bool    b = true;      // 불리언

// 벡터 타입
float2  uv = float2(0.5, 0.5);           // 2D 벡터
float3  pos = float3(1, 2, 3);           // 3D 벡터
float4  color = float4(1, 0, 0, 1);      // RGBA

// 스위즐링 (Swizzling)
float3 rgb = color.rgb;      // x, y, z로도 접근 가능
float2 yx = pos.yx;          // 순서 바꾸기
float4 rrrr = color.rrrr;    // 반복

// 행렬 타입
float4x4 worldMatrix;        // 4x4 행렬
float3x3 normalMatrix;       // 3x3 행렬

// 배열
float weights[8];
타입 용도 예시
float3 위치, 방향, 색상 position, normal, rgb
float4 동차 좌표, RGBA clipPos, rgba
float4x4 변환 행렬 MVP matrix
float2 텍스처 좌표 UV coordinates

Part 3: 셰이더 구조

기본적인 셰이더 구조는 어떻게 되나요?

Vertex Shader와 Pixel(Fragment) Shader가 기본입니다:

기본 셰이더 구조
// ========== 상수 버퍼 (CPU에서 전달받는 데이터) ==========
cbuffer PerFrame : register(b0)
{
    float4x4 viewProj;
    float3 cameraPos;
    float time;
};

cbuffer PerObject : register(b1)
{
    float4x4 world;
    float4 color;
};

// ========== 텍스처와 샘플러 ==========
Texture2D diffuseTexture : register(t0);
SamplerState linearSampler : register(s0);

// ========== 입출력 구조체 ==========
struct VSInput
{
    float3 position : POSITION;
    float3 normal   : NORMAL;
    float2 uv       : TEXCOORD0;
};

struct PSInput
{
    float4 position : SV_Position;
    float3 worldPos : TEXCOORD0;
    float3 normal   : TEXCOORD1;
    float2 uv       : TEXCOORD2;
};

// ========== Vertex Shader ==========
PSInput VS_Main(VSInput input)
{
    PSInput output;

    float4 worldPos = mul(float4(input.position, 1), world);
    output.position = mul(worldPos, viewProj);
    output.worldPos = worldPos.xyz;
    output.normal = mul(input.normal, (float3x3)world);
    output.uv = input.uv;

    return output;
}

// ========== Pixel Shader ==========
float4 PS_Main(PSInput input) : SV_Target
{
    float3 albedo = diffuseTexture.Sample(linearSampler, input.uv).rgb;
    float3 normal = normalize(input.normal);

    // 간단한 램버트 라이팅
    float3 lightDir = normalize(float3(1, 1, -1));
    float NdotL = max(dot(normal, lightDir), 0);

    float3 finalColor = albedo * NdotL;
    return float4(finalColor, 1);
}

Part 4: 시맨틱

POSITION, SV_Position 같은 건 뭔가요?

시맨틱(Semantic)은 데이터의 용도를 GPU에게 알려줍니다. 입력 데이터와 셰이더 스테이지를 연결합니다.

시맨틱 용도 사용 위치
POSITION 정점 위치 (입력) VS 입력
SV_Position 클립 공간 위치 VS 출력 / PS 입력
NORMAL 법선 벡터 VS 입력
TEXCOORD0~n 텍스처 좌표 / 커스텀 데이터 VS↔PS
SV_Target 렌더 타겟 출력 PS 출력
SV_Depth 깊이 출력 PS 출력
SV_ 접두사는 뭔가요?

SV = System Value로, GPU가 특별히 처리하는 값입니다. SV_Position은 래스터라이저가 사용하고, SV_Target은 렌더 타겟에 기록됩니다.

Part 5: 내장 함수

자주 쓰는 내장 함수는?

HLSL은 그래픽스에 유용한 함수를 많이 제공합니다:

자주 쓰는 내장 함수
// 수학 함수
float a = abs(x);           // 절대값
float s = sign(x);          // 부호 (-1, 0, 1)
float f = floor(x);         // 내림
float c = ceil(x);          // 올림
float r = frac(x);          // 소수부
float m = fmod(x, y);       // 나머지

// 보간
float l = lerp(a, b, t);    // 선형 보간
float s = smoothstep(a, b, x);  // 부드러운 보간
float h = step(edge, x);    // x >= edge ? 1 : 0

// 벡터 연산
float d = dot(a, b);        // 내적
float3 c = cross(a, b);     // 외적
float len = length(v);      // 길이
float3 n = normalize(v);    // 정규화
float dist = distance(a, b); // 거리
float3 r = reflect(i, n);   // 반사
float3 t = refract(i, n, eta); // 굴절

// 범위 제한
float c = clamp(x, min, max);   // 범위 제한
float s = saturate(x);      // clamp(x, 0, 1)
float3 m = min(a, b);       // 최소값
float3 x = max(a, b);       // 최대값

// 텍스처 샘플링
float4 c = tex.Sample(samp, uv);
float4 c = tex.SampleLevel(samp, uv, mip);
float4 c = tex.Load(int3(x, y, mip));

Part 6: 실전 예제

실제로 자주 쓰는 패턴은?

몇 가지 유용한 셰이더 패턴:

유용한 패턴들
// 1. 프레넬 효과 (가장자리 빛나기)
float fresnel = pow(1.0 - saturate(dot(normal, viewDir)), 5.0);

// 2. 디졸브 효과
float noise = noiseTexture.Sample(samp, uv).r;
clip(noise - dissolveAmount);  // 음수면 픽셀 버림

// 3. UV 스크롤
float2 scrolledUV = uv + time * float2(0.1, 0.0);

// 4. Checkerboard 패턴
float checker = fmod(floor(uv.x * 10) + floor(uv.y * 10), 2);

// 5. 림 라이트
float rim = 1.0 - saturate(dot(normal, viewDir));
float3 rimColor = rimLight * pow(rim, rimPower);

// 6. 반구 라이팅 (스카이/그라운드)
float NdotUp = dot(normal, float3(0, 1, 0)) * 0.5 + 0.5;
float3 ambient = lerp(groundColor, skyColor, NdotUp);

// 7. 트리플래너 매핑 가중치
float3 blending = abs(normal);
blending = blending / (blending.x + blending.y + blending.z);
🎯 디버깅 팁

값 시각화: return float4(value, value, value, 1);

법선 시각화: return float4(normal * 0.5 + 0.5, 1);

UV 시각화: return float4(uv, 0, 1);

깊이 시각화: return float4(depth.xxx, 1);

📌 핵심 정리

  • HLSL은 DirectX용 셰이더 언어, C와 비슷한 문법
  • float2/3/4, float4x4 등 벡터/행렬 타입 내장
  • 시맨틱으로 데이터 용도 지정 (POSITION, SV_Target 등)
  • cbuffer로 CPU 데이터 전달, Texture/Sampler로 텍스처 접근
  • 내장 함수: dot, normalize, lerp, saturate 등 그래픽스 필수 함수
