Normal Mapping과 Parallax Mapping

Part 1: 왜 노말 맵이 필요한가?

폴리곤 수를 늘리지 않고 디테일을 추가하려면?

벽돌 벽을 렌더링한다고 가정합니다. 각 벽돌의 요철을 실제 지오메트리로 표현하면 수백만 폴리곤이 필요합니다.

노말 맵은 표면의 법선 방향을 텍스처로 저장해, 평평한 면에서 울퉁불퉁한 라이팅 효과를 만듭니다.

Normal Mapping 원리:

실제 지오메트리 (평면):
─────────────────────────────
    ↑     ↑     ↑     ↑        모든 법선이 위를 향함
    N     N     N     N        → 평평한 라이팅

Normal Map 적용 후:
─────────────────────────────
   ↗    ↑    ↖    ↗    ↑      법선이 다양한 방향
   N    N    N    N    N       → 울퉁불퉁한 라이팅

실제 표면은 평면이지만,
라이팅 계산에 사용되는 법선이 다르므로
입체감 있는 결과를 얻음

노말 맵의 기본 원리

Normal Mapping은 평평한 사진에 조명 트릭을 쓰는 것입니다.

실제로 튀어나온 게 아니라, 빛이 닿는 방식만 바꿔서 눈을 속입니다.

Part 2: Normal Map 구조

노말 맵은 왜 파란색인가요?

노말 맵의 RGB 채널은 XYZ 방향을 나타냅니다. 대부분의 법선이 Z축(위쪽)을 향하므로 B(파랑) 채널 값이 높아 파랗게 보입니다.

값 범위 [0, 1]을 [-1, 1]로 변환해서 사용합니다.

Normal Map RGB → XYZ 매핑:

텍스처 값 [0, 1] → 법선 벡터 [-1, 1]
  변환식: normal = texValue * 2.0 - 1.0

RGB 채널 의미:
┌─────────────────────────────────────────┐
│  R (빨강) = X 방향 (좌우)               │
│  G (초록) = Y 방향 (상하/앞뒤)          │
│  B (파랑) = Z 방향 (표면에서 수직)      │
└─────────────────────────────────────────┘

예시:
(128, 128, 255) → (0, 0, 1)  = 위를 향함 (기본)
(255, 128, 128) → (1, 0, 0)  = 오른쪽으로 기울어짐
(0, 128, 128)   → (-1, 0, 0) = 왼쪽으로 기울어짐

Normal Map의 색상 인코딩

공간 설명 장단점
Tangent Space 표면 기준 로컬 좌표 재사용 가능, TBN 행렬 필요
Object Space 오브젝트 기준 좌표 계산 간단, 애니메이션에 부적합
World Space 월드 기준 좌표 거의 사용 안함

Part 3: Tangent Space와 TBN 행렬

Tangent Space는 무엇인가요?

각 정점마다 정의되는 로컬 좌표계입니다. T(Tangent), B(Bitangent), N(Normal) 세 축으로 구성됩니다.

노말 맵의 법선을 월드/뷰 공간으로 변환하는 데 사용합니다.

Tangent Space 좌표계:

              N (Normal - 표면에 수직)
              ↑
              │
              │
              ●───────→ T (Tangent - U 방향)
             ╱
            ╱
           ↓
          B (Bitangent - V 방향)

TBN 행렬:
┌               ┐
│ Tx  Ty  Tz   │
│ Bx  By  Bz   │   × tangentSpaceNormal
│ Nx  Ny  Nz   │
└               ┘
    = worldSpaceNormal

TBN 좌표계

TBN 행렬 계산 (C++)
// 삼각형에서 Tangent/Bitangent 계산
void CalculateTangent(
    const Vertex& v0, const Vertex& v1, const Vertex& v2,
    XMFLOAT3& tangent, XMFLOAT3& bitangent)
{
    // 엣지 벡터
    XMFLOAT3 edge1 = v1.position - v0.position;
    XMFLOAT3 edge2 = v2.position - v0.position;

    // UV 델타
    XMFLOAT2 deltaUV1 = v1.uv - v0.uv;
    XMFLOAT2 deltaUV2 = v2.uv - v0.uv;

    // Tangent/Bitangent 계산
    float f = 1.0f / (deltaUV1.x * deltaUV2.y - deltaUV2.x * deltaUV1.y);

    tangent.x = f * (deltaUV2.y * edge1.x - deltaUV1.y * edge2.x);
    tangent.y = f * (deltaUV2.y * edge1.y - deltaUV1.y * edge2.y);
    tangent.z = f * (deltaUV2.y * edge1.z - deltaUV1.y * edge2.z);

    bitangent.x = f * (-deltaUV2.x * edge1.x + deltaUV1.x * edge2.x);
    bitangent.y = f * (-deltaUV2.x * edge1.y + deltaUV1.x * edge2.y);
    bitangent.z = f * (-deltaUV2.x * edge1.z + deltaUV1.x * edge2.z);

    // 정규화
    XMStoreFloat3(&tangent, XMVector3Normalize(XMLoadFloat3(&tangent)));
    XMStoreFloat3(&bitangent, XMVector3Normalize(XMLoadFloat3(&bitangent)));
}
Normal Mapping Shader (HLSL)
// Vertex Shader
struct VSOutput
{
    float4 position : SV_Position;
    float2 uv       : TEXCOORD0;
    float3 worldPos : TEXCOORD1;
    float3x3 TBN    : TEXCOORD2;  // TBN 행렬
};

VSOutput VS_NormalMap(VSInput input)
{
    VSOutput output;

    output.position = mul(float4(input.position, 1), MVP);
    output.uv = input.uv;
    output.worldPos = mul(float4(input.position, 1), world).xyz;

    // TBN을 월드 공간으로 변환
    float3 T = normalize(mul(input.tangent, (float3x3)world));
    float3 N = normalize(mul(input.normal, (float3x3)world));
    float3 B = cross(N, T) * input.tangent.w;  // Handedness

    output.TBN = float3x3(T, B, N);

    return output;
}

// Pixel Shader
float4 PS_NormalMap(VSOutput input) : SV_Target
{
    // 노말 맵 샘플링
    float3 normalTS = normalMap.Sample(samp, input.uv).rgb;
    normalTS = normalTS * 2.0 - 1.0;  // [0,1] → [-1,1]

    // Tangent Space → World Space
    float3 normalWS = normalize(mul(normalTS, input.TBN));

    // 라이팅 계산 (월드 공간에서)
    float3 lightDir = normalize(lightPosition - input.worldPos);
    float NdotL = max(dot(normalWS, lightDir), 0);

    float3 albedo = diffuseMap.Sample(samp, input.uv).rgb;
    return float4(albedo * NdotL, 1);
}

Part 4: Parallax Mapping

노말 맵으로는 실루엣이 안 바뀌는데, 더 입체적으로 보이게 하려면?

Parallax Mapping은 높이 맵을 사용해 UV 좌표를 오프셋합니다. 시점에 따라 텍스처가 "밀려 보이는" 효과를 줍니다.

Parallax Mapping 원리:

    눈 👁️
      \
       \  view direction
        \
    ─────●─────────────  실제 표면 (height = 0)
         │
         │ offset
         │
    ─────◉─────────────  가상 표면 (height map)

원래 UV (●) 대신 오프셋된 UV (◉)에서 텍스처 샘플링
→ 깊이감 있는 효과

Parallax UV Offset

Simple Parallax Mapping
// 간단한 Parallax Mapping
float2 ParallaxMapping(float2 uv, float3 viewDirTS)
{
    // 높이 맵 샘플링
    float height = heightMap.Sample(samp, uv).r;

    // UV 오프셋 계산
    float2 offset = viewDirTS.xy / viewDirTS.z * (height * heightScale);

    return uv - offset;
}

float4 PS_Parallax(VSOutput input) : SV_Target
{
    // View Direction을 Tangent Space로 변환
    float3 viewDirWS = normalize(cameraPos - input.worldPos);
    float3 viewDirTS = mul(viewDirWS, transpose(input.TBN));

    // Parallax로 UV 오프셋
    float2 parallaxUV = ParallaxMapping(input.uv, viewDirTS);

    // 오프셋된 UV로 텍스처 샘플링
    float3 albedo = diffuseMap.Sample(samp, parallaxUV).rgb;
    float3 normal = normalMap.Sample(samp, parallaxUV).rgb * 2.0 - 1.0;

    // ... 라이팅
}

Part 5: Steep Parallax와 POM

Simple Parallax는 급경사에서 왜곡이 심한데요?

Steep Parallax Mapping은 여러 번 샘플링하여 정확한 교차점을 찾습니다. Parallax Occlusion Mapping (POM)은 이진 검색으로 더 정밀하게 계산합니다.

Steep Parallax vs Simple Parallax:

Simple (1번 샘플링):           Steep (여러 번 샘플링):
    👁️                              👁️
      \                               \
       \                               \  ●───●───●
        \                               \     ●───●
         \                               \        ◉
    ──────●──────                   ──────────────────
          ↓                         높이 맵 표면을 따라가며
       오차 발생                     교차점 ◉ 찾음

레이마칭 방식의 정밀도 향상

Parallax Occlusion Mapping (POM)
float2 ParallaxOcclusionMapping(float2 uv, float3 viewDirTS)
{
    const float minLayers = 8;
    const float maxLayers = 32;

    // 시야각에 따른 레이어 수 조절
    float numLayers = lerp(maxLayers, minLayers,
                           abs(dot(float3(0, 0, 1), viewDirTS)));

    float layerDepth = 1.0 / numLayers;
    float currentDepth = 0.0;

    float2 P = viewDirTS.xy * heightScale;
    float2 deltaUV = P / numLayers;

    float2 currentUV = uv;
    float currentHeight = heightMap.Sample(samp, currentUV).r;

    // 표면 아래로 내려갈 때까지 레이마칭
    [loop]
    while (currentDepth < currentHeight)
    {
        currentUV -= deltaUV;
        currentHeight = heightMap.Sample(samp, currentUV).r;
        currentDepth += layerDepth;
    }

    // 이진 검색으로 정밀도 향상
    float2 prevUV = currentUV + deltaUV;
    float prevDepth = currentDepth - layerDepth;
    float prevHeight = heightMap.Sample(samp, prevUV).r;

    float afterDepth = currentHeight - currentDepth;
    float beforeDepth = prevHeight - prevDepth;

    float weight = afterDepth / (afterDepth - beforeDepth);
    return lerp(currentUV, prevUV, weight);
}

// Self-Shadowing (선택)
float ParallaxSoftShadow(float2 uv, float3 lightDirTS)
{
    float shadow = 0.0;
    float2 currentUV = uv;
    float currentHeight = heightMap.Sample(samp, currentUV).r;

    float2 deltaUV = lightDirTS.xy * heightScale / 16.0;

    for (int i = 0; i < 16; i++)
    {
        currentUV += deltaUV;
        float sampledHeight = heightMap.Sample(samp, currentUV).r;

        if (sampledHeight > currentHeight + (i + 1) * 0.0625)
            shadow += 0.0625;
    }

    return 1.0 - saturate(shadow);
}
기법 샘플링 수 품질 성능
Normal Mapping 1 법선만 빠름
Simple Parallax 2 낮음 빠름
Steep Parallax 8~32 중간 중간
POM 16~64 높음 느림

// BC5 압축 (RG만 저장, B 재구성)
float3 UnpackNormalBC5(float2 packedNormal)
{
    float3 normal;
    normal.xy = packedNormal * 2.0 - 1.0;
    normal.z = sqrt(1.0 - saturate(dot(normal.xy, normal.xy)));
    return normal;
}

// DXT5nm 형식 (AG 채널 사용)
float3 UnpackNormalDXT5nm(float4 packed)
{
    float3 normal;
    normal.xy = packed.ag * 2.0 - 1.0;
    normal.z = sqrt(1.0 - saturate(dot(normal.xy, normal.xy)));
    return normal;
}

📌 핵심 정리

  • Normal Mapping: 텍스처로 법선 방향 저장, 적은 폴리곤으로 디테일 표현
  • 노말 맵은 Tangent Space에서 정의, TBN 행렬로 월드 공간 변환
  • Parallax Mapping: 높이 맵으로 UV 오프셋, 깊이감 추가
  • POM: 레이마칭으로 정밀한 높이 계산, 자체 그림자 가능
  • 노말 맵 품질은 압축 방식, 밉맵 처리, 좌표계에 주의
