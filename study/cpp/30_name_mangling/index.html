컴파일러는 함수 이름을 난도질한다

C++ Name Mangling(Decoration)의 원리와 extern "C"

Part 1: Name Mangling이란?

Name Mangling이 뭔가요?

Name Mangling은 컴파일러가 함수나 변수의 이름을 임의로 변경하는 작업입니다.

C++에서 이런 작업이 일어나는 이유는 함수 오버로딩 때문입니다. 같은 이름의 함수들이 있고 시그니처가 다를 때, 컴파일러는 이들의 이름을 내부적으로 다르게 변환합니다.

Name Mangling 예시
SomeFunc()          -> 컴파일러 내부: SomeFunc_
SomeFunc(int some)  -> 컴파일러 내부: SomeFunc_i
🎯 비유로 이해하기

학교에 "김민수"가 3명 있으면, 선생님이 "1반 김민수", "2반 김민수"처럼 구분하는 것과 같습니다. 컴파일러도 같은 이름의 함수를 인자 정보를 붙여 구분합니다.

Part 2: C에서도 Name Mangling이 있는가?

C 언어에서도 Name Mangling이 발생하나요?

C에서는 함수 오버로딩이 없기 때문에 C++ 수준의 Name Mangling이 필요 없습니다.

다만 컴파일러에 따라 앞에 _를 붙이는 등의 간단한 변환을 하기도 합니다.

대부분의 경우 "C는 간단한 Name Mangling을 한다"라는 의견이 주류입니다.

Part 3: extern "C"

extern "C"는 무엇이고 왜 사용하나요?

extern "C"는 C++ 코드를 C 스타일의 Name Mangling으로 컴파일하도록 지시하는 명령어입니다.

이를 통해 C++에서 작성한 함수를 C에서도 사용할 수 있게 됩니다.

C++
extern "C" int Some(int a)
{
    // C 링커가 이 함수를 찾을 수 있게 됨
    return a * 2;
}

// 여러 함수를 묶을 수도 있음
extern "C" {
    void Func1();
    void Func2(int x);
}

C/C++ 혼용 프로젝트나 DLL 라이브러리 제작 시 필수적으로 사용됩니다.

📌 핵심 정리

  • Name Mangling은 컴파일러가 오버로딩된 함수를 구분하기 위해 이름을 변경하는 작업
  • C++은 함수 오버로딩 때문에 복잡한 Name Mangling을 수행
  • C는 오버로딩이 없어 간단한 수준의 Mangling만 수행
  • extern "C"를 사용하면 C 스타일 Mangling으로 컴파일하여 C와의 호환성 확보
