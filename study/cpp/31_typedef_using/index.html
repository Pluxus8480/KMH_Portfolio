타입에 별명을 붙이는 두 가지 방법

C++ typedef와 using(별칭 선언)의 차이와 선택 기준

Part 1: typedef와 using, 뭐가 다른가?

typedef와 using은 어떻게 다른가요?

C++11 이후로 using을 이용한 별칭 선언이 가능해졌습니다. 기본적인 기능은 동일하지만 표현 방식이 다릅니다.

C++
// 동일한 별칭 선언
typedef std::unique_ptr<std::unordered_map<std::string, std::string>> UPtrMapSS;
using UPtrMapSS = std::unique_ptr<std::unordered_map<std::string, std::string>>;

Part 2: using을 권장하는 이유

왜 using 사용이 권장되나요?

1. 포인터 관여 시 가독성이 더 좋습니다.

C++ - 함수 포인터 별칭
// typedef: 별칭이 중간에 위치하여 읽기 어려움
typedef void (*FP)(int, const std::string&);

// using: 별칭 = 타입 형태로 직관적
using FP = void (*)(int, const std::string&);

2. using은 템플릿화가 가능합니다. 이것이 가장 큰 차이점입니다.

C++ - 템플릿 별칭 (using만 가능)
template<typename T>
using AllocWList = std::list<T, MyAlloc<T>>;

AllocWList<Some> some; // std::list<Some, MyAlloc<Some>>

Effective Modern C++에서도 using 사용을 권장하고 있습니다.

📌 핵심 정리

  • typedef와 using은 기본적으로 동일한 별칭 선언 기능을 제공
  • using은 별칭 = 타입 형태로 가독성이 더 좋음
  • using은 템플릿 별칭이 가능하여 typedef보다 유연함
  • C++11 이후에는 using 사용을 권장
