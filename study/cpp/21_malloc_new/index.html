메모리를 할당하는 두 가지 철학

C의 malloc/free와 C++의 new/delete 비교

Part 1: 함수 vs 연산자

malloc/free와 new/delete의 가장 큰 차이는 무엇인가요?

malloc/free는 함수이고, new/delete는 연산자입니다. 이것이 가장 근본적인 차이입니다.

🎯 비유로 이해하기

malloc/free는 "빈 땅을 사서 직접 건물을 지어야 하는 것"과 같고, new/delete는 "건설사에 의뢰하면 설계도(생성자)대로 건물을 지어주는 것"과 같습니다.

Part 2: 상세 비교

구체적으로 어떤 차이들이 있나요?

malloc/free

  • #include "stdlib.h"를 해줘야 사용 가능
  • malloc은 void*를 반환하기 때문에 해당 타입의 포인터로 형변환을 해줘야 합니다
  • 생성자를 호출할 수단이 없기에 초기값을 줄 수 없다는 단점이 있습니다
  • free 역시 소멸자를 호출하지 않고 단순히 메모리를 해제합니다
  • realloc을 사용해 메모리 크기를 조정할 수 있습니다

new/delete

  • C++ 내장 키워드로 별도 include가 필요 없습니다
  • 반환값이 해당 타입의 포인터로 자동 반환됩니다
  • 생성자를 호출해 초기값을 줄 수 있습니다
  • delete는 소멸자를 호출해 줍니다
  • 직접적으로 메모리 크기를 조정할 수 없습니다
C++ - malloc/free
#include <cstdlib>

// malloc: void* 반환 -> 형변환 필요
int* p1 = (int*)malloc(sizeof(int) * 10);
// 생성자 호출 불가, 초기화 불가
free(p1);  // 소멸자 호출 안 됨

// realloc으로 크기 조정 가능
p1 = (int*)realloc(p1, sizeof(int) * 20);
C++ - new/delete
// new: 타입에 맞는 포인터 자동 반환
int* p2 = new int[10];         // 형변환 불필요
MyClass* obj = new MyClass();  // 생성자 호출됨!
delete[] p2;
delete obj;  // 소멸자 호출됨!

Part 3: 비교 정리

한눈에 비교할 수 있게 정리해 주세요.
구분 malloc/free new/delete
분류 함수 (stdlib.h) 연산자 (내장 키워드)
반환 타입 void* (형변환 필요) 해당 타입 포인터 (자동)
생성자/소멸자 호출 안 됨 호출됨
크기 재조정 realloc 가능 불가능
초기화 불가능 가능

재할당이 빈번하게 필요한 경우에는 realloc을 사용할 수 있는 malloc이 더 나은 선택이 될 수도 있습니다.

malloc으로 할당한 메모리는 반드시 free로, new로 할당한 메모리는 반드시 delete로 해제해야 합니다. 혼용하면 정의되지 않은 동작(UB)이 발생합니다.

📌 핵심 정리

  • malloc/free는 함수, new/delete는 연산자
  • new/delete는 생성자/소멸자를 호출하지만, malloc/free는 하지 않음
  • malloc은 void*를 반환해 형변환이 필요하고, new는 해당 타입 포인터를 자동 반환
  • malloc은 realloc으로 크기 조정이 가능하지만, new는 불가능
  • 할당/해제 방식을 혼용하면 안 된다 (malloc-free, new-delete 짝을 맞출 것)
