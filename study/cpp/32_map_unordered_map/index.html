정렬할 것인가, 속도를 택할 것인가

C++ map과 unordered_map의 내부 구조와 성능 차이

Part 1: 두 컨테이너의 근본적 차이

map과 unordered_map은 뭐가 다른가요?

둘 다 key-value 쌍으로 되어 있지만 내부 자료구조가 완전히 다릅니다.

구분 map unordered_map
내부 구조 레드-블랙 트리 해시 테이블
정렬 key에 따라 자동 정렬 정렬 없음
평균 시간 복잡도 O(log N) O(1)
최악 시간 복잡도 O(log N) O(N)

Part 2: 해시 충돌과 체이닝

unordered_map에서 최악의 경우 O(N)이 나오는 이유는?

해시 함수를 통과한 뒤 값이 같은 경우(해시 충돌)가 발생할 수 있습니다.

이런 경우 체이닝(Chaining)을 통해 같은 버킷에 리스트처럼 연결하여 관리합니다. 이로 인해 최악의 시간 복잡도가 O(N)이 될 수 있습니다.

참고로 해시 함수는 역함수가 존재하지 않습니다.

체이닝 외에 충돌을 해결하는 방법이 있나요?

Open Addressing 방식이 있습니다. 충돌 발생 시 다른 빈 자리에 저장하는 방법입니다.

  • Linear Probing: hash(n), hash(n)+1, hash(n)+2 순서로 빈 슬롯 탐색. 테이블 끝에 도달하면 처음으로 돌아옴
  • Quadratic Probing: +1², +2², +3² 방식으로 간격을 넓혀가며 탐색
  • Double Hashing: 서로 다른 두 해시 함수를 사용

Part 3: 좋은 해시 함수란?

좋은 해시 함수의 기준은 무엇인가요?

SUHA (Simple Uniform Hashing Assumption): 각 해시 Key가 균등하게 분배되는 것을 의미합니다.

해싱 충돌을 최대한 줄여 모든 버킷에 고르게 데이터가 분산되는 것이 좋은 해시 함수의 조건입니다.

SUHA는 이론적인 가정으로, 실제로는 해시 함수의 성능 분석 시 기준으로 사용됩니다.

📌 핵심 정리

  • map: 레드-블랙 트리 기반, 정렬 보장, O(log N)
  • unordered_map: 해시 테이블 기반, 정렬 없음, 평균 O(1) / 최악 O(N)
  • 해시 충돌 해결 방법: 체이닝(Chaining), Open Addressing(Linear/Quadratic Probing, Double Hashing)
  • 좋은 해시 함수는 키를 균등하게 분배하여 충돌을 최소화 (SUHA)
