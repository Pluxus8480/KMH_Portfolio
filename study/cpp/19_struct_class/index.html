C++에서 struct와 class의 차이점

Part 1: 기본 접근 지정자의 차이

struct와 class는 어떤 차이가 있나요?

C++에서 struct와 class의 가장 큰 차이는 기본 접근 지정자입니다.

class는 기본 접근 지정자가 private이고, struct는 기본 접근 지정자가 public입니다.

즉, 접근 지정자를 명시하지 않으면 class의 멤버는 외부에서 접근할 수 없고, struct의 멤버는 외부에서 자유롭게 접근할 수 있습니다.

Part 2: 초기화 방식의 차이

초기화 방식에서도 차이가 있나요?

네, struct는 중괄호 초기화(aggregate initialization)가 가능하지만, class는 기본 접근 지정자가 private이기 때문에 동일한 방식의 초기화가 불가능합니다.

C++
struct SomeStruct
{
    int var;
    int varPrivate;
};

class SomeClass
{
    int var;
    int varPrivate;
};

int main(){
    SomeStruct someS = {1, 2};   // OK - public이므로 가능
    SomeClass  someC = {1, 2};   // ERROR - private이므로 불가능
}

struct는 모든 멤버가 기본적으로 public이기 때문에 중괄호 초기화가 바로 가능하지만, class는 멤버가 private이므로 외부에서 직접 값을 넣을 수 없습니다.

Part 3: 실무에서의 사용 관례

그러면 언제 struct를 쓰고 언제 class를 쓰나요?

문법적으로는 접근 지정자 외에 거의 동일하지만, 실무에서는 다음과 같은 관례가 있습니다.

구분 struct class
기본 접근 지정자 public private
주 용도 단순 데이터 묶음 (POD) 캡슐화가 필요한 객체
관례 멤버 함수가 거의 없는 경우 멤버 함수와 은닉이 필요한 경우

struct와 class는 기본 접근 지정자 외에는 기능적으로 동일합니다. 상속, 가상함수, 템플릿 등 모든 기능을 둘 다 사용할 수 있습니다.

📌 핵심 정리

  • struct의 기본 접근 지정자는 public, class는 private
  • struct는 중괄호 초기화가 바로 가능하지만, class는 멤버가 private이라 불가능
  • 기능적으로는 동일하며, 관례적으로 struct는 단순 데이터, class는 캡슐화된 객체에 사용
