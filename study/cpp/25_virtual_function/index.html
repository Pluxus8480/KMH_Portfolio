C++ 순수 가상 함수와 추상 클래스

Part 1: 순수 가상 함수란?

순수 가상 함수는 무엇인가요?

순수 가상 함수는 함수의 정의(구현)가 이루어지지 않고 선언만 한 가상 함수입니다.

C++
class Animal {
public:
    virtual void Speak() = 0;  // 순수 가상 함수 - = 0 으로 선언
};

이렇게 순수 가상 함수가 하나라도 있는 클래스를 추상 클래스(abstract class)라고 부릅니다.

🎯 비유로 이해하기

추상 클래스는 "설계도"와 같습니다. "동물은 소리를 낸다"는 규칙(순수 가상 함수)은 정해져 있지만, 구체적으로 어떤 소리를 내는지는 각 동물(자식 클래스)이 직접 정의해야 합니다.

추상 클래스는 객체로 만들 수 있나요?

추상 클래스는 객체로 만들 수 없습니다. 오직 상속으로만 사용됩니다.

그리고 추상 클래스를 상속받은 자식 클래스는 무조건 해당 순수 가상 함수를 override 해줘야 합니다. 구현하지 않으면 자식 클래스도 추상 클래스가 됩니다.

C++
class Animal {
public:
    virtual void Speak() = 0;
};

class Dog : public Animal {
public:
    void Speak() override {   // 반드시 구현해야 함!
        std::cout << "멍멍!" << std::endl;
    }
};

// Animal a;    // ERROR - 추상 클래스는 객체 생성 불가
Dog d;          // OK - 순수 가상 함수를 구현했으므로 가능

Part 2: 순수 가상 함수 vs 일반 가상 함수

일반 가상 함수와 어떤 차이가 있나요?
구분 순수 가상 함수 일반 가상 함수
선언 virtual void foo() = 0; virtual void foo() { ... }
기본 구현 없음 있음 (부모의 구현 사용 가능)
자식 재정의 필수 (안 하면 컴파일 에러) 선택 (안 해도 됨)
부모 객체 생성 불가능 (추상 클래스) 가능

순수 가상 함수는 반드시 재정의를 해줘야 하지만, 일반 가상 함수는 재정의를 하지 않아도 되기 때문에 프로그램에서 오류로 보지 않습니다.

인터페이스처럼 "이 함수는 반드시 구현해라"라는 강제 계약이 필요할 때 순수 가상 함수를 사용합니다.

📌 핵심 정리

  • 순수 가상 함수는 = 0으로 선언하며, 구현 없이 선언만 존재
  • 순수 가상 함수가 있는 클래스는 추상 클래스가 되어 객체 생성 불가
  • 자식 클래스는 순수 가상 함수를 반드시 override 해야 함
  • 일반 가상 함수는 재정의가 선택이지만, 순수 가상 함수는 필수
