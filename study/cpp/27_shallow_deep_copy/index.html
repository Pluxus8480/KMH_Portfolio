복사했는데 왜 원본이 바뀌지?

C++의 얕은 복사(Shallow Copy)와 깊은 복사(Deep Copy)

Part 1: 얕은 복사와 깊은 복사의 정의

얕은 복사와 깊은 복사는 무엇인가요?

얕은 복사(Shallow Copy): 멤버의 값들만 그대로 복사합니다.

깊은 복사(Deep Copy): 얕은 복사 + 객체가 가리키고 있는 포인터가 참조하는 대상까지 함께 복사합니다.

🎯 비유로 이해하기

얕은 복사는 "집 열쇠를 복사하는 것"과 같습니다. 열쇠(포인터)는 두 개가 되지만 열리는 집(데이터)은 같은 집입니다.

깊은 복사는 "집 자체를 똑같이 하나 더 짓는 것"입니다. 각자 자기 집을 가지게 됩니다.

Part 2: C++에서의 동작

C++에서 얕은 복사가 문제가 되는 경우는?

멤버 변수에 포인터가 있을 때 얕은 복사를 하면, 두 객체가 같은 메모리를 가리키게 됩니다.

C++ - 얕은 복사의 문제
class MyArray {
    int* data;
    int size;
public:
    MyArray(int s) : size(s), data(new int[s]) {}
    ~MyArray() { delete[] data; }
};

MyArray a(10);
MyArray b = a;  // 얕은 복사! data 포인터만 복사됨

// a와 b의 data가 같은 메모리를 가리킴
// a가 소멸되면 data가 해제됨
// b가 소멸될 때 이미 해제된 메모리를 다시 해제 -> 크래시!

얕은 복사로 인해 이중 해제(double free)가 발생하면 프로그램이 크래시됩니다. 포인터 멤버가 있는 클래스에서는 반드시 깊은 복사를 구현해야 합니다.

깊은 복사는 어떻게 구현하나요?

복사 생성자와 복사 대입 연산자를 직접 정의하여 포인터가 가리키는 데이터까지 새로 할당하고 복사합니다.

C++ - 깊은 복사 구현
class MyArray {
    int* data;
    int size;
public:
    MyArray(int s) : size(s), data(new int[s]) {}

    // 깊은 복사 - 복사 생성자
    MyArray(const MyArray& other) : size(other.size) {
        data = new int[size];  // 새 메모리 할당
        for (int i = 0; i < size; i++) {
            data[i] = other.data[i];  // 값 복사
        }
    }

    // 깊은 복사 - 복사 대입 연산자
    MyArray& operator=(const MyArray& other) {
        if (this != &other) {
            delete[] data;  // 기존 메모리 해제
            size = other.size;
            data = new int[size];  // 새 메모리 할당
            for (int i = 0; i < size; i++) {
                data[i] = other.data[i];
            }
        }
        return *this;
    }

    ~MyArray() { delete[] data; }
};

포인터 멤버가 있는 클래스에서는 복사 생성자, 복사 대입 연산자, 소멸자를 함께 정의해야 합니다. 이를 Rule of Three라고 합니다.

Part 3: C#과의 차이점

C#에서의 얕은/깊은 복사는 다른가요?

C#을 경험한 개발자라면 헷갈릴 수 있는 부분입니다. C++과 C#에서의 정의가 다릅니다.

구분 C++ C#
얕은 복사 멤버 값만 복사 (포인터 주소 복사) 객체에 대한 참조만 생성 (같은 힙 메모리 참조)
깊은 복사 포인터가 가리키는 대상까지 복사 힙 메모리에 새로 할당하여 완전 다른 객체 생성

C#의 "얕은 복사"는 C++의 관점에서 보면 "참조 복사"에 가깝고, C++의 "얕은 복사"는 C#에서는 존재하지 않는 개념입니다. 두 언어를 함께 다룰 때 혼동하지 않도록 주의하세요.

📌 핵심 정리

  • 얕은 복사: 멤버 값만 복사 - 포인터가 같은 메모리를 가리키게 됨
  • 깊은 복사: 포인터가 가리키는 데이터까지 새로 할당하여 복사
  • 포인터 멤버가 있으면 얕은 복사로 인해 이중 해제(double free) 위험
  • 복사 생성자, 복사 대입 연산자, 소멸자를 함께 정의하라 (Rule of Three)
  • C++과 C#의 얕은/깊은 복사 정의가 다르므로 혼동 주의
