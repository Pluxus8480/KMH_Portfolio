C++ static 멤버 변수와 멤버 함수

Part 1: static 멤버 변수

static 멤버 변수는 무엇인가요?

클래스의 static 멤버 변수는 모든 인스턴스가 공유하는 멤버로, 프로그램 수명 내 한 번만 초기화되며 계속해서 메모리에 올라가 있게 됩니다.

🎯 비유로 이해하기

일반 멤버 변수가 "각 학생이 가진 개인 사물함"이라면, static 멤버 변수는 "교실에 하나만 있는 공용 게시판"과 같습니다. 모든 학생(인스턴스)이 같은 게시판을 보고 수정할 수 있습니다.

static 멤버 변수는 어디서 초기화하나요?

static 멤버 변수는 전역 범위에서 초기화해야 합니다. 생성자 내에서는 초기화할 수 없습니다.

C++
class Some
{
private:
    static int var;

    // Some()
    //     : var(1)
    // {}    -> 생성자 내에서도 초기화 불가능
};

int Some::var = 0;  // 전역 범위에서 초기화 가능

헤더와 cpp 파일을 분리해서 개발하는 경우, 반드시 cpp 파일에서 초기화를 해줘야 합니다.

주의: private 멤버라 하더라도 전역 범위에서의 초기화는 가능합니다. 이는 "초기화"는 접근이 아니라 정의(definition)이기 때문입니다.

static const 변수는 클래스 내에서 초기화가 가능한가요?

static const 변수의 경우에는 클래스 내에서 초기화하는 것이 가능합니다.

const의 특성상 값을 변경하는 것이 불가능하며 컴파일 타임에 초기화가 되므로 허용됩니다.

C++
class Some
{
private:
    static const int MAX_COUNT = 100;  // OK! 클래스 내 초기화 가능
};

Part 2: static 멤버 함수

static 멤버 함수는 무엇이 다른가요?

클래스 내 static 함수는 모든 인스턴스가 공유하는 함수로, 객체 없이도 호출할 수 있습니다.

static 멤버 함수 내에서는 일반 멤버 변수, 즉 this가 붙는 멤버에 대해서는 연산을 할 수 없습니다.

다만 static 멤버 변수와 static 멤버 함수에 대한 연산은 사용할 수 있습니다.

C++
class Counter
{
private:
    static int count;
    int id;

public:
    static int GetCount() {
        // count 접근 가능 (static 멤버)
        // id 접근 불가능 (일반 멤버 - this 필요)
        return count;
    }
};

// 객체 없이 호출 가능
int c = Counter::GetCount();

static 멤버 함수에는 this 포인터가 없으므로, 일반 멤버 변수나 일반 멤버 함수에 접근하면 컴파일 에러가 발생합니다.

📌 핵심 정리

  • static 멤버 변수는 모든 인스턴스가 공유하며 프로그램 수명 동안 유지됨
  • static 멤버 변수는 전역 범위(cpp 파일)에서 초기화해야 함 (static const 제외)
  • static 멤버 함수는 객체 없이 호출 가능하지만, this가 없어 일반 멤버에 접근 불가
  • static const 변수는 예외적으로 클래스 내부에서 초기화 가능
