"같다"에도 두 가지 의미가 있다

C++ 상등성(Equality)과 동등성(Equivalence)의 차이

Part 1: 상등성과 동등성의 정의

상등성과 동등성은 어떻게 다른가요?

상등성(Equality): operator ==에 기반을 둡니다. "두 값이 정확히 같은가?"

동등성(Equivalence): operator <에 기반을 둡니다. "두 값 사이에 순서 차이가 없는가?"

C++ - 동등성 판별
// 동등성: 어느 쪽도 다른 쪽보다 "작지 않다"면 동등하다
if (!(var1 < var2) && !(var2 < var1))
{
    cout << "이건 동등하다";
}
표준 컨테이너는 어떤 방식을 사용하나요?

표준의 연관 컨테이너들(map, set 등)은 동등성(Equivalence)에 기반을 두고 있습니다.

🎯 비유로 이해하기

상등성: "이 두 사람은 같은 사람인가?" - 주민등록번호까지 모두 일치해야 "같다"

동등성: "이 두 사람은 같은 등수인가?" - 점수 기준으로 순위가 같으면 "같다" (이름이 달라도)

Part 2: 왜 연관 컨테이너는 동등성을 쓰는가?

map이나 set이 == 대신 <를 사용하는 이유가 있나요?

연관 컨테이너는 내부적으로 정렬된 상태를 유지합니다 (레드-블랙 트리). 정렬을 위해 이미 operator <를 사용하고 있으므로, 별도로 operator ==를 요구하지 않고 <만으로 동등성을 판별합니다.

이 덕분에 사용자 정의 타입을 키로 사용할 때 operator <만 정의하면 됩니다.

C++
struct MyKey {
    int id;
    // operator < 만 정의하면 map/set에서 사용 가능
    bool operator<(const MyKey& other) const {
        return id < other.id;
    }
};

std::map<MyKey, std::string> myMap; // OK

📌 핵심 정리

  • 상등성(Equality): operator == 기반 - "정확히 같은가?"
  • 동등성(Equivalence): operator < 기반 - "순서상 차이가 없는가?"
  • 표준 연관 컨테이너(map, set)는 동등성에 기반
  • 동등성 판별: !(a < b) && !(b < a)이면 a와 b는 동등
