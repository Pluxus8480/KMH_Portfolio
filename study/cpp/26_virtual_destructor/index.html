소멸할 때 빠뜨리는 것은 없는가?

상속 관계에서 가상 소멸자가 필요한 이유

Part 1: 왜 가상 소멸자가 필요한가?

상속 관계에서 소멸자를 virtual로 선언해야 하는 이유가 무엇인가요?

상속 관계에서 소멸자가 비가상(non-virtual)으로 되어 있으면 심각한 문제가 발생할 수 있습니다.

🎯 비유로 이해하기

건물을 철거할 때, 2층(자식)만 철거하고 1층(부모)의 내부 시설은 그대로 방치하는 것과 같습니다. 1층에 있는 장비(동적 할당된 자원)가 해제되지 않아 자원 낭비가 발생합니다.

부모 클래스에서 동적으로 할당한 자원을 소멸자에서 해제하기로 되어 있다고 합시다. 소멸자가 비가상이면, 자식 클래스의 인스턴스가 부모 포인터를 통해 소멸할 때 자식 클래스의 소멸자만 호출되므로 부모 클래스의 동적 할당 자원이 해제되지 않습니다.

이는 메모리 누수의 직접적인 원인이 됩니다.

C++ - 잘못된 예 (비가상 소멸자)
class Base {
    int* data;
public:
    Base() : data(new int[100]) {}
    ~Base() { delete[] data; }  // 비가상 소멸자!
};

class Derived : public Base {
    int* extra;
public:
    Derived() : extra(new int[200]) {}
    ~Derived() { delete[] extra; }
};

Base* ptr = new Derived();
delete ptr;  // Base의 소멸자만 호출! Derived::extra 메모리 누수!

Part 2: 올바른 가상 소멸자 선언

어떻게 해결하나요?

상속 관계에서 소멸자는 virtual 선언을 해주는 것이 좋습니다.

C++ - 올바른 예 (가상 소멸자)
class Base {
    int* data;
public:
    Base() : data(new int[100]) {}
    virtual ~Base() { delete[] data; }  // 가상 소멸자!
};

class Derived : public Base {
    int* extra;
public:
    Derived() : extra(new int[200]) {}
    ~Derived() override { delete[] extra; }  // 자동으로 virtual
};

Base* ptr = new Derived();
delete ptr;  // Derived::~Derived() 호출 후 Base::~Base() 호출 - 안전!

부모 클래스에 virtual 키워드를 붙이면 자식 클래스의 소멸자는 자동으로 virtual이 됩니다.

소멸자가 호출되는 순서는 어떻게 되나요?

소멸자는 자식 -> 부모 순서로 호출됩니다.

이는 생성자가 부모 -> 자식 순서로 호출되는 것과 정반대입니다.

생성 순서
Base() -> Derived()
→
소멸 순서
~Derived() -> ~Base()

📌 핵심 정리

  • 상속 관계에서 소멸자가 비가상이면 메모리 누수 위험이 있다
  • 부모 클래스의 소멸자에 virtual을 붙이면 자식도 자동으로 virtual
  • 소멸자 호출 순서는 자식 -> 부모 (생성의 역순)
  • 상속 관계에서는 항상 가상 소멸자를 선언하는 것이 안전하다
