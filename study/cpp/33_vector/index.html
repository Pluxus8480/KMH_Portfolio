C++ vector의 메모리 관리, list 비교, 그리고 숨겨진 함정들

Part 1: vector vs list

vector와 list의 차이점은 무엇인가요?
구분 vector list
메모리 배치 연속적 비연속적 (흩어져 있음)

Part 2: size와 capacity

vector의 size와 capacity는 어떻게 다른가요?

size: 실제 원소들이 들어있는 공간의 크기

capacity: 벡터가 현재 할당받은 전체 공간의 크기

  • reserve(): capacity를 변경하는 함수
  • resize(): size를 변경하는 함수
vector에 원소를 삽입하면 내부에서 무슨 일이 일어나나요?

vector는 연속적인 메모리 공간을 미리 할당합니다. 원소 삽입 시 메모리 공간이 부족하면:

1. 새 메모리 할당
더 큰 공간 확보
→
2. 기존 원소 복사
새 공간으로 이동
→
3. 새 원소 삽입
끝에 추가
→
4. 기존 메모리 해제
이전 공간 반환

reserve가 많이 되어 있지만 실제 사용 공간이 적다면, 새 벡터에 복사 후 swap하는 전략(shrink-to-fit)이 좋습니다.

Part 3: erase-remove 패턴

vector에서 값을 어떻게 삭제하나요?

vector에서는 erase-remove 구문을 함께 사용합니다.

C++
vector<int> v = {1, 2, 3, 2, 4, 2};

// 특정 값 삭제
v.erase(remove(v.begin(), v.end(), 2), v.end());

// 조건에 따른 삭제
v.erase(remove_if(v.begin(), v.end(), [](int x){ return x > 3; }), v.end());

remove는 실제로 삭제하지 않습니다! 조건에 부합하는 원소들을 뒤로 밀고, "밀린 원소들의 시작 위치"를 반환합니다. 실제 삭제는 erase가 담당합니다.

Part 4: vector<bool>의 함정

vector<bool>은 일반 vector와 다른가요?

vector<bool>은 실제 bool을 담지 않습니다!

프록시 패턴을 통해 마치 bool을 담고 있는 것처럼 동작합니다. 내부적으로는 원소 한 자리에 8비트의 비트필드를 사용합니다.

실제 bool이 담긴 자료구조가 필요하다면 std::bitset이나 deque<bool>을 사용하세요.

Part 5: vector vs deque

vector와 deque는 어떻게 다른가요?

vector는 모든 원소가 연속적인 공간에 있지만, deque는 여러 블록이 흩어져 있고 이를 관리하는 컨트롤 블록을 가집니다.

deque는 여러 벡터(청크)들이 흩어진 형태로, 결과적으로 모든 원소가 연속적인 공간에 있지 않습니다.

deque는 앞뒤 양쪽에서 효율적인 삽입/삭제가 필요할 때 유리합니다.

📌 핵심 정리

  • vector: 연속 메모리, 랜덤 접근 O(1), 중간 삽입/삭제 O(N)
  • size는 실제 원소 수, capacity는 할당된 공간 크기
  • 삭제 시 erase-remove 패턴 사용 (remove만으로는 실제 삭제 안 됨)
  • vector<bool>은 프록시 패턴으로 구현되어 일반 vector와 다르게 동작
  • deque는 비연속적 블록 구조로 양쪽 삽입/삭제에 유리
