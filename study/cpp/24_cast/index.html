타입을 바꾸는 네 가지 무기

C++ 캐스팅 연산자 static_cast, const_cast, reinterpret_cast, dynamic_cast

Part 1: 왜 C++ 캐스팅이 필요한가?

C 스타일 캐스팅 대신 왜 C++ 캐스팅을 사용하나요?

캐스팅은 명시적 캐스팅과 묵시적 캐스팅으로 나뉩니다.

묵시적 캐스팅은 다른 표기 없이 호환되는 타입으로 변환하지만, 데이터 손실이 발생할 수 있고 유지보수 측면에서도 알아내기 어렵습니다.

C 스타일 캐스팅
float f = 1.1f;
int a = (int)f;  // C 스타일 - 코드 분간이 어려움

C 스타일 캐스팅은 어떤 종류의 변환이 일어나는지 명확하지 않고, 위험한 변환도 아무런 경고 없이 허용합니다. C++의 캐스팅 연산자들은 용도별로 구분되어 의도를 명확하게 합니다.

Part 2: static_cast

static_cast는 언제 사용하나요?

기존 C 스타일의 형변환을 대체합니다. 형변환 가능 여부를 컴파일 타임에 검사합니다.

C++
float f = 1.1f;
int a = static_cast<int>(f);  // 안전한 형변환

가장 일반적으로 사용되는 캐스팅으로, 기본 타입 간 변환이나 상속 관계의 업캐스팅에 사용됩니다.

Part 3: const_cast

const_cast는 무엇을 하나요?

const를 제거하거나 부여하는 데 사용됩니다. 주로 const 제거 용도로 사용됩니다.

C++
void NonConstFunc(Some s);

// ...
const Some s;
NonConstFunc(s);                      // ERROR!
NonConstFunc(const_cast<Some>(s));    // OK - const 제거

const_cast로 const를 제거한 뒤 실제로 값을 수정하면 정의되지 않은 동작(UB)이 발생할 수 있습니다. 레거시 API 호환 등 불가피한 경우에만 사용하세요.

Part 4: reinterpret_cast

reinterpret_cast는 어떤 캐스팅인가요?

모든 포인터 유형을 다른 포인터 방식으로 변경할 수 있으며, 포인터 타입을 정수로 바꾸는 것도 가능합니다.

메모리 단위로 분해, 재조립을 하는 가장 강력한 캐스팅입니다.

C++
int* p = new int(42);

// 포인터를 정수로 변환
uintptr_t addr = reinterpret_cast<uintptr_t>(p);

// 완전히 다른 포인터 타입으로 변환
char* cp = reinterpret_cast<char*>(p);

매우 강력하지만 그만큼 위험합니다. 메모리 레이아웃에 대한 이해가 필요하며, 일반적인 코드에서는 거의 사용하지 않습니다.

Part 5: dynamic_cast

dynamic_cast는 언제 사용하나요?

클래스의 포인터나 참조에만 사용할 수 있으며, 런타임에 캐스팅을 진행합니다.

RTTI(Runtime Type Information)를 활용하여 안전한 다운캐스팅을 지원합니다.

C++
class Base { virtual void foo() {} };
class Derived : public Base {};

Base* base = new Derived();
Derived* derived = dynamic_cast<Derived*>(base);

if (derived != nullptr) {
    // 캐스팅 성공 - 안전하게 사용 가능
}

dynamic_cast는 클래스에 virtual 함수가 있어야 사용 가능합니다. 다형성이 없는 객체 간 캐스팅에서는 컴파일 에러가 발생합니다.

캐스팅이 실패하면 포인터의 경우 nullptr을, 참조의 경우 std::bad_cast 예외를 반환합니다.

Part 6: 비교 정리

네 가지 캐스팅을 한눈에 비교해 주세요.
캐스팅 검사 시점 주 용도
static_cast 컴파일 타임 일반적인 형변환 (C 스타일 대체)
const_cast 컴파일 타임 const 제거/부여
reinterpret_cast 없음 (강제) 포인터/정수 간 저수준 변환
dynamic_cast 런타임 다형성 객체의 안전한 다운캐스팅

📌 핵심 정리

  • static_cast: 컴파일 타임 검사, 가장 일반적인 형변환에 사용
  • const_cast: const 제거/부여 전용, 불가피한 경우에만 사용
  • reinterpret_cast: 가장 강력하고 위험한 저수준 캐스팅
  • dynamic_cast: 런타임 RTTI 기반, 다형성 객체의 안전한 다운캐스팅 (virtual 필수)
