변하지 않는 약속, const

C++ const 키워드의 다양한 활용법

Part 1: const의 기본 개념

const는 무엇인가요?

const는 해당 값이 상수임을 지정하고, 프로그래머가 초기화 이후 이를 수정할 수 없게 하는 키워드입니다.

컴파일러가 const로 선언된 변수에 대한 수정 시도를 감지하면 컴파일 에러를 발생시켜 실수를 방지해 줍니다.

Part 2: const와 포인터

포인터에서 const의 위치에 따라 의미가 달라진다고 하던데요?

맞습니다. 포인터 *의 왼쪽에 오는지, 오른쪽에 오는지에 따라 const의 의미가 완전히 달라집니다.

1. const가 * 왼쪽에 오는 경우 (가리키는 값을 상수화)

C++
const int* var = &someNumber;

someNumber = 2;  // OK - 원본 변수 자체는 변경 가능
*var = 2;        // ERROR! - 포인터를 통한 값 변경 불가

*의 왼쪽에 오는 const는 가리키는 값에 대해 상수화한다는 의미입니다.

2. const가 * 오른쪽에 오는 경우 (포인터 자체를 상수화)

C++
int* const var = &someNumber;

*var = 2;               // OK - 가리키는 값은 변경 가능
var = &anotherNumber;   // ERROR! - 포인터가 가리키는 주소 변경 불가

*의 오른쪽에 오는 const는 포인터 변수가 상수화되어 가리키는 주소를 고정한다는 의미입니다.

🎯 비유로 이해하기

const int*는 "가게(주소)는 바꿀 수 있지만 물건(값)은 못 바꾸는 것"이고,

int* const는 "가게(주소)는 고정이지만 가게 안 물건(값)은 바꿀 수 있는 것"입니다.

Part 3: 클래스에서의 const

클래스에서 const는 어떻게 사용되나요?

const 멤버 변수

한번 결정되면 변경되지 않는 값을 의미합니다. 클래스에서 선언하면 반드시 초기화를 해줘야 합니다.

C++11부터는 클래스 내부에서 직접 초기화가 가능하지만, 대부분의 경우 생성자의 초기화 리스트에서 초기화합니다.

const 멤버 함수

반환값이 const인 경우와 함수 자체가 const인 경우로 나뉩니다.

C++
class MyClass {
    int value;
public:
    // 반환값이 const: 반환값을 수정할 수 없음
    const int& GetValue() { return value; }

    // 함수가 const: 함수 내에서 멤버 변수를 수정할 수 없음
    int GetValue() const { return value; }
};

const 멤버 함수 내에서는 멤버 변수를 읽기 전용으로만 사용할 수 있고, 다른 멤버 함수를 호출할 때도 const인 함수만 호출 가능합니다.

const 멤버 함수에서 멤버 변수를 수정하거나 non-const 멤버 함수를 호출하면 컴파일 에러가 발생합니다.

📌 핵심 정리

  • const는 값을 상수화하여 수정을 방지하는 키워드
  • const int*: 가리키는 값을 상수화 / int* const: 포인터 자체를 상수화
  • const 멤버 변수는 반드시 초기화해야 하며, 주로 초기화 리스트에서 설정
  • const 멤버 함수는 멤버 변수를 읽기 전용으로 취급하고, const 함수만 호출 가능
