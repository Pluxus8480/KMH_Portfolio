주소를 가리키는 두 가지 방법

C++에서 포인터와 배열의 차이점

Part 1: 포인터와 배열의 공통점

배열도 포인터인가요?

배열도 연속적인 원소들의 첫 번째 주소를 가리키고 있다는 점에서 포인터라고 할 수 있습니다.

배열 이름은 배열의 첫 번째 요소를 가리키는 주소값으로 사용될 수 있습니다.

C++
int arr[5] = {1, 2, 3, 4, 5};
int* ptr = arr;  // 배열 이름이 첫 번째 요소의 주소로 변환

// 둘 다 같은 값을 출력
std::cout << arr[2] << std::endl;   // 3
std::cout << *(ptr + 2) << std::endl; // 3

Part 2: 핵심 차이점

그러면 포인터와 배열의 차이는 무엇인가요?

배열은 상수 포인터처럼 동작합니다. 배열 이름 자체를 다른 주소로 변경할 수 없습니다.

반면 포인터 변수는 증감 연산자, 대입 연산자 등을 자유롭게 사용할 수 있습니다.

C++
int arr[5] = {1, 2, 3, 4, 5};
int* ptr = arr;

// 포인터 변수는 자유롭게 변경 가능
ptr++;           // OK - 다음 요소를 가리킴
ptr = nullptr;   // OK - 다른 주소 대입 가능

// 배열 이름은 변경 불가능
arr++;           // ERROR - 배열 이름은 상수
arr = ptr;       // ERROR - 대입 불가능
구분 포인터 배열
주소 변경 가능 (대입, 증감) 불가능 (상수)
sizeof 포인터 크기 (4/8바이트) 전체 배열 크기
메모리 별도의 포인터 변수 공간 필요 연속된 메모리 블록 자체

sizeof 연산에서 큰 차이가 나타납니다. 배열에 sizeof를 적용하면 전체 배열 크기가 나오지만, 포인터에 적용하면 포인터 변수 자체의 크기(보통 4 또는 8바이트)만 나옵니다.

Part 3: 함수 인자로 전달할 때

함수에 배열을 넘기면 어떻게 되나요?

배열을 함수 인자로 전달하면 포인터로 붕괴(decay)됩니다. 이때 sizeof 정보를 잃어버리므로 배열 크기를 별도로 전달해야 합니다.

C++
void PrintSize(int arr[]) {
    // 여기서 arr은 이미 int* 로 붕괴됨
    std::cout << sizeof(arr) << std::endl; // 포인터 크기 출력 (8)
}

int main() {
    int arr[10];
    std::cout << sizeof(arr) << std::endl; // 40 (10 * 4바이트)
    PrintSize(arr);                         // 8 (포인터 크기)
}

📌 핵심 정리

  • 배열은 연속된 메모리의 첫 주소를 가리키는 상수처럼 동작한다
  • 포인터 변수는 증감/대입 연산이 가능하지만, 배열 이름은 불가능하다
  • sizeof 결과가 다르다: 배열은 전체 크기, 포인터는 포인터 변수 크기
  • 배열을 함수에 전달하면 포인터로 붕괴(decay)되어 크기 정보를 잃는다
