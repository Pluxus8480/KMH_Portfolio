std::sort 안에는 세 개의 알고리즘이 숨어있다

C++ <algorithm> sort의 내부 구현: 인트로 소트(Introsort)

Part 1: std::sort는 무슨 알고리즘인가?

<algorithm> 헤더의 sort 함수는 어떤 알고리즘을 사용하나요?

std::sort는 인트로 소트(Introsort)를 사용합니다.

이전에는 퀵 정렬을 사용했지만, 최악의 경우 O(N²) 시간복잡도라는 고질적인 문제 때문에 인트로 소트를 채용하게 되었습니다.

인트로 소트는 3개의 알고리즘이 결합된 하이브리드 정렬입니다:

  • 퀵 정렬 (Quick Sort)
  • 힙 정렬 (Heap Sort)
  • 삽입 정렬 (Insertion Sort)

내부 구현을 사용자가 알 수 없게 잘 캡슐화한 훌륭한 사례입니다.

Part 2: 인트로 소트의 동작 방식

세 가지 알고리즘은 언제 각각 사용되나요?
파티션 크기 ≤ 16
삽입 정렬
→
16 < 크기 < 2logN
퀵 정렬
→
깊이 ≥ 2logN
힙 정렬로 전환
  • 파티션의 원소가 16개 이하로 작아지면 삽입 정렬
  • 파티션 크기가 적당하면(16 ~ 2logN) 퀵 정렬
  • 재귀 깊이가 2logN을 넘을 가능성이 있으면 힙 정렬로 전환
왜 작은 배열에서 삽입 정렬을 사용하나요?

삽입 정렬은 작은 배열에 대해, 그리고 거의 정렬된 배열에 대해 매우 빠른 속도를 보이기 때문입니다.

퀵 정렬의 재귀 오버헤드보다 삽입 정렬의 단순 반복이 더 효율적인 크기가 바로 16개 이하입니다.

왜 머지 소트는 사용하지 않나요?

머지 소트는 N만큼의 추가 공간이 필요하기 때문에 사용하지 않습니다. 인트로 소트는 in-place 정렬을 지향합니다.

Part 3: List의 정렬은?

list도 std::sort로 정렬할 수 있나요?

아닙니다! list는 std::sort로 정렬할 수 없습니다.

리스트는 랜덤 접근이 불가능하기 때문에 <algorithm> 헤더의 sort가 동작하지 않습니다.

list는 자체 멤버 함수인 list.sort()를 사용해야 하며, 내부적으로 머지 소트를 사용합니다. 시간 복잡도는 O(N log N)입니다.

C++
std::list<int> myList = {5, 3, 1, 4, 2};

// std::sort(myList.begin(), myList.end()); // ERROR!
myList.sort(); // OK - 멤버 함수 sort 사용

📌 핵심 정리

  • std::sort는 인트로 소트(퀵 + 힙 + 삽입 정렬의 하이브리드)
  • 작은 배열(≤16)은 삽입 정렬, 중간은 퀵 정렬, 깊이 초과 시 힙 정렬
  • 머지 소트는 추가 메모리 필요로 채택되지 않음
  • list는 랜덤 접근 불가하여 멤버 함수 sort() 사용 (내부는 머지 소트)
