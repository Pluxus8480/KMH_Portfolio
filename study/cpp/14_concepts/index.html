C++20 Concepts로 타입 제약을 명확하게

Part 1: Concepts가 필요한 이유

기존 템플릿의 문제점은?

템플릿 에러 메시지가 수백 줄에 달하고 이해하기 어렵습니다!

기존 템플릿 문제
// 정렬 함수
template<typename Container>
void sortContainer(Container& c) {
    std::sort(c.begin(), c.end());
}

// 잘못된 사용
int x = 42;
sortContainer(x);  // 에러!

// 에러 메시지 (축약)
// error: no member named 'begin' in 'int'
// error: no member named 'end' in 'int'
// ... (수십 줄의 템플릿 인스턴스화 정보)
// ... iterator_category ...
// ... value_type ...
// 실제 문제가 뭔지 파악하기 어려움!
Concepts 해결책
#include <concepts>
#include <ranges>

// Concept으로 제약
template<std::ranges::range Container>
void sortContainer(Container& c) {
    std::sort(c.begin(), c.end());
}

int x = 42;
sortContainer(x);

// 에러 메시지:
// error: constraints not satisfied
// note: 'int' does not satisfy 'range'
// 명확하고 간단!
🎯 비유로 이해하기

Concepts는 구인 공고의 "지원 자격"과 같습니다. "C++ 경력 3년 이상"처럼 조건을 명시하면, 자격 미달 시 즉시 알 수 있습니다.

Part 2: Concept 정의

Concept 문법
// 기본 문법: concept 이름 = 제약조건;
template<typename T>
concept Integral = std::is_integral_v<T>;

template<typename T>
concept SignedIntegral = Integral<T> && std::is_signed_v<T>;

// requires 표현식
template<typename T>
concept Addable = requires(T a, T b) {
    a + b;  // 이 표현식이 유효해야 함
};

// 반환 타입 제약
template<typename T>
concept Hashable = requires(T a) {
    { std::hash<T>{}(a) } -> std::convertible_to<std::size_t>;
};

// 복합 요구사항
template<typename T>
concept Container = requires(T c) {
    // 타입 요구사항
    typename T::value_type;
    typename T::iterator;

    // 표현식 요구사항
    { c.begin() } -> std::input_or_output_iterator;
    { c.end() } -> std::sentinel_for<decltype(c.begin())>;
    { c.size() } -> std::convertible_to<std::size_t>;
};

// 중첩 요구사항
template<typename T>
concept Sortable = requires(T c) {
    requires std::ranges::random_access_range<T>;
    requires std::sortable<std::ranges::iterator_t<T>>;
};

Part 3: Concept 적용 방법

4가지 적용 방법
// 1. requires 절
template<typename T>
    requires std::integral<T>
T add(T a, T b) {
    return a + b;
}

// 2. 후위 requires 절
template<typename T>
T multiply(T a, T b) requires std::integral<T> {
    return a * b;
}

// 3. 제약된 템플릿 파라미터
template<std::integral T>
T subtract(T a, T b) {
    return a - b;
}

// 4. 축약 함수 템플릿 (auto + concept)
auto divide(std::integral auto a, std::integral auto b) {
    return a / b;
}

// 모두 동일하게 동작
add(1, 2);        // OK
add(1.5, 2.5);    // 에러: double은 integral이 아님

// 클래스 템플릿에도 적용
template<std::integral T>
class Counter {
    T value = 0;
public:
    void increment() { ++value; }
    T get() const { return value; }
};

Part 4: 표준 Concepts

주요 표준 Concepts
#include <concepts>

// 기본 타입 Concepts
std::same_as<T, U>          // T와 U가 같은 타입
std::derived_from<D, B>     // D가 B에서 파생
std::convertible_to<F, T>   // F에서 T로 변환 가능
std::integral<T>            // 정수형
std::floating_point<T>      // 부동소수점
std::signed_integral<T>     // 부호 있는 정수
std::unsigned_integral<T>   // 부호 없는 정수

// 비교 Concepts
std::equality_comparable<T> // == 비교 가능
std::totally_ordered<T>     // <, >, <=, >= 비교 가능
std::three_way_comparable<T> // <=> 비교 가능

// 객체 Concepts
std::movable<T>             // 이동 가능
std::copyable<T>            // 복사 가능
std::semiregular<T>         // 기본 생성 + 복사 가능
std::regular<T>             // semiregular + equality_comparable

// 호출 가능 Concepts
std::invocable<F, Args...>  // F(Args...) 호출 가능
std::predicate<F, Args...>  // bool 반환하는 호출 가능

// Iterator Concepts (<iterator>)
std::input_iterator<I>
std::output_iterator<I, T>
std::forward_iterator<I>
std::bidirectional_iterator<I>
std::random_access_iterator<I>
std::contiguous_iterator<I>

// Range Concepts (<ranges>)
std::ranges::range<R>
std::ranges::input_range<R>
std::ranges::forward_range<R>
std::ranges::bidirectional_range<R>
std::ranges::random_access_range<R>
std::ranges::contiguous_range<R>

Part 5: 실전 활용

커스텀 Concepts 예제
// 1. 게임 엔티티 Concept
template<typename T>
concept GameEntity = requires(T e) {
    { e.update(0.0f) };
    { e.render() };
    { e.getPosition() } -> std::same_as<Vector3>;
    { e.isAlive() } -> std::convertible_to<bool>;
};

template<GameEntity E>
void updateAll(std::vector<E*>& entities, float dt) {
    for (auto* e : entities) {
        if (e->isAlive()) {
            e->update(dt);
        }
    }
}

// 2. 직렬화 가능 Concept
template<typename T>
concept Serializable = requires(T obj, std::ostream& os, std::istream& is) {
    { obj.serialize(os) } -> std::same_as<void>;
    { T::deserialize(is) } -> std::same_as<T>;
};

template<Serializable T>
void saveToFile(const T& obj, const std::string& path) {
    std::ofstream file(path);
    obj.serialize(file);
}

// 3. 수학 연산 Concept
template<typename T>
concept Arithmetic = requires(T a, T b) {
    { a + b } -> std::convertible_to<T>;
    { a - b } -> std::convertible_to<T>;
    { a * b } -> std::convertible_to<T>;
    { a / b } -> std::convertible_to<T>;
};

template<Arithmetic T>
T lerp(T a, T b, float t) {
    return a + (b - a) * t;
}

// 4. 콜백 Concept
template<typename F, typename... Args>
concept Callback = std::invocable<F, Args...>;

template<typename T>
concept EventHandler = requires(T handler) {
    { handler.onEvent() } -> std::same_as<void>;
};

// 오버로딩으로 분기
void process(std::integral auto x) {
    std::cout << "정수: " << x << "\n";
}

void process(std::floating_point auto x) {
    std::cout << "실수: " << x << "\n";
}

void process(auto x) {
    std::cout << "기타: " << x << "\n";
}

Part 6: Concept vs 이전 기법

기법 가독성 에러 메시지 유연성
무제약 템플릿 높음 매우 나쁨 높음
SFINAE 낮음 보통 높음
static_assert 보통 좋음 낮음
Concepts 높음 매우 좋음 높음
비교 예제
// SFINAE 방식 (C++17 이전)
template<typename T,
         std::enable_if_t<std::is_integral_v<T>, int> = 0>
T oldAdd(T a, T b) { return a + b; }

// Concepts 방식 (C++20)
template<std::integral T>
T newAdd(T a, T b) { return a + b; }

// 또는
std::integral auto autoAdd(std::integral auto a, std::integral auto b) {
    return a + b;
}

// Concept 조합
template<typename T>
concept NumericContainer =
    std::ranges::range<T> &&
    std::is_arithmetic_v<std::ranges::range_value_t<T>>;

template<NumericContainer C>
auto sum(const C& container) {
    using T = std::ranges::range_value_t<C>;
    T result{};
    for (const auto& elem : container) {
        result += elem;
    }
    return result;
}

Concepts는 템플릿 프로그래밍을 더 읽기 쉽고 디버깅하기 쉽게 만듭니다!

📌 핵심 정리

  • Concept: 템플릿 파라미터의 타입 제약 조건
  • requires: 타입이 만족해야 할 표현식/타입 정의
  • 적용 방법: requires 절, 제약된 파라미터, auto + concept
  • 장점: 명확한 에러 메시지, 높은 가독성
  • 표준 Concepts: integral, range, invocable 등
