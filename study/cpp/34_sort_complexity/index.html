주요 정렬 알고리즘의 시간/공간 복잡도 비교 정리

Part 1: 정렬 알고리즘 복잡도 비교

주요 정렬 알고리즘의 복잡도를 정리해주세요.
Algorithm 공간 복잡도 평균 시간 복잡도 최악 시간 복잡도
Bubble Sort O(1) O(n²) O(n²)
Selection Sort O(1) O(n²) O(n²)
Insertion Sort O(1) O(n²) O(n²)
Merge Sort O(n) O(n log n) O(n log n)
Heap Sort O(1) O(n log n) O(n log n)
Quick Sort O(1) O(n log n) O(n²)
Count Sort O(n) O(n) O(n)
Radix Sort O(n) O(n) O(n)

Part 2: 분류별 특징

O(n²) 알고리즘과 O(n log n) 알고리즘의 차이는?

O(n²) 알고리즘 (Bubble, Selection, Insertion): 구현이 단순하고 추가 메모리가 필요 없지만, 데이터가 커지면 급격히 느려집니다.

O(n log n) 알고리즘 (Merge, Heap, Quick): 대규모 데이터에 적합하며 실무에서 주로 사용됩니다.

Quick Sort는 평균적으로 가장 빠르지만, 최악의 경우 O(n²)이 될 수 있다는 점에 주의해야 합니다.

Count Sort와 Radix Sort는 왜 O(n)이 가능한가요?

비교 기반이 아닌 비비교 정렬이기 때문입니다. 데이터의 특성(정수, 범위 제한)을 활용하여 비교 없이 정렬합니다.

단, 추가 메모리 O(n)이 필요하고 데이터 타입에 제한이 있습니다.

📌 핵심 정리

  • O(n²) 정렬: Bubble, Selection, Insertion - 소규모 데이터에 적합
  • O(n log n) 정렬: Merge, Heap, Quick - 대규모 데이터의 표준
  • Quick Sort는 평균 최고 성능이지만 최악 O(n²) 위험 존재
  • Merge Sort는 안정 정렬이지만 O(n) 추가 공간 필요
  • Count/Radix Sort는 O(n)이지만 데이터 타입 제한 있음
