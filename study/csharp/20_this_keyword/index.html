this가 하나인 줄 알았는데, 세 가지 얼굴이 있다고?

C#의 this 키워드 - 인스턴스 참조, 생성자 체이닝, 확장 메서드

Part 1: 클래스의 현재 인스턴스 this

가장 기본적인 this의 용도는 무엇인가요?

다른 언어에서도 지원하는 가장 기본적인 용법으로, 클래스의 현재 인스턴스를 가리킵니다. 매개변수 이름과 클래스 필드 이름이 같을 때 구분하는 데 사용합니다.

클래스 내에서 클래스 필드를 사용할 때는 모두 this가 생략된 것이라고 볼 수 있습니다.

Part 2: 생성자 this()

생성자에서 this()를 사용하는 이유는?

생성자 간에 중복되는 초기화 코드를 줄이기 위해 사용합니다. this()는 자기 자신의 다른 생성자를 가리킵니다.

C# - this() 없이 (중복 코드)
class MyClass
{
    int a;
    int b;

    public MyClass()
    {
        a = 10; // 중복!
    }

    public MyClass(int b)
    {
        a = 10; // 중복!
        this.b = b;
    }
}
C# - this() 사용 (중복 제거)
class MyClass
{
    int a;
    int b;

    public MyClass()
    {
        a = 10;
    }

    public MyClass(int b) : this() // 기본 생성자 호출
    {
        this.b = b;
    }
}

this()에 인자를 넘겨서 다른 생성자를 지정할 수도 있습니다. 생성자에서만 활용 가능한 문법입니다.

Part 3: 확장 메서드의 this

정적 함수 파라미터의 this는 무엇을 의미하나요?

정적 함수의 첫 번째 파라미터에 this를 붙이면 확장 메서드가 됩니다. 이를 통해 마치 멤버 함수를 호출하듯 함수를 사용할 수 있습니다.

C#
public static void Shuffle<T>(this IList<T> list)
{
    // 구현
}

List<MyClass> exList = new List<MyClass>();
Shuffle(exList);  // 일반 정적 메서드 호출
exList.Shuffle(); // 확장 메서드 호출 - this 덕분에!

기존 클래스의 함수와 동일한 시그니처로 확장 메서드를 정의하면 호출되지 않습니다. 컴파일러가 인스턴스 함수를 먼저 탐색하고, 그 다음에 확장 메서드를 보기 때문입니다.

📌 핵심 정리

  • 인스턴스 this: 현재 객체 인스턴스를 가리키며, 필드와 매개변수 이름 충돌 시 구분에 사용
  • 생성자 this(): 생성자 간 코드 중복을 줄이기 위한 생성자 체이닝
  • 확장 메서드 this: 정적 메서드를 인스턴스 메서드처럼 호출할 수 있게 해주는 문법
  • 확장 메서드는 인스턴스 메서드보다 우선순위가 낮다
