C#의 const와 readonly - 컴파일 타임 vs 런타임 상수의 차이

Part 1: const와 readonly의 차이

C#에서 const와 readonly는 어떻게 다른가요?
구분 const readonly
상수 시점 컴파일 타임 (값으로 대체) 런타임 (참조 코드 생성)
메모리 위치 스택 힙
초기화 시점 선언과 동시에 생성자에서 가능
사용 가능 타입 내장 자료형만 모든 타입

Part 2: 왜 readonly가 더 좋은가?

Effective C#에서 "const보다 readonly를 사용하라"고 하는 이유는?

가장 큰 차이는 readonly가 상수에 대한 참조 코드를 생성한다는 점입니다.

const의 값을 변경하면 이를 사용하는 모든 곳을 재컴파일해야 합니다. 하지 않으면 이전 const 값이 그대로 남아 있어 버그가 발생합니다.

readonly의 경우 일부만 리빌드해도 다른 코드들은 참조를 가지고 있으므로 리빌드 없이 올바르게 동작합니다.

🎯 비유로 이해하기

const는 마치 복사기로 찍어 나눠준 종이와 같습니다. 원본이 바뀌어도 이미 나눠준 종이는 그대로입니다.

readonly는 게시판에 붙여둔 공지와 같습니다. 공지를 수정하면 모든 사람이 최신 내용을 볼 수 있습니다.

그래도 const를 써야 하는 경우가 있나요?

네, 아래의 경우에는 const를 사용해야 합니다:

  • 특성(Attribute)의 매개변수
  • switch/case 문의 레이블
  • enum 정의

위 경우를 제외하면 가급적 readonly를 사용하는 것이 유연성과 유지보수 면에서 좋습니다.

📌 핵심 정리

  • const는 컴파일 타임 상수(스택), readonly는 런타임 상수(힙)이다
  • const 변경 시 모든 참조 코드 재컴파일 필요, readonly는 불필요
  • readonly는 생성자에서 초기화 가능하고 모든 타입과 사용 가능
  • Attribute, switch/case, enum 외에는 readonly 사용을 권장
