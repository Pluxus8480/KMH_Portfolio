Part 1: 패턴 매칭이란?

기존 if-else나 switch의 어떤 점이 문제인가요?

게임 개발에서 자주 마주치는 상황을 봅시다. 아이템 타입에 따라 다른 처리를 해야 할 때:

C# - 전통적인 방식 (if-else 지옥)
void UseItem(Item item)
{
    if (item is WeaponItem)
    {
        var weapon = (WeaponItem)item;
        if (weapon.Durability > 0)
        {
            if (weapon.Type == WeaponType.Melee)
                MeleeAttack(weapon);
            else if (weapon.Type == WeaponType.Ranged)
                RangedAttack(weapon);
        }
        else
        {
            ShowBrokenMessage();
        }
    }
    else if (item is PotionItem)
    {
        var potion = (PotionItem)item;
        if (potion.Quantity > 0)
            DrinkPotion(potion);
    }
    // ... 더 많은 분기가 계속 중첩됨
}

패턴 매칭은 타입 검사, 캐스팅, 조건 판단을 하나의 표현식으로 결합하여 가독성을 극적으로 개선합니다.

C#에서 패턴 매칭은 어떤 버전부터 지원되나요?
C# 버전 추가된 패턴 예시
C# 7.0 타입 패턴, const 패턴 case int x:
C# 8.0 switch 표현식, 속성 패턴, 튜플 패턴 { Length: > 0 }
C# 9.0 관계/논리 패턴, not/and/or > 0 and < 100
C# 10.0 확장 속성 패턴 { Inner.Value: 42 }
C# 11.0 리스트 패턴 [1, 2, .., 9]

Part 2: 타입 패턴과 switch 표현식

위의 if-else 코드를 패턴 매칭으로 바꾸면 어떻게 되나요?
C# - switch 표현식으로 리팩토링
void UseItem(Item item)
{
    var action = item switch
    {
        WeaponItem { Durability: > 0, Type: WeaponType.Melee } w
            => () => MeleeAttack(w),

        WeaponItem { Durability: > 0, Type: WeaponType.Ranged } w
            => () => RangedAttack(w),

        WeaponItem { Durability: <= 0 }
            => () => ShowBrokenMessage(),

        PotionItem { Quantity: > 0 } p
            => () => DrinkPotion(p),

        _ => () => ShowInvalidMessage()
    };

    action();
}

하나의 switch 표현식 안에서 타입 검사, 속성 조건, 변수 바인딩이 모두 이루어집니다.

switch 표현식은 모든 경우를 빠짐없이 처리하도록 컴파일러가 강제합니다. _ (discard) 패턴은 기본 케이스입니다.

is 패턴은 어떻게 사용하나요?
C# - is 패턴 활용
// 타입 패턴 + 변수 바인딩
if (gameObject is Enemy { IsAlive: true } enemy)
{
    enemy.TakeDamage(10);
}

// 부정 패턴
if (target is not null and not Boss)
{
    ApplyNormalDamage(target);
}

// 관계형 패턴
if (player.Health is > 0 and <= 25)
{
    ShowLowHealthWarning();
}

Part 3: 속성 패턴과 위치 패턴

속성 패턴을 더 자세히 설명해 주세요.

속성 패턴은 객체의 프로퍼티 값을 중괄호 안에서 직접 검사합니다.

C# - 속성 패턴 응용
// 게임 캐릭터 상태 판별
string GetCharacterState(Character c) => c switch
{
    { Health: 0 }
        => "사망",
    { Health: <= 25, IsShielded: false }
        => "위험",
    { Health: <= 25, IsShielded: true }
        => "보호 중",
    { Mana: 0, Health: > 25 }
        => "마나 고갈",
    { IsBuff: true, BuffDuration: > 0 }
        => "버프 활성",
    _
        => "정상"
};

// 중첩 속성 패턴 (C# 10)
string DescribeWeapon(Weapon w) => w switch
{
    { Stats.CritRate: >= 50.0 } => "치명타 특화",
    { Stats.AttackSpeed: >= 2.0 } => "속공형",
    { Enchantment.Element: Element.Fire } => "화염 부여",
    _ => "일반 무기"
};
위치 패턴(Positional Pattern)은 무엇인가요?

Deconstruct 메서드가 정의된 타입에 대해 튜플 형식으로 분해하여 매칭할 수 있습니다.

C# - 위치 패턴
record struct GridPos(int X, int Y);

string GetQuadrant(GridPos pos) => pos switch
{
    (0, 0) => "원점",
    ( > 0,  > 0) => "1사분면",
    ( < 0,  > 0) => "2사분면",
    ( < 0,  < 0) => "3사분면",
    ( > 0,  < 0) => "4사분면",
    (0, _) => "Y축 위",
    (_, 0) => "X축 위"
};

// 튜플 패턴으로 여러 변수를 동시에 매칭
string GetInteraction(bool hasKey, bool doorLocked) =>
    (hasKey, doorLocked) switch
{
    (true, true) => "문을 열었습니다!",
    (false, true) => "잠겨 있습니다. 열쇠가 필요합니다.",
    (_, false) => "문이 이미 열려 있습니다."
};

위치 패턴은 record 타입과 궁합이 좋습니다. record는 자동으로 Deconstruct를 생성하기 때문입니다.

Part 4: 관계형/논리 패턴

숫자 범위를 매칭할 수 있나요?

C# 9.0부터 >, <, >=, <= 관계 연산과 and, or, not 논리 조합이 가능합니다.

C# - 관계/논리 패턴 게임 활용
// 데미지 등급 판별
string GetDamageRank(float damage) => damage switch
{
    <= 0 => "MISS",
    > 0 and <= 50 => "약함",
    > 50 and <= 150 => "보통",
    > 150 and <= 500 => "강함",
    > 500 and <= 999 => "치명적",
    >= 1000 => "즉사급"
};

// 레벨에 따른 콘텐츠 해금
bool CanAccessContent(Player player, Content content) =>
    (player.Level, content.RequiredLevel, content.IsPremium) switch
    {
        (_, _, true) when !player.IsPremiumUser => false,
        (var lv, var req, _) when lv >= req => true,
        _ => false
    };
when 절은 언제 사용하나요?

when 절은 패턴 매칭만으로 표현하기 어려운 복잡한 조건을 추가할 때 사용합니다.

C# - when 가드 절
string EvaluateHit(Attack attack, Defense defense) =>
    (attack, defense) switch
{
    var (a, d) when a.Element == d.WeakElement
        => $"효과 발군! {a.Damage * 2} 데미지",

    var (a, d) when a.Element == d.ResistElement
        => $"효과 미비... {a.Damage / 2} 데미지",

    var (a, d) when a.IsCritical && d.Shield <= 0
        => $"크리티컬! {a.Damage * 1.5} 데미지",

    var (a, _)
        => $"{a.Damage} 데미지"
};

주의: when 절은 컴파일러가 완전성(exhaustiveness)을 검증하지 못합니다. 항상 기본 케이스(_)를 마지막에 추가하세요.

Part 5: 리스트 패턴 (C# 11)

리스트 패턴은 어떻게 동작하나요?

C# 11부터 배열이나 리스트의 요소를 패턴으로 매칭할 수 있습니다.

C# - 리스트 패턴
// 기본 리스트 패턴
int[] scores = { 100, 85, 92 };

string result = scores switch
{
    [] => "점수 없음",
    [100] => "만점 1과목",
    [100, 100, 100] => "올 만점!",
    [>= 90, >= 90, >= 90] => "전과목 A",
    [_, _, < 60] => "마지막 과목 과락",
    [var first, .., var last] => $"첫 점수: {first}, 마지막: {last}"
};

// 커맨드 입력 패턴 매칭
string ParseCommand(string[] tokens) => tokens switch
{
    ["attack", var target] => $"{target} 공격!",
    ["use", var item, "on", var target] => $"{target}에게 {item} 사용",
    ["move", var dir] when dir is "north" or "south" or "east" or "west"
        => $"{dir} 방향으로 이동",
    [var cmd, ..] => $"알 수 없는 명령: {cmd}",
    [] => "명령어를 입력하세요"
};

..은 슬라이스 패턴으로, 0개 이상의 요소를 건너뛸 수 있습니다. 리스트의 시작과 끝만 검사할 때 유용합니다.

Part 6: 게임 상태 관리 실전 예제

패턴 매칭을 게임 FSM에 적용하면 어떻게 될까요?
C# - 패턴 매칭 기반 게임 상태 전이
record GameEvent;
record DamageEvent(float Amount) : GameEvent;
record HealEvent(float Amount) : GameEvent;
record InputEvent(string Key) : GameEvent;

enum PlayerState { Idle, Running, Attacking, Stunned, Dead }

PlayerState Transition(PlayerState current, GameEvent evt) =>
    (current, evt) switch
{
    (PlayerState.Dead, _)
        => PlayerState.Dead, // 사망 상태는 변경 불가

    (_, DamageEvent { Amount: >= 100 })
        => PlayerState.Stunned,

    (_, DamageEvent d) when GetHealth() - d.Amount <= 0
        => PlayerState.Dead,

    (PlayerState.Idle, InputEvent { Key: "Space" })
        => PlayerState.Attacking,

    (PlayerState.Idle, InputEvent { Key: "W" or "A" or "S" or "D" })
        => PlayerState.Running,

    (PlayerState.Running, InputEvent { Key: "Space" })
        => PlayerState.Attacking,

    (PlayerState.Stunned, _) when StunTimer <= 0
        => PlayerState.Idle,

    _ => current // 상태 유지
};

패턴 매칭을 사용한 FSM은 상태 전이 규칙이 한눈에 보이며, 새로운 상태나 이벤트를 추가할 때 컴파일러가 누락된 케이스를 잡아줍니다.

📌 핵심 정리

  • switch 표현식은 값을 반환하며, 모든 케이스의 완전성을 컴파일러가 검증한다.
  • 속성 패턴은 객체의 프로퍼티를 중괄호 안에서 직접 매칭한다.
  • 위치 패턴은 Deconstruct를 지원하는 타입을 튜플처럼 분해하여 매칭한다.
  • 관계/논리 패턴(and, or, not, >, <)으로 복합 조건을 간결하게 표현할 수 있다.
  • 게임 FSM, 커맨드 파싱, 상태 분류 등에서 if-else보다 훨씬 읽기 쉬운 코드를 작성할 수 있다.
