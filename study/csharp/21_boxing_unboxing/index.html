박싱(Boxing)과 언박싱(Unboxing) - 값 타입과 참조 타입 사이의 비용

Part 1: 값 타입과 참조 타입

박싱과 언박싱을 이해하려면 먼저 무엇을 알아야 하나요?

값 타입과 참조 타입을 이해해야 합니다.

구분 값 타입 참조 타입
예시 구조체, 열거형 모든 클래스
상속 System.ValueType System.Object
메모리 위치 스레드 스택 힙 (주소는 스택)

Part 2: 박싱과 언박싱이란?

박싱과 언박싱의 정의는?

박싱: 값 타입을 참조 타입으로 변환 (스택 → 힙)

언박싱: 참조 타입을 값 타입으로 변환 (힙 → 스택)

C#
int value = 42;

// 박싱 - 값 타입(int)을 참조 타입(object)으로
object boxed = value;

// 언박싱 - 참조 타입(object)을 값 타입(int)으로
int unboxed = (int)boxed;
🎯 비유로 이해하기

박싱은 작은 메모를 택배 상자에 넣는 것과 같습니다. 메모(값)는 그대로인데, 상자(힙 공간)를 할당하고 포장하는 비용이 듭니다.

언박싱은 상자를 뜯어서 메모를 꺼내는 것입니다. 역시 추가 작업이 필요합니다.

왜 박싱/언박싱을 피해야 하나요?

박싱/언박싱은 기본 연산에 비해 비용이 크며, 힙에 가비지가 쌓여 GC에 부담을 줍니다. 가급적 사용하지 않는 것이 좋습니다.

어떻게 해결할 수 있나요?

제네릭(Generic)을 활용하면 박싱/언박싱 없이 타입 안전한 코드를 작성할 수 있습니다.

C#
// 나쁜 예 - ArrayList는 object를 사용하므로 박싱 발생
ArrayList list = new ArrayList();
list.Add(42);  // 박싱!
int val = (int)list[0];  // 언박싱!

// 좋은 예 - List<int>는 제네릭이므로 박싱 없음
List<int> list = new List<int>();
list.Add(42);  // 박싱 없음
int val = list[0];  // 언박싱 없음

📌 핵심 정리

  • 값 타입은 스택, 참조 타입은 힙에 저장된다
  • 박싱은 값→참조, 언박싱은 참조→값 타입 변환이다
  • 박싱/언박싱은 힙 할당과 GC 부담을 유발하므로 가급적 피해야 한다
  • 제네릭(Generic)을 활용하면 박싱/언박싱을 방지할 수 있다
