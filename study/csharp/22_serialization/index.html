게임을 종료해도 데이터가 살아있는 비결

직렬화(Serialization)와 역직렬화(Deserialization) - 객체에 영속성 부여하기

Part 1: 직렬화와 역직렬화란?

직렬화와 역직렬화의 정의는?

직렬화(Serialization): 객체를 바이트 단위로 변환하여 디스크에 저장하거나 네트워크로 전송할 수 있게 만드는 과정

역직렬화(Deserialization): 직렬화된 바이트 배열을 원래 객체로 복원하는 과정

🎯 비유로 이해하기

직렬화는 3D 가구를 납작하게 분해해서 택배 상자에 넣는 것과 같습니다. 보내기 쉽고 저장하기 쉬운 형태로 변환하는 것이죠.

역직렬화는 택배로 받은 부품들을 다시 조립해서 원래 가구로 만드는 것입니다.

Part 2: 왜 직렬화를 하는가?

직렬화의 주된 목적은 무엇인가요?

데이터에 영속성(Persistence)을 부여하기 위함입니다. 영속성이란 프로그램이 종료되더라도 사라지지 않는 특성을 의미합니다.

프로그램 종료 후에도 객체 정보를 유지하고 싶을 때 직렬화를 사용합니다. 게임에서는 플레이어 세이브 데이터가 대표적인 예입니다.

Part 3: 유니티에서 직렬화 가능한 조건

유니티에서 모든 것을 직렬화할 수 있나요?

아닙니다. 유니티에서는 직렬화를 위한 조건이 있으며, 다른 .NET 환경과 약간 다르게 동작합니다.

직렬화 가능 조건:

  • public이거나 [SerializeField] 속성이 있어야 함
  • static, const, readonly가 아니어야 함
  • 직렬화 가능한 필드 타입이어야 함

직렬화 가능한 필드 타입:

  • [Serializable] 속성이 있는 비추상, 비일반 커스텀 클래스
  • [Serializable] 속성이 있는 커스텀 구조체
  • UnityEngine.Object에서 파생된 오브젝트 레퍼런스
  • int, double, bool 같은 기본 데이터 형식
  • 열거형(enum) 타입
  • Vector2, Vector3, Color 등 Unity 내장 타입
C#
[Serializable]
public class PlayerData
{
    public string playerName;     // 직렬화 O (public)
    public int score;             // 직렬화 O (기본 타입)

    [SerializeField]
    private int health;           // 직렬화 O (SerializeField)

    private int secretValue;      // 직렬화 X (private, 속성 없음)
    public static int totalCount; // 직렬화 X (static)
    public const int MAX = 100;   // 직렬화 X (const)
}

📌 핵심 정리

  • 직렬화는 객체를 바이트로 변환하여 저장/전송하는 과정이다
  • 데이터에 영속성을 부여하기 위해 사용한다 (세이브 데이터 등)
  • 유니티에서는 public 또는 [SerializeField]가 있어야 직렬화 가능하다
  • static, const, readonly 필드는 직렬화할 수 없다
