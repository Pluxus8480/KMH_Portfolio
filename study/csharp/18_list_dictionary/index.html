C#의 컨테이너, 속은 C++과 같다?

List, Dictionary, SortedSet - 내부 자료구조와 시간복잡도 완전 분석

Part 1: C# 컨테이너의 내부 자료구조

C#의 주요 컨테이너들은 내부적으로 어떤 자료구조를 사용하나요?

C++에 대한 지식이 있다면 쉽게 대응시킬 수 있습니다. List는 vector, Dictionary는 unordered_map으로 생각하면 편합니다.

C# 컨테이너 내부 자료구조 C++ 대응
List<> 배열 vector
SortedSet<> 레드-블랙 트리 set
HashSet<> 해시 테이블 unordered_set
Dictionary<,> 해시 테이블 unordered_map
SortedList<,> 배열 -
SortedDictionary<,> 레드-블랙 트리 map

Part 2: List - C++의 vector와 유사하다

List의 삽입 시 최악의 시간복잡도가 O(N)인 이유는?

List는 메모리에 배열처럼 연속으로 올라갑니다. 원소 삽입 시 용량을 초과하면 새 공간을 할당해 기존 원소들을 복사하므로 최대 O(N) 시간이 소요됩니다.

Remove의 경우에는 별도로 용량 변화가 없습니다. 내부 배열 크기는 유지됩니다.

Part 3: 해시 기반 vs 트리 기반

해시 테이블과 레드-블랙 트리의 성능 차이는?

레드-블랙 트리를 사용하는 자료구조(SortedSet, SortedDictionary)는 정렬된 완전 이진트리이므로 삽입, 삭제에 O(logN)이 소요됩니다.

해시를 사용하는 자료구조(HashSet, Dictionary)는 대부분 O(1)이지만, 해시 충돌이 심한 최악의 경우 O(N)이 될 수 있습니다.

Part 4: Dictionary vs SortedDictionary

둘 중 어떤 것을 사용해야 하나요?

대부분의 경우 Dictionary가 성능이 더 뛰어납니다.

50,000개 삽입 기준으로 Dictionary가 약 3배 빠르고, 검색에서는 SortedDictionary가 아주 근소한 차이로 빠릅니다.

항시 정렬된 상태로 데이터를 저장해야 하는 경우를 제외하면 Dictionary 사용이 권장됩니다.

📌 핵심 정리

  • List는 배열 기반(vector), Dictionary는 해시 기반(unordered_map)이다
  • List 삽입 시 용량 초과하면 O(N) 복사가 발생한다
  • 해시 기반은 O(1) 평균, 트리 기반은 O(logN) 보장이다
  • 정렬이 필요하지 않다면 Dictionary를 사용하는 것이 성능상 유리하다
