Part 1: AI 시스템 개요

언리얼의 AI 시스템은 어떻게 구성되어 있나요?

언리얼 AI는 세 가지 핵심 컴포넌트로 구성됩니다:

• AI Controller - AI의 "뇌", 의사결정 주체

• Behavior Tree - 행동 로직을 정의하는 "사고 과정"

• Blackboard - AI의 "기억", 데이터 저장소

Part 3: Blackboard

Blackboard는 무엇인가요?

Blackboard는 AI의 기억 저장소입니다. 타겟 위치, 적 참조, 상태 플래그 등을 저장하고 Behavior Tree에서 읽어 사용합니다.

Blackboard 설정
// Blackboard Asset에서 정의

Key Type: Vector

Key Type: Bool

// C++에서 Blackboard 값 설정
void AMyAIController::SetTargetActor(AActor* Target)
{
    if (UBlackboardComponent* BB = GetBlackboardComponent())
    {
        BB->SetValueAsObject(FName("TargetActor"), Target);
        BB->SetValueAsVector(FName("TargetLocation"),
            Target->GetActorLocation());
    }
}

// Behavior Tree Task에서 값 읽기
EBTNodeResult::Type UBTTask_Attack::ExecuteTask(
    UBehaviorTreeComponent& OwnerComp, uint8* NodeMemory)
{
    UBlackboardComponent* BB = OwnerComp.GetBlackboardComponent();
    AActor* Target = Cast(
        BB->GetValueAsObject(TargetKey.SelectedKeyName));

    if (Target)
    {
        // 공격 수행
        return EBTNodeResult::Succeeded;
    }
    return EBTNodeResult::Failed;
}

Part 4: Task, Decorator, Service

커스텀 Task는 어떻게 만드나요?

UBTTaskNode를 상속받아 구현합니다. ExecuteTask에서 행동을 수행합니다.

커스텀 Task
UCLASS()
class UBTTask_FindPatrolPoint : public UBTTaskNode
{
    GENERATED_BODY()

public:
    UPROPERTY(EditAnywhere)
    FBlackboardKeySelector PatrolPointKey;

    virtual EBTNodeResult::Type ExecuteTask(
        UBehaviorTreeComponent& OwnerComp,
        uint8* NodeMemory) override
    {
        AMyAIController* AIController =
            Cast(OwnerComp.GetAIOwner());

        if (AIController)
        {
            FVector NextPoint = AIController->GetNextPatrolPoint();

            OwnerComp.GetBlackboardComponent()->SetValueAsVector(
                PatrolPointKey.SelectedKeyName, NextPoint);

            return EBTNodeResult::Succeeded;
        }

        return EBTNodeResult::Failed;
    }
};
Decorator와 Service의 차이점은?

Decorator는 조건문입니다. 조건이 참이면 하위 노드 실행, 거짓이면 스킵합니다.

Service는 주기적으로 실행되는 업데이트 로직입니다. Blackboard 값 갱신에 사용합니다.

Service 예시 - 적 감지
UCLASS()
class UBTService_CheckForEnemy : public UBTService
{
    GENERATED_BODY()

public:
    UBTService_CheckForEnemy()
    {
        Interval = 0.5f;  // 0.5초마다 실행
        RandomDeviation = 0.1f;
    }

protected:
    virtual void TickNode(UBehaviorTreeComponent& OwnerComp,
        uint8* NodeMemory, float DeltaSeconds) override
    {
        // 적 감지 로직
        AActor* Enemy = FindNearestEnemy(OwnerComp);

        // Blackboard 업데이트
        OwnerComp.GetBlackboardComponent()->SetValueAsObject(
            FName("TargetEnemy"), Enemy);
    }
};

Part 5: EQS (Environment Query System)

AI가 "어디로 이동할지"는 어떻게 결정하나요?

EQS를 사용합니다. 환경을 쿼리해서 최적의 위치를 찾습니다. 예: "적에게서 떨어지면서 엄폐물 뒤"

EQS 쿼리 예시 - "엄폐 위치 찾기"
1. Generator: 주변 포인트 생성
AI 주변에 그리드 형태로 탐색 포인트 생성
↓
2. Tests: 각 포인트 점수 계산
  • 적과의 거리 (멀수록 +)
  • 시야선 차단 여부 (차단되면 +)
  • 네비게이션 가능 (불가능하면 제외)
↓
3. 결과: 최고 점수 포인트 반환
적 시야 밖, 엄폐물 뒤의 최적 위치 선택

Part 6: AI Perception

AI가 적을 감지하는 방법은?

AI Perception Component를 사용합니다. 시야, 청각, 촉각 등 다양한 감각을 시뮬레이션합니다.

AI Perception 설정
// AI Controller에서 Perception 설정
AMyAIController::AMyAIController()
{
    // Perception Component 생성
    PerceptionComponent = CreateDefaultSubobject(
        TEXT("PerceptionComp"));

    // 시야 감각 설정
    SightConfig = CreateDefaultSubobject(
        TEXT("SightConfig"));
    SightConfig->SightRadius = 1500.0f;
    SightConfig->LoseSightRadius = 2000.0f;
    SightConfig->PeripheralVisionAngleDegrees = 60.0f;
    SightConfig->DetectionByAffiliation.bDetectEnemies = true;

    PerceptionComponent->ConfigureSense(*SightConfig);
    PerceptionComponent->SetDominantSense(SightConfig->GetSenseImplementation());

    // 감지 이벤트 바인딩
    PerceptionComponent->OnTargetPerceptionUpdated.AddDynamic(
        this, &AMyAIController::OnTargetPerceptionUpdated);
}

void AMyAIController::OnTargetPerceptionUpdated(
    AActor* Actor, FAIStimulus Stimulus)
{
    if (Stimulus.WasSuccessfullySensed())
    {
        // 적 발견!
        SetTargetEnemy(Actor);
    }
    else
    {
        // 적을 놓침
        ClearTargetEnemy();
    }
}
감각 유형 용도
Sight 시야 내 Actor 감지
Hearing 소리 이벤트 감지
Damage 피해를 받았을 때
Touch 물리적 접촉
Team 같은 팀 감지

📌 핵심 정리

  • Behavior Tree는 트리 구조로 AI 행동 로직을 정의합니다
  • Blackboard는 AI의 기억, Behavior Tree와 데이터 공유
  • Selector는 OR, Sequence는 AND 조건
  • EQS로 환경을 쿼리해서 최적의 위치를 찾습니다
  • AI Perception으로 시야, 청각 등 감각을 시뮬레이션합니다
