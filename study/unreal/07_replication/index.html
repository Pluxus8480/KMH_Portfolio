Replication과 네트워크 동기화

Part 1: Replication의 기본 개념

Replication이 뭔가요?

Replication은 서버의 게임 상태를 클라이언트에 복제하는 것입니다. 멀티플레이어 게임에서 모든 플레이어가 "같은 세상"을 보게 만듭니다.

언리얼은 서버 권한(Server Authority) 모델을 사용합니다. 서버가 진실의 원천이고, 클라이언트는 그것을 복제받습니다.

Part 2: Actor Replication

Actor를 복제하려면 어떻게 해야 하나요?

Actor 자체가 복제되려면 bReplicates = true로 설정해야 합니다. 이것만으로도 Actor의 생성/파괴가 동기화됩니다.

Actor Replication 기본 설정
AMyActor::AMyActor()
{
    // Actor 복제 활성화
    bReplicates = true;

    // 이동 복제 (Transform 변경 자동 동기화)
    SetReplicateMovement(true);

    // 복제 빈도 (기본 100Hz)
    NetUpdateFrequency = 100.0f;

    // 최소 복제 빈도
    MinNetUpdateFrequency = 33.0f;
}
설정 효과
bReplicates Actor 자체의 복제 여부
SetReplicateMovement Transform 자동 복제
NetUpdateFrequency 최대 업데이트 빈도
NetPriority 복제 우선순위

Part 3: Property Replication

변수를 복제하려면 어떻게 하나요?

UPROPERTY(Replicated)로 선언하고, GetLifetimeReplicatedProps에 등록합니다.

Property Replication
// 헤더 파일
UCLASS()
class AMyCharacter : public ACharacter
{
    GENERATED_BODY()

public:
    // 복제되는 변수
    UPROPERTY(Replicated)
    float Health;

    // 변경 시 콜백을 받는 복제 변수
    UPROPERTY(ReplicatedUsing = OnRep_Ammo)
    int32 Ammo;

    UFUNCTION()
    void OnRep_Ammo();

    // 필수: 복제할 프로퍼티 목록
    virtual void GetLifetimeReplicatedProps(
        TArray& OutLifetimeProps) const override;
};

// 소스 파일
void AMyCharacter::GetLifetimeReplicatedProps(
    TArray& OutLifetimeProps) const
{
    Super::GetLifetimeReplicatedProps(OutLifetimeProps);

    // 기본 복제
    DOREPLIFETIME(AMyCharacter, Health);

    // 조건부 복제 (소유자에게만)
    DOREPLIFETIME_CONDITION(AMyCharacter, Ammo, COND_OwnerOnly);
}

void AMyCharacter::OnRep_Ammo()
{
    // 클라이언트에서 Ammo가 변경됐을 때 호출
    // UI 업데이트 등 처리
    UpdateAmmoUI(Ammo);
}
조건 복제 대상
COND_None 모든 클라이언트
COND_OwnerOnly 소유자 클라이언트만
COND_SkipOwner 소유자 제외
COND_SimulatedOnly 시뮬레이팅 프록시만
COND_InitialOnly 초기값만 (한 번)

Part 4: RPC (Remote Procedure Call)

함수를 원격으로 호출하려면 어떻게 하나요?

RPC를 사용합니다. 세 가지 유형이 있습니다:

RPC 유형
UCLASS()
class AMyCharacter : public ACharacter
{
    GENERATED_BODY()

public:
    // Server RPC: 클라이언트 → 서버
    // 클라이언트에서 호출하면 서버에서 실행
    UFUNCTION(Server, Reliable)
    void Server_Fire();

    // Client RPC: 서버 → 특정 클라이언트
    // 서버에서 호출하면 해당 클라이언트에서 실행
    UFUNCTION(Client, Reliable)
    void Client_ShowDamageNumber(float Damage);

    // Multicast RPC: 서버 → 모든 클라이언트
    // 서버에서 호출하면 모든 클라이언트에서 실행
    UFUNCTION(NetMulticast, Unreliable)
    void Multicast_PlayExplosionEffect();
};

// 사용 예시
void AMyCharacter::Fire()
{
    if (!HasAuthority())  // 클라이언트라면
    {
        Server_Fire();    // 서버에게 요청
    }
    else                  // 서버라면
    {
        // 실제 발사 로직
        PerformFire();
    }
}

void AMyCharacter::Server_Fire_Implementation()
{
    // 서버에서 실행됨
    if (CanFire())
    {
        PerformFire();
        Multicast_PlayFireEffect();  // 모두에게 이펙트
    }
}
RPC 유형 호출자 실행자 용도
Server 클라이언트 서버 입력, 액션 요청
Client 서버 소유 클라이언트 UI, 피드백
NetMulticast 서버 모든 클라이언트 이펙트, 사운드
Reliable과 Unreliable의 차이는?

Reliable: 반드시 도착 보장, 순서 보장. 중요한 게임플레이에 사용

Unreliable: 도착 미보장, 빠름. 이펙트, 애니메이션에 사용

Reliable RPC를 남발하면 대역폭 문제 발생!

Part 5: 네트워크 역할 (Role)

Actor의 네트워크 역할은 어떻게 구분하나요?

각 Actor는 Role과 RemoteRole을 가집니다.

SERVER

Player1의 캐릭터: Authority (권한)

Player2의 캐릭터: Authority (권한)

▼
CLIENT 1

내 캐릭터: AutonomousProxy (자율 프록시)

상대 캐릭터: SimulatedProxy (시뮬레이션)

CLIENT 2

내 캐릭터: AutonomousProxy (자율 프록시)

상대 캐릭터: SimulatedProxy (시뮬레이션)

같은 Actor가 위치에 따라 다른 Role

Role 확인 방법
void AMyCharacter::SomeFunction()
{
    // 권한 확인 (서버인지)
    if (HasAuthority())
    {
        // 서버에서만 실행할 코드
    }

    // 로컬 컨트롤 확인 (내가 조작하는지)
    if (IsLocallyControlled())
    {
        // 로컬 플레이어만 실행할 코드
    }

    // Role로 분기
    switch (GetLocalRole())
    {
        case ROLE_Authority:
            // 서버
            break;
        case ROLE_AutonomousProxy:
            // 로컬 플레이어의 캐릭터
            break;
        case ROLE_SimulatedProxy:
            // 다른 플레이어의 캐릭터
            break;
    }
}

Part 6: Ownership과 관련성

어떤 Actor가 누구에게 복제되나요?

Relevancy(관련성)가 있는 Actor만 복제됩니다. 멀리 있는 Actor는 복제하지 않아 대역폭을 절약합니다.

Relevancy 조건 설명
bAlwaysRelevant 항상 모든 클라이언트에 복제
bNetUseOwnerRelevancy Owner의 Relevancy 따름
bOnlyRelevantToOwner Owner에게만 복제
NetCullDistanceSquared 거리 기반 컬링
Relevancy 설정
AMyActor::AMyActor()
{
    bReplicates = true;

    // 항상 복제 (글로벌 매니저 등)
    bAlwaysRelevant = true;

    // 또는 거리 기반 컬링
    bAlwaysRelevant = false;
    NetCullDistanceSquared = 150000000.0f; // ~12km
}

// Owner 설정
void AMyCharacter::PickupWeapon(AWeapon* Weapon)
{
    // 이 무기의 Owner를 나로 설정
    // 이제 이 무기는 나의 Relevancy를 따름
    Weapon->SetOwner(this);
}

📌 핵심 정리

  • 언리얼은 서버 권한 모델 - 서버가 진실의 원천입니다
  • bReplicates로 Actor를, UPROPERTY(Replicated)로 변수를 복제합니다
  • RPC는 Server(클라→서버), Client(서버→클라), Multicast(서버→모두)
  • Authority는 서버, AutonomousProxy는 내 캐릭터, SimulatedProxy는 남의 캐릭터
  • Relevancy로 필요한 Actor만 복제하여 대역폭 최적화
