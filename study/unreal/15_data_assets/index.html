데이터 기반 게임플레이 (Data Assets)

Part 1: 데이터 주도 설계란?

왜 데이터를 코드에서 분리해야 하나요?

아이템이 100개인 게임에서 데미지 밸런스를 조정한다고 가정해봅시다. 모든 값이 코드에 있다면? 코드 수정 → 컴파일 → 테스트를 100번 반복해야 합니다.

데이터를 분리하면 에셋만 수정하고 바로 테스트할 수 있습니다. 디자이너도 프로그래머 없이 밸런스를 조정할 수 있죠.

🎯 비유로 이해하기

하드코딩은 레시피가 요리사 머릿속에만 있는 것입니다.

데이터 주도 설계는 레시피 북을 만들어 누구나 수정할 수 있게 하는 것입니다.

Part 2: Primary Data Asset

Data Asset은 어떻게 만드나요?

UPrimaryDataAsset을 상속받아 정의합니다. Content Browser에서 에셋으로 생성하고 관리합니다.

Data Asset 정의
// ItemDataAsset.h
UCLASS(BlueprintType)
class UItemDataAsset : public UPrimaryDataAsset
{
    GENERATED_BODY()

public:
    // 고유 ID (Asset Manager용)
    virtual FPrimaryAssetId GetPrimaryAssetId() const override
    {
        return FPrimaryAssetId("Item", GetFName());
    }

    // 아이템 정보
    UPROPERTY(EditDefaultsOnly, BlueprintReadOnly, Category = "Info")
    FText ItemName;

    UPROPERTY(EditDefaultsOnly, BlueprintReadOnly, Category = "Info")
    FText Description;

    UPROPERTY(EditDefaultsOnly, BlueprintReadOnly, Category = "Info")
    UTexture2D* Icon;

    // 스탯
    UPROPERTY(EditDefaultsOnly, BlueprintReadOnly, Category = "Stats")
    float Damage;

    UPROPERTY(EditDefaultsOnly, BlueprintReadOnly, Category = "Stats")
    float AttackSpeed;

    // 가격
    UPROPERTY(EditDefaultsOnly, BlueprintReadOnly, Category = "Economy")
    int32 BuyPrice;

    UPROPERTY(EditDefaultsOnly, BlueprintReadOnly, Category = "Economy")
    int32 SellPrice;
};
Content Browser에서의 Data Asset:

Content/
├── Items/
│   ├── DA_Sword_Iron.uasset      ← UItemDataAsset
│   ├── DA_Sword_Steel.uasset
│   ├── DA_Sword_Legendary.uasset
│   └── ...
│
└── Enemies/
    ├── DA_Enemy_Goblin.uasset    ← UEnemyDataAsset
    ├── DA_Enemy_Orc.uasset

각 에셋을 더블클릭하면 에디터에서 값 수정 가능!

Part 3: Data Asset 사용

코드에서 Data Asset을 어떻게 사용하나요?

UPROPERTY로 참조하거나, Asset Manager를 통해 로드합니다.

직접 참조
// 무기 클래스에서 Data Asset 사용
UCLASS()
class AWeapon : public AActor
{
    GENERATED_BODY()

public:
    // 에디터에서 Data Asset 할당
    UPROPERTY(EditDefaultsOnly, Category = "Data")
    UItemDataAsset* WeaponData;

    float GetDamage() const
    {
        return WeaponData ? WeaponData->Damage : 0.0f;
    }

    void Attack(AActor* Target)
    {
        if (WeaponData)
        {
            float FinalDamage = WeaponData->Damage;
            UGameplayStatics::ApplyDamage(
                Target, FinalDamage, nullptr, this, nullptr);
        }
    }
};
Asset Manager로 로드
// 비동기 로드
void AMyCharacter::LoadWeaponData(FPrimaryAssetId AssetId)
{
    UAssetManager& AssetManager = UAssetManager::Get();

    // 비동기 로드 요청
    TArray Bundles;
    FStreamableDelegate Delegate = FStreamableDelegate::CreateUObject(
        this, &AMyCharacter::OnWeaponDataLoaded, AssetId);

    AssetManager.LoadPrimaryAsset(AssetId, Bundles, Delegate);
}

void AMyCharacter::OnWeaponDataLoaded(FPrimaryAssetId AssetId)
{
    // 로드된 에셋 가져오기
    UAssetManager& AssetManager = UAssetManager::Get();
    UItemDataAsset* Data = Cast(
        AssetManager.GetPrimaryAssetObject(AssetId));

    if (Data)
    {
        UE_LOG(LogTemp, Log, TEXT("Loaded: %s, Damage: %f"),
            *Data->ItemName.ToString(), Data->Damage);
    }
}

Part 4: Data Table

엑셀처럼 표 형태로 데이터를 관리하고 싶은데요?

Data Table을 사용합니다. CSV/JSON에서 임포트하거나 에디터에서 직접 편집할 수 있습니다.

Data Table Row 정의

Data Table 에디터

Data Table 사용
// 코드에서 Data Table 참조
UPROPERTY(EditDefaultsOnly)
UDataTable* ItemTable;

// Row 조회
FItemTableRow* GetItemData(FName RowName)
{
    if (ItemTable)
    {
        return ItemTable->FindRow(
            RowName, TEXT("ItemLookup"));
    }
    return nullptr;
}

// 사용 예
FItemTableRow* SwordData = GetItemData(FName("Iron_Sword"));
if (SwordData)
{
    float Damage = SwordData->Damage;  // 10
}

Part 5: Curve와 Composite Curve

레벨에 따른 스탯 증가 곡선은 어떻게 정의하나요?

Curve Asset을 사용합니다. X축에 레벨, Y축에 스탯을 매핑하고 곡선을 그립니다.

Curve Float: CC_DamageByLevel

스탯
│
100├                          ●
   │                      ●
 75├                  ●
   │              ●
 50├          ●
   │      ●
 25├  ●
   │●
  0└──────────────────────────→ 레벨
    0   5   10  15  20  25  30

비선형 성장 곡선으로 자연스러운 밸런스!

레벨-스탯 커브

Curve 사용
// Curve Float 참조
UPROPERTY(EditDefaultsOnly)
UCurveFloat* DamageCurve;

// 레벨에 따른 데미지 계산
float GetDamageForLevel(int32 Level)
{
    if (DamageCurve)
    {
        return DamageCurve->GetFloatValue((float)Level);
    }
    return 0.0f;
}

// Composite Curve Table (여러 스탯 한 번에)
UPROPERTY(EditDefaultsOnly)
UCurveTable* StatCurves;

// 복합 스탯 조회
void GetAllStatsForLevel(int32 Level, float& OutDamage,
    float& OutDefense, float& OutHP)
{
    if (StatCurves)
    {
        OutDamage = StatCurves->FindCurve(
            FName("Damage"), TEXT(""))->Eval((float)Level);
        OutDefense = StatCurves->FindCurve(
            FName("Defense"), TEXT(""))->Eval((float)Level);
        OutHP = StatCurves->FindCurve(
            FName("HP"), TEXT(""))->Eval((float)Level);
    }
}

Part 6: 베스트 프랙티스

어떤 데이터를 분리해야 하나요?

자주 조정될 값은 모두 데이터로 분리하세요:

데이터 유형 추천 방식
아이템/스킬 정보 Data Asset
대량의 유사 데이터 Data Table + CSV
레벨링 곡선 Curve Float/Table
게임 설정 Config/Settings Object
다국어 텍스트 String Table
검증 (Validation) 추가
// Data Asset에 검증 로직 추가
UCLASS()
class UItemDataAsset : public UPrimaryDataAsset
{
    // ... 기존 코드 ...

#if WITH_EDITOR
    // 에디터에서 저장 시 검증
    virtual EDataValidationResult IsDataValid(
        TArray& ValidationErrors) override
    {
        EDataValidationResult Result = EDataValidationResult::Valid;

        if (Damage < 0)
        {
            ValidationErrors.Add(
                FText::FromString("Damage cannot be negative!"));
            Result = EDataValidationResult::Invalid;
        }

        if (BuyPrice < SellPrice)
        {
            ValidationErrors.Add(
                FText::FromString("Buy price should be >= Sell price"));
            Result = EDataValidationResult::Invalid;
        }

        return Result;
    }
#endif
};

📌 핵심 정리

  • 데이터 주도 설계로 밸런스 조정이 컴파일 없이 가능
  • UPrimaryDataAsset: 개별 에셋으로 관리, Asset Manager 연동
  • Data Table: 표 형태, CSV 임포트/익스포트 지원
  • Curve: 레벨/시간에 따른 값 변화 정의
  • Validation을 추가해 데이터 오류 조기 발견
