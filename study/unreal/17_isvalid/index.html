nullptr 체크만으로 충분할까?

언리얼 엔진의 IsValid, == nullptr, check의 차이와 올바른 사용법

Part 1: IsValid 함수

IsValid는 단순한 null 체크인가요?

아닙니다. IsValid는 단순 nullptr 체크뿐 아니라 내부 플래그도 검사하는 작업을 거칩니다.

C++ - IsValid 구현
FORCEINLINE bool IsValid(const UObject *Test)
{
    return Test &&
        FInternalUObjectBaseUtilityIsValidFlagsChecker
            ::CheckObjectValidBasedOnItsFlags(Test);
}

IsValid는 세 가지를 검사합니다: nullptr인지, 가비지(Garbage)인지, PendingKill인지.

내부 플래그 검사는 구체적으로 뭘 확인하나요?
C++ - 플래그 검사 Functor
struct FInternalUObjectBaseUtilityIsValidFlagsChecker
{
    FORCEINLINE static bool CheckObjectValidBasedOnItsFlags(
        const UObject* Test)
    {
        return !Test->HasAnyFlags(
            RF_InternalPendingKill | RF_InternalGarbage);
    }
};

PendingKill과 InternalGarbage 플래그를 확인합니다. GC에 의해 수거 대기 중인 객체도 걸러내는 것이죠.

Part 2: == nullptr과의 차이

그럼 == nullptr은 뭐가 부족한 건가요?
구분 == nullptr IsValid
nullptr 검사 O O
PendingKill 검사 X O
Garbage 검사 X O
안전성 낮음 높음

== nullptr만으로는 GC가 수거 예정인 객체를 잡아내지 못합니다. UObject를 다룰 때는 IsValid를 사용하는 것이 안전합니다.

Part 3: check, ensure 등의 Assert

check와 ensure는 어떤 함수인가요?

이런 Assert 함수들은 에디터에서만 동작하는 함수로, 내부 조건이 true/false인지에 따라 동작이 달라집니다.

이들은 포인터의 operator bool() 연산에 기반하므로, 결론적으로는 nullptr과 비교하는 것과 동일한 결과를 냅니다.

중요: 빌드(Shipping)를 하게 되면 check/ensure 코드는 사라집니다. 따라서 런타임 안전성 검사로 사용해서는 안 됩니다.

정리하면 언제 어떤 걸 써야 하나요?
함수 용도 빌드 포함 여부
IsValid() UObject 유효성 검사 (nullptr + GC 상태) O (항상 포함)
== nullptr 단순 포인터 null 체크 O (항상 포함)
check() 개발 중 디버깅용 Assert X (빌드 시 제거)
ensure() 개발 중 경고용 Assert X (빌드 시 제거)

핵심 정리

  • IsValid는 nullptr 체크 + PendingKill + Garbage 플래그까지 검사하는 가장 안전한 방법이다
  • == nullptr은 단순 포인터 비교만 수행하므로, GC 대기 중인 UObject를 잡아내지 못한다
  • check/ensure는 에디터 전용 Assert로, Shipping 빌드에서는 코드가 제거된다
  • UObject를 다룰 때는 반드시 IsValid를 사용하는 것이 권장된다
