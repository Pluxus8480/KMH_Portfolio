Lumen 글로벌 일루미네이션의 원리

Part 1: 글로벌 일루미네이션이란?

직접 조명만으로는 왜 부족한가요?

현실의 빛은 물체에 부딪혀 반사(bounce)됩니다. 빨간 벽 옆에 서면 피부가 붉게 보이는 것처럼요. 이걸 간접 조명(Indirect Lighting)이라고 합니다.

직접 조명만 있으면 그림자 영역이 완전히 검게 되고, 비현실적으로 보입니다. 글로벌 일루미네이션(GI)은 이 간접 조명을 시뮬레이션합니다.

🎯 비유로 이해하기

직접 조명은 손전등으로 직접 비추는 것입니다.

글로벌 일루미네이션은 흰 천장이 빛을 반사해서 방 전체가 밝아지는 것입니다.

Part 2: Lumen의 핵심 기술

Lumen은 어떻게 실시간으로 GI를 계산하나요?

Lumen은 Software Ray Tracing과 Hardware Ray Tracing을 하이브리드로 사용합니다. 핵심은 "충분히 좋은" 근사치를 빠르게 계산하는 것입니다.

│     └─────────────────────────────────────────────┘    │

Part 3: Surface Cache와 Mesh Cards

Surface Cache는 무엇인가요?

모든 메시의 표면 정보를 아틀라스 텍스처에 저장합니다. 광선이 표면에 닿았을 때, 복잡한 머티리얼 계산 대신 캐시된 값을 바로 읽어옵니다.

Surface Cache 개념
// Surface Cache에 저장되는 정보
struct SurfaceCacheData {
    float3 Albedo;        // 기본 색상
    float3 Normal;        // 노멀 방향
    float3 Emissive;      // 발광 색상
    float Roughness;      // 거칠기
    float Metallic;       // 금속성
};

// 광선이 표면에 닿았을 때
HitResult hit = TraceRay(ray);
if (hit.Valid) {
    // 복잡한 머티리얼 계산 대신 캐시 조회
    SurfaceCacheData data = SampleSurfaceCache(hit.UV);
    return CalculateLighting(data, lightDirection);
}
Mesh Cards는 뭔가요?

메시를 6방향 카드(직육면체)로 단순화한 표현입니다. 복잡한 메시 대신 이 카드에서 빠르게 조명 정보를 샘플링합니다.

Mesh Cards가 너무 단순화되면 조명 누수(Light Leaking)가 발생할 수 있습니다.

Part 4: Software vs Hardware Ray Tracing

Lumen의 두 가지 레이 트레이싱 방식은 뭐가 다른가요?

Software 방식은 Signed Distance Field(SDF)를 사용하고, Hardware 방식은 RTX 코어를 사용합니다.

항목 Software (SDF) Hardware (RT Core)
필요 하드웨어 일반 GPU RTX GPU 필요
정확도 근사치 (SDF 해상도 제한) 정확한 지오메트리
성능 더 빠름 (단순 기하) 더 느림 (복잡 기하)
디테일 세밀한 디테일 손실 완전한 디테일
SDF Ray Marching 개념
// SDF를 이용한 레이 마칭
float TraceSDF(Ray ray) {
    float t = 0.0;

    for (int i = 0; i < MAX_STEPS; i++) {
        float3 p = ray.Origin + ray.Direction * t;

        // SDF 샘플링 - 가장 가까운 표면까지의 거리
        float d = SampleSDF(p);

        if (d < HIT_THRESHOLD) {
            return t;  // 히트!
        }

        // 안전하게 d만큼 전진 가능
        t += d;

        if (t > MAX_DISTANCE) break;
    }

    return -1.0;  // 미스
}

Part 5: Radiance Cache

Radiance Cache는 어떻게 동작하나요?

월드 공간에 프로브(probe)를 배치하고, 각 프로브가 주변 조명 정보를 저장합니다. 렌더링할 때는 가까운 프로브들의 값을 보간해서 사용합니다.

매 프레임 모든 프로브를 업데이트하는 게 아니라, 점진적으로 갱신합니다. 그래서 조명이 변하면 약간의 지연이 있습니다.

        ┌─────────────────────────────────────┐
        │        Radiance Cache 구조          │
        │                                     │
        │   ●─────●─────●─────●              │
        │   │     │     │     │              │
        │   ●─────●─────●─────●    ● = 프로브 │
        │   │     │     │     │              │
        │   ●─────●─────●─────●              │
        │         ↓                          │
        │   각 프로브에 저장되는 정보:        │
        │   • 6방향 조명 색상                 │
        │   • 스카이 가시성                   │
        │   • 이전 프레임 히스토리            │
        └─────────────────────────────────────┘

Part 6: 반사(Reflections)

Lumen에서 반사는 어떻게 처리되나요?

Lumen Reflections는 Screen Space와 Ray Traced 반사를 조합합니다.

화면에 보이는 영역은 SSR로, 화면 밖은 레이 트레이싱으로 처리합니다. 거친 표면은 Radiance Cache를 사용하고, 매끄러운 표면은 정확한 반사를 계산합니다.

표면 거칠기 반사 방식 품질/성능
Roughness > 0.4 Radiance Cache 사용 빠름, 부드러운 반사
Roughness 0.2~0.4 SSR + Ray Traced 혼합 중간
Roughness < 0.2 정확한 Ray Traced 고품질, 느림

📌 핵심 정리

  • Lumen은 실시간 글로벌 일루미네이션과 반사를 제공합니다
  • Surface Cache가 표면 정보를 미리 계산해 빠른 조회를 가능하게 합니다
  • Software RT는 SDF 기반으로 일반 GPU에서, Hardware RT는 RTX 코어를 사용합니다
  • Radiance Cache가 공간의 조명 정보를 저장하고 보간합니다
  • 조명 변화에 약간의 지연이 있지만, 실시간으로 동적 GI를 구현합니다
