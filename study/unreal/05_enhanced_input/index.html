입력 시스템의 진화

Enhanced Input System 완벽 활용

Part 1: 왜 새로운 입력 시스템이 필요했나?

기존 입력 시스템의 문제점은 무엇인가요?

UE4의 기존 입력 시스템은 Action/Axis Mapping이 프로젝트 설정에 하드코딩되어 있었습니다.

런타임에 키 바인딩을 바꾸거나, 상황에 따라 다른 입력 처리를 하기가 매우 어려웠죠. 예를 들어:

• 수영 중에는 WASD가 수영 동작으로
• 차량 탑승 중에는 같은 키가 운전 조작으로
• UI가 열려있으면 입력 무시

🎯 비유로 이해하기

기존 시스템은 하나의 리모컨에 모든 버튼이 고정된 것입니다.

Enhanced Input은 상황에 따라 버튼 기능이 바뀌는 스마트 리모컨입니다.

Part 2: Enhanced Input의 핵심 구성요소

Enhanced Input은 어떤 구조로 되어 있나요?

세 가지 핵심 요소가 있습니다:

Enhanced Input의 3단 구조

구성요소 역할 예시
Input Action 추상적인 "행동" 정의 IA_Move, IA_Jump, IA_Fire
Input Mapping Context 물리 키와 Action 연결 IMC_OnFoot, IMC_Vehicle

Part 3: Input Action 만들기

Input Action은 어떻게 정의하나요?

Input Action은 값의 타입을 정의합니다. bool(눌림/안눌림), 1D(아날로그 축), 2D(스틱/마우스), 3D(VR 컨트롤러) 등이 있습니다.

Input Action 설정
// IA_Move (2D 이동)
Value Type: Axis2D (Vector2D)
Description: "Character Movement"

// IA_Jump (버튼)
Value Type: Digital (bool)
Description: "Character Jump"

// IA_Look (마우스/스틱 회전)
Value Type: Axis2D (Vector2D)
Description: "Camera Rotation"
C++에서 Input Action 바인딩
void AMyCharacter::SetupPlayerInputComponent(UInputComponent* Input)
{
    Super::SetupPlayerInputComponent(Input);

    // Enhanced Input Component로 캐스팅
    if (UEnhancedInputComponent* EnhancedInput =
        Cast(Input))
    {
        // 이동 - 2D Axis
        EnhancedInput->BindAction(MoveAction, ETriggerEvent::Triggered,
            this, &AMyCharacter::Move);

        // 점프 - Started/Completed
        EnhancedInput->BindAction(JumpAction, ETriggerEvent::Started,
            this, &AMyCharacter::StartJump);
        EnhancedInput->BindAction(JumpAction, ETriggerEvent::Completed,
            this, &AMyCharacter::StopJump);
    }
}

void AMyCharacter::Move(const FInputActionValue& Value)

Part 4: Input Mapping Context 활용

상황에 따라 입력을 바꾸려면 어떻게 하나요?

Input Mapping Context(IMC)를 런타임에 추가/제거하면 됩니다. 우선순위도 설정할 수 있어서, 높은 우선순위의 IMC가 입력을 "소비"하면 낮은 IMC는 무시됩니다.

Mapping Context 전환
void AMyCharacter::EnterVehicle(AVehicle* Vehicle)
{
    if (APlayerController* PC = Cast(Controller))
    {
        if (UEnhancedInputLocalPlayerSubsystem* Subsystem =
            ULocalPlayer::GetSubsystem(
                PC->GetLocalPlayer()))
        {
            // 도보 컨텍스트 제거
            Subsystem->RemoveMappingContext(IMC_OnFoot);

            // 차량 컨텍스트 추가 (우선순위 1)
            Subsystem->AddMappingContext(IMC_Vehicle, 1);
        }
    }
}

void AMyCharacter::ExitVehicle()
{
    // 반대로 수행
    Subsystem->RemoveMappingContext(IMC_Vehicle);
    Subsystem->AddMappingContext(IMC_OnFoot, 0);
}
        IMC 우선순위 스택

        ┌─────────────────┐
        │  IMC_UIMenu     │ ← Priority 2 (최우선)
        │  (UI 입력만)    │   Escape, 마우스 클릭
        ├─────────────────┤
        │  IMC_Vehicle    │ ← Priority 1
        │  (차량 조작)    │   WASD = 운전
        ├─────────────────┤
        │  IMC_OnFoot     │ ← Priority 0 (기본)
        │  (도보 이동)    │   WASD = 걷기
        └─────────────────┘

    높은 우선순위가 입력을 소비하면
    낮은 우선순위는 해당 입력을 받지 못함

Part 5: Modifiers와 Triggers

Modifier는 어떤 것들이 있나요?

Modifier는 입력값을 변환합니다. 순서대로 적용됩니다.

Modifier 기능 사용 예
Dead Zone 일정 값 이하 무시 스틱 드리프트 방지
Negate 값 반전 Y축 반전 옵션
Scalar 값 스케일 마우스 감도
Swizzle 축 교환 XY → YX
Smooth 값 보간 부드러운 입력
Trigger는 언제 사용하나요?

Trigger는 Action이 발동되는 조건을 정의합니다.

Trigger 발동 시점 사용 예
Down 누르고 있는 동안 이동, 연사
Pressed 누른 순간 점프 시작
Released 뗀 순간 활 발사
Hold 일정 시간 누름 차지 공격
Tap 빠르게 눌렀다 뗌 회피
Chorded 다른 키와 조합 Shift+W = 달리기

Part 6: 실전 예제 - 복합 입력

Shift를 누르면 달리는 것은 어떻게 구현하나요?

Chorded Action Trigger를 사용합니다.

달리기 구현 (IMC 설정)
// Input Mapping Context: IMC_OnFoot

// 걷기 매핑
IA_Move:
  - Key: W
    Modifiers: [Swizzle YXZ]  // 전후 이동
  - Key: S
    Modifiers: [Swizzle YXZ, Negate]
  - Key: A
    Modifiers: [Negate]  // 좌우 이동
  - Key: D
    Modifiers: []

// 달리기 매핑 (Shift + 이동)
IA_Sprint:
  - Key: Left Shift
    Triggers: [Down]  // 누르고 있는 동안

// 코드에서
void AMyCharacter::Move(const FInputActionValue& Value)
{
    FVector2D Input = Value.Get();

    float Speed = bIsSprinting ? SprintSpeed : WalkSpeed;
    AddMovementInput(GetActorForwardVector(), Input.Y * Speed);
    AddMovementInput(GetActorRightVector(), Input.X * Speed);
}

void AMyCharacter::Sprint(const FInputActionValue& Value)
{
    bIsSprinting = Value.Get();
}

📌 핵심 정리

  • Input Action은 추상적인 행동, IMC는 물리 키와의 연결입니다
  • IMC를 런타임에 추가/제거해서 상황별 입력을 구현합니다
  • Modifier로 입력값을 변환하고, Trigger로 발동 조건을 정의합니다
  • 우선순위가 높은 IMC가 입력을 먼저 처리합니다
  • Chorded Trigger로 Shift+Key 같은 조합 입력을 구현합니다
