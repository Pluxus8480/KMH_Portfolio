World Partition과 레벨 스트리밍

Part 1: 왜 World Partition이 필요한가?

기존 레벨 스트리밍 방식의 문제점은 무엇인가요?

기존 방식에서는 레벨 디자이너가 수동으로 레벨을 분할하고 스트리밍 볼륨을 배치해야 했습니다. 오픈 월드처럼 거대한 맵에서는 관리가 매우 어려웠죠.

또한 여러 사람이 같은 레벨을 편집하면 충돌이 발생했습니다.

🎯 비유로 이해하기

기존 방식은 거대한 퍼즐을 수동으로 조각내는 것입니다.

World Partition은 자동으로 그리드를 그려서 필요한 조각만 보여주는 것입니다.

Part 2: World Partition 개요

World Partition은 어떻게 동작하나요?

World Partition은 월드를 자동으로 그리드 셀로 분할합니다. 각 셀은 독립적으로 로드/언로드되며, 플레이어 근처 셀만 메모리에 유지됩니다.

Part 3: One File Per Actor (OFPA)

여러 사람이 같은 맵을 어떻게 동시에 편집하나요?

OFPA는 각 Actor를 별도의 파일로 저장합니다. 서로 다른 Actor를 편집하면 충돌 없이 동시 작업이 가능합니다.

OFPA 파일 구조
// 기존 방식
Maps/
  MyLevel.umap          // 모든 Actor가 한 파일에

// World Partition + OFPA
Maps/
  MyLevel.umap          // 레벨 설정만
  __ExternalActors__/
    MyLevel/
      A/
        Actor_001.uasset
        Actor_002.uasset
      B/
        Actor_003.uasset
        ...

// 장점
• Actor별 독립적 버전 관리
• 충돌 최소화
• 필요한 Actor만 로드 (에디터 성능 향상)

Part 4: Data Layers

런타임에 콘텐츠를 선택적으로 로드하려면?

Data Layers를 사용합니다. 같은 위치에 여러 버전의 콘텐츠를 두고 조건에 따라 다른 것을 로드합니다.

Data Layers 예시: 시간대별 마을

┌─────────────────────────────────────────────────┐
│                같은 위치                        │
├─────────────────────────────────────────────────┤
│                                                  │
│  Layer: Daytime (낮)                            │
│  ┌─────────────────────────────────┐           │
│  │  🌳 나무들                       │           │
│  │  🏠 밝은 집                      │           │
│  │  👥 NPC들이 돌아다님             │           │
│  └─────────────────────────────────┘           │
│                                                  │
│  Layer: Nighttime (밤)                          │
│  ┌─────────────────────────────────┐           │
│  │  🌙 어두운 조명                  │           │
│  │  🏚️ 창문에 불빛                  │           │
│  │  🦇 밤 NPC들                     │           │
│  └─────────────────────────────────┘           │
│                                                  │
│  게임 시간에 따라 Layer 전환                    │
│                                                  │
└─────────────────────────────────────────────────┘

Data Layer로 낮/밤 전환

Data Layer 제어
// Blueprint에서 Data Layer 활성화
void AMyGameMode::SwitchToNight()
{
    UDataLayerSubsystem* DataLayerSubsystem =
        GetWorld()->GetSubsystem();

    // 낮 레이어 비활성화
    DataLayerSubsystem->SetDataLayerRuntimeState(
        DaytimeLayer,
        EDataLayerRuntimeState::Unloaded);

    // 밤 레이어 활성화
    DataLayerSubsystem->SetDataLayerRuntimeState(
        NighttimeLayer,
        EDataLayerRuntimeState::Activated);
}

Part 5: HLOD (Hierarchical LOD)

먼 거리의 오브젝트는 어떻게 최적화하나요?

HLOD는 먼 거리의 여러 메시를 하나의 단순화된 메시로 병합합니다. 드로우콜을 크게 줄일 수 있습니다.

HLOD 동작 방식:

가까운 거리: 개별 메시 렌더링
┌─────────────────────────────────────┐
│   🏠  🏠  🏠  🌳  🌳  🏠  🏠       │
│   각각 별도 드로우콜 (7 calls)     │
└─────────────────────────────────────┘
                │
                │ 거리 증가
                ▼
먼 거리: HLOD 프록시 메시
┌─────────────────────────────────────┐
│   ┌─────────────────────────┐       │
│   │  ████████████████████  │       │
│   │   (병합된 프록시 메시)  │       │
│   └─────────────────────────┘       │
│   1 드로우콜                        │
└─────────────────────────────────────┘

HLOD로 원거리 최적화

World Partition HLOD 설정
// World Settings에서 HLOD 설정
World Partition:
  ☑ Enable HLOD
  HLOD Layer: Default HLOD Layer

// HLOD Layer 설정
Layer Type: Instancing  // 또는 MeshMerge, MeshSimplify
Cell Size: 25600        // HLOD 셀 크기
Loading Range: 50000    // HLOD 활성화 거리

// HLOD 빌드
World Partition → Build HLOD

// 결과
• 원거리: HLOD 프록시 렌더링
• 근거리: 원본 메시로 전환
• 전환은 자동으로 처리됨

Part 6: 스트리밍 소스 설정

무엇을 기준으로 셀이 로드되나요?

Streaming Source가 스트리밍의 기준점입니다. 기본적으로 플레이어 카메라지만, 커스터마이징 가능합니다.

커스텀 스트리밍 소스
// 플레이어 외에 추가 스트리밍 소스
// 예: 원거리 스나이퍼 스코프

UCLASS()
class AMyStreamingSource : public AActor
{
    GENERATED_BODY()

public:
    AMyStreamingSource()
    {
        // 스트리밍 소스 컴포넌트 추가
        StreamingSourceComponent =
            CreateDefaultSubobject(
                TEXT("StreamingSource"));

        StreamingSourceComponent->bStreamingSourceEnabled = true;
        StreamingSourceComponent->TargetGrids.Add(FName("MainGrid"));
    }
};

// 스나이퍼 스코프 조준 시
void AMyCharacter::AimSniper()
{
    // 조준점에 스트리밍 소스 생성
    // → 조준 위치의 셀도 미리 로드
    StreamingSource->SetActorLocation(AimLocation);
    StreamingSource->StreamingSourceComponent->
        bStreamingSourceEnabled = true;
}

📌 핵심 정리

  • World Partition은 월드를 그리드 셀로 자동 분할합니다
  • OFPA로 Actor별 파일 저장 → 팀 협업 용이
  • Data Layers로 조건부 콘텐츠 로드 (낮/밤, 퀘스트 전/후)
  • HLOD로 원거리 메시를 병합해 드로우콜 감소
  • Streaming Source로 로드 기준점 커스터마이징
