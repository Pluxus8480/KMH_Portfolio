Material Editor 심화

Part 1: Material의 본질

Material이란 정확히 무엇인가요?

Material은 표면의 시각적 특성을 정의하는 셰이더입니다. "이 표면은 금속인가? 거칠기는? 어떤 색인가?"를 GPU에게 알려줍니다.

언리얼의 Material Editor에서 만드는 노드 그래프는 결국 HLSL 셰이더 코드로 컴파일됩니다.

🎯 비유로 이해하기

Material은 물체의 피부와 같습니다.

피부(Material)가 어떤지에 따라 같은 조명 아래서도 금속처럼 보이거나, 플라스틱처럼 보이거나, 피부처럼 보입니다.

Part 2: PBR Material 속성

Material의 주요 입력값은 무엇인가요?

언리얼은 PBR(Physically Based Rendering) 워크플로우를 사용합니다. 현실 물리 법칙에 기반한 재질 표현입니다.

속성 설명 값 범위
Base Color 표면 기본 색상 (알베도) RGB 0~1
Metallic 금속성 (0=비금속, 1=금속) 0 또는 1
Roughness 표면 거칠기 0(매끄러움)~1(거침)
Normal 표면 미세 요철 (법선 맵) RGB(-1~1)
Emissive 자체 발광 색상 RGB, HDR 가능
Opacity 투명도 (Translucent만) 0(투명)~1(불투명)

Part 3: 자주 쓰는 노드

어떤 노드들을 많이 사용하나요?

Material Editor에는 수백 개의 노드가 있지만, 자주 쓰는 것들은 한정적입니다:

노드 단축키 용도
Texture Sample T 텍스처 샘플링
Multiply M 값 곱하기
Add A 값 더하기
Lerp L 선형 보간
Constant 1,2,3,4 상수 값
Parameter S, V 외부 파라미터
TexCoord U UV 좌표
Panner - UV 스크롤
Lerp를 이용한 텍스처 블렌딩
// 두 텍스처를 마스크로 블렌딩

TextureA ───┐
            ├── Lerp ──→ Base Color
TextureB ───┤
            │
Mask ───────┘ (Alpha)

// Lerp(A, B, Alpha) = A * (1-Alpha) + B * Alpha
// Alpha가 0이면 A, 1이면 B, 0.5면 절반씩

Part 4: Material Instance

왜 Material Instance를 사용하나요?

Material을 수정하면 셰이더를 다시 컴파일해야 합니다. 수 분이 걸릴 수 있죠. Material Instance는 파라미터 값만 바꾸므로 즉시 반영됩니다.

M_Character (Parent Material)
Parameters: BaseColor, RoughnessValue, NormalIntensity
↓ 상속
MI_Char_Red
BaseColor: (1, 0, 0)
MI_Char_Blue
BaseColor: (0, 0, 1)
MI_Char_Gold
BaseColor: (1, 0.8, 0)
Instance는 파라미터만 다름 → 셰이더 공유!

Parent Material과 Instance 관계

C++에서 Material Instance 파라미터 변경
// Dynamic Material Instance 생성
UMaterialInstanceDynamic* DynMaterial =
    UMaterialInstanceDynamic::Create(BaseMaterial, this);

// 메시에 적용
MeshComponent->SetMaterial(0, DynMaterial);

// 파라미터 변경 (실시간)
DynMaterial->SetVectorParameterValue(
    FName("BaseColor"), FLinearColor::Red);
DynMaterial->SetScalarParameterValue(
    FName("Roughness"), 0.3f);

Part 5: 고급 기법

자주 사용되는 고급 기법은 무엇인가요?

몇 가지 유용한 Material 기법을 소개합니다:

1. 디졸브 효과 (Dissolve)
// Opacity Mask에 연결
NoiseTexture ─── Subtract ─── Opacity Mask
                    │
DissolveAmount ────┘ (0~1 파라미터)

// DissolveAmount가 증가하면 노이즈의
// 낮은 부분부터 점점 사라짐
2. 프레넬 효과 (Rim Light)
// 가장자리 빛나기
Fresnel ───┐
           ├── Multiply ─── Add ─── Emissive
RimColor ──┘              │
                          │
BaseEmissive ─────────────┘

// Fresnel: 카메라와 표면이 수직일수록 1에 가까움
// → 오브젝트 가장자리가 빛나는 효과
3. World Aligned 텍스처
// UV 대신 월드 좌표로 텍스처 매핑
WorldPosition ─── Divide ─── TextureSample
                    │
TextureScale ──────┘

// 복잡한 지형에서 이음새 없는 텍스처링
// Tri-planar mapping에 활용

Part 6: 최적화 팁

Material 성능 최적화는 어떻게 하나요?

셰이더 복잡도가 높으면 GPU 성능에 직접적인 영향을 미칩니다:

최적화 기법 효과
텍스처 채널 패킹 R=AO, G=Roughness, B=Metallic으로 텍스처 수 감소
불필요한 노드 제거 Instruction 수 감소
Static Switch 사용 런타임 분기 대신 컴파일타임 분기
Quality Switch 저사양에서 간단한 버전 사용
Material LOD 거리에 따라 다른 Material
Shader Complexity 확인
// 에디터에서
View Mode → Optimization Viewmodes → Shader Complexity

// 녹색: 가벼움 (50~100 instructions)
// 노랑: 중간 (100~200)
// 빨강: 무거움 (200~300)
// 흰색: 매우 무거움 (300+)

// Material Editor에서 Stats 창 확인
Window → Stats
→ 각 셰이더 타입별 Instruction 수 표시

📌 핵심 정리

  • Material은 표면의 시각적 특성을 정의하는 셰이더입니다
  • Base Color, Metallic, Roughness, Normal이 PBR의 핵심 속성입니다
  • Material Instance를 사용하면 셰이더 재컴파일 없이 파라미터만 변경 가능
  • Dynamic Material Instance로 런타임에 머티리얼 속성 변경
  • Shader Complexity를 확인하고 텍스처 패킹, Quality Switch로 최적화
