문자열이 세 종류나 필요한 이유

언리얼 엔진의 FString, FName, FText - 상황별 올바른 문자열 클래스 선택

Part 1: 세 가지 문자열 클래스

왜 언리얼에는 문자열 클래스가 세 가지나 있나요?

각각의 문자열 클래스는 사용 목적과 성능 특성이 다릅니다. 상황에 맞는 클래스를 선택해야 최적의 성능을 낼 수 있습니다.

비유로 이해하기

FName은 "이름표" - 한번 붙이면 바꾸지 않고 빠르게 비교할 수 있습니다.

FString은 "메모장" - 자유롭게 쓰고 지우고 수정할 수 있지만 무겁습니다.

FText는 "번역가" - 다국어 지원을 위한 전문 도구입니다.

Part 2: FName

FName의 특징은?
  • 가벼움 - 주어진 문자열이 재사용되어도 데이터 테이블에 한 번만 저장
  • 대소문자 구별 없음 (Case-Insensitive)
  • 한번 지정 후 변경 불가능 (Immutable)

이런 특징들로 인해 FName을 키(Key)로 지정해 찾는 연산에 많이 사용됩니다. 에셋 이름, 테이블 검색 등에 적합합니다.

C++
// FName 사용 예시
FName AssetName = FName(TEXT("MyAsset"));
FName SameName = FName(TEXT("myasset")); // 대소문자 구별 없음

// 빠른 비교 가능 (내부적으로 인덱스 비교)
if (AssetName == SameName) // true

Part 3: FString

FString은 어떤 경우에 쓰나요?

다른 두 클래스와 달리 여러 조작이 가능한 문자열 클래스입니다.

  • 검색, 텍스트 조작, SubString 추출 등 다양한 기능 제공
  • 문자열 합치기, 수정, 포맷팅 가능
  • 다른 두 클래스보다 상대적으로 무거움

C++의 std::string과 가장 유사한 역할을 합니다. 문자열을 자유롭게 조작해야 할 때 사용합니다.

C++
// FString 사용 예시
FString Greeting = TEXT("Hello");
Greeting += TEXT(" World"); // 문자열 합치기 가능
FString Sub = Greeting.Mid(0, 5); // SubString 추출
int32 Idx = Greeting.Find(TEXT("World")); // 검색

Part 4: FText

FText는 왜 따로 있나요?

로컬라이징(다국어 지원)을 위한 문자열 클래스입니다.

UMG 등 여러 가지 유저들에게 보이는 텍스트들은 이 클래스를 사용합니다.

UI에 표시되는 모든 텍스트는 FText를 사용해야 합니다. 이렇게 해야 나중에 다국어 번역이 가능해집니다.

Part 5: 비교 정리

세 가지를 한눈에 비교해주세요.
구분 FName FString FText
용도 에셋 이름, 키 검색 문자열 조작 UI 표시, 로컬라이징
수정 가능 X (Immutable) O (Mutable) X (Immutable)
대소문자 구별 안함 구별함 구별함
성능 가벼움 (해시 비교) 무거움 중간
다국어 지원 X X O

핵심 정리

  • FName: 가볍고, 대소문자 구별 없으며, 변경 불가. 에셋 이름이나 키 검색에 사용
  • FString: 자유로운 문자열 조작이 가능하지만 상대적으로 무거움. std::string과 유사
  • FText: 로컬라이징을 위한 클래스. UI에 표시되는 텍스트에 사용
  • 상황에 맞는 문자열 클래스를 선택하는 것이 성능 최적화의 기본이다
