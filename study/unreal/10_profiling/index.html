Part 2: 언리얼 프로파일링 도구

어떤 도구들을 사용할 수 있나요?

언리얼은 강력한 내장 프로파일링 도구를 제공합니다:

도구 용도 실행 방법
Unreal Insights 종합 프로파일링 -trace=cpu,frame,bookmark
stat 명령어 실시간 통계 콘솔에서 stat xxx
GPU Visualizer GPU 병목 분석 Ctrl+Shift+, 또는 ProfileGPU
Session Frontend 원격 프로파일링 Window → Developer Tools
유용한 stat 명령어
// 기본 프레임 통계
stat fps           // FPS 표시
stat unit          // Game/Draw/GPU 시간 분리
stat unitgraph     // 그래프로 시각화

// 상세 분석
stat game          // 게임 스레드 상세
stat gpu           // GPU 렌더링 상세
stat scenerendering // 렌더 패스별 시간
stat rhi           // RHI 레벨 통계

// 특정 시스템
stat physics       // 물리 엔진
stat anim          // 애니메이션
stat ai            // AI 시스템
stat particles     // 파티클 시스템
stat ScriptTime    // Blueprint 실행 시간

// 메모리
stat memory        // 메모리 사용량
stat streaming     // 스트리밍 상태

Part 3: CPU vs GPU 병목 구분

CPU와 GPU 중 어디가 병목인지 어떻게 알 수 있나요?

stat unit 명령어로 확인합니다. Game, Draw, GPU 중 가장 큰 값이 병목입니다.

stat unit 결과 해석:

Frame: 33.3ms (30 fps)
├── Game:  5.0ms  ← CPU 게임 로직 (Tick, Physics 등)
├── Draw: 10.0ms  ← CPU 렌더 스레드 (컬링, 드로우콜)
└── GPU:  20.0ms  ← GPU 렌더링 시간

이 경우: GPU가 병목! (20ms로 가장 큼)

┌──────────────────────────────────────────────────────┐
│                    병목 판단                          │
├──────────────────────────────────────────────────────┤
│ Game > Draw, GPU     → CPU 게임로직 병목             │
│ Draw > Game, GPU     → CPU 렌더스레드 병목 (드로우콜)│
│ GPU > Game, Draw     → GPU 병목 (셰이더, 폴리곤)    │
└──────────────────────────────────────────────────────┘

Part 4: CPU 최적화

CPU 병목을 해결하는 방법은?

병목 원인에 따라 다른 접근이 필요합니다:

원인 증상 해결책
Tick 과부하 Game 시간 높음 Tick 간격 조절, Timer 사용
Blueprint 과다 ScriptTime 높음 C++로 이식, 루프 최적화
Physics 과부하 stat physics 높음 콜리전 단순화, 서브스텝 조절
드로우콜 과다 Draw 시간 높음 인스턴싱, 메시 병합
Tick 최적화 기법
// ❌ 나쁜 예: 모든 Actor가 매 프레임 Tick
void AMyActor::Tick(float DeltaTime)
{
    // 멀리 있어도 항상 실행
    DoExpensiveCalculation();
}

// ✅ 좋은 예 1: Tick 비활성화
AMyActor::AMyActor()
{
    PrimaryActorTick.bCanEverTick = false;  // Tick 사용 안함
}

// ✅ 좋은 예 2: Tick 간격 조절
AMyActor::AMyActor()
{
    PrimaryActorTick.TickInterval = 0.1f;  // 0.1초마다
}

// ✅ 좋은 예 3: Timer 사용
void AMyActor::BeginPlay()
{
    GetWorld()->GetTimerManager().SetTimer(
        TimerHandle,
        this,
        &AMyActor::DoPeriodicWork,
        1.0f,  // 1초마다
        true   // 반복
    );
}

// ✅ 좋은 예 4: 거리 기반 Tick
void AMyActor::Tick(float DeltaTime)
{
    float Distance = GetDistanceToPlayer();
    if (Distance > 5000.0f)
    {
        // 멀면 저빈도 업데이트
        SetActorTickInterval(0.5f);
        return;
    }
    SetActorTickInterval(0.0f);  // 가까우면 매 프레임
    DoWork();
}

Part 5: GPU 최적화

GPU 병목을 해결하는 방법은?

GPU 병목은 주로 폴리곤 수, 셰이더 복잡도, 텍스처 대역폭에서 발생합니다.

GPU Visualizer 분석
// ProfileGPU 또는 Ctrl+Shift+, 실행

주요 항목:
├── BasePass:       8.5ms  ← 기본 지오메트리 렌더링
├── Lighting:       4.0ms  ← 라이팅 계산
├── Translucency:   2.0ms  ← 반투명 오브젝트
├── PostProcessing: 3.0ms  ← 후처리 효과
└── UI:             0.5ms  ← UI 렌더링

병목 판단:
• BasePass 높음 → 폴리곤/드로우콜 문제
• Lighting 높음 → 광원 수/그림자 문제
• PostProcessing 높음 → 후처리 효과 과다
최적화 기법 효과 적용 방법
LOD 시스템 먼 거리 폴리곤 감소 메시에 LOD 설정
오클루전 컬링 안 보이는 것 스킵 Precomputed Visibility
텍스처 스트리밍 VRAM 절약 Texture Streaming Pool
HLOD 원거리 메시 병합 World Settings
인스턴싱 드로우콜 감소 Instanced Static Mesh

Part 6: Unreal Insights 활용

Unreal Insights는 어떻게 사용하나요?

Unreal Insights는 가장 강력한 프로파일링 도구입니다. 트레이스 파일을 기록하고 상세 분석할 수 있습니다.

Insights 실행 방법
// 방법 1: 에디터에서
Window → Developer Tools → Session Frontend → Profiler

// 방법 2: 실행 인자로 트레이스
MyGame.exe -trace=cpu,frame,bookmark,file

// 방법 3: 콘솔 명령어
trace.start cpu,gpu,frame,bookmark
// ... 게임 플레이 ...
trace.stop

// Insights 뷰어 실행
UnrealInsights.exe [trace파일경로]
Unreal Insights 분석 화면:

Timeline View
