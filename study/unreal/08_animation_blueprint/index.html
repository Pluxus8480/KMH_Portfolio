Animation Blueprint와 상태 머신

Part 1: Animation Blueprint란?

Animation Blueprint가 필요한 이유는 무엇인가요?

게임 캐릭터는 수십~수백 개의 애니메이션을 상황에 맞게 재생해야 합니다. 걷기, 뛰기, 점프, 공격, 피격, 사망...

Animation Blueprint(ABP)는 이 복잡한 애니메이션 로직을 시각적으로 관리할 수 있게 해줍니다.

🎯 비유로 이해하기

Animation Blueprint는 교통 관제 시스템과 같습니다.

수많은 애니메이션(차량)이 있고, 현재 상태(교통 상황)에 따라 어떤 애니메이션을 재생할지(어떤 길로 보낼지) 결정합니다.

Part 3: State Machine (상태 머신)

상태 머신은 어떻게 동작하나요?

상태 머신은 상태(State)와 전이(Transition)로 구성됩니다. 캐릭터의 현재 상태에 따라 다른 애니메이션을 재생합니다.

Locomotion State Machine
Idle
Speed > 0 → ← Speed == 0
Walk
Jump Pressed ▼
Jump
IsInAir == false → Walk

간단한 상태 머신 예시

상태 전이 조건
// Idle → Walk 전이 조건
// Transition Rule (Blueprint)
Return Speed > 0.0f;

// Walk → Idle 전이 조건
Return Speed <= 0.0f;

// Any State → Jump 전이 조건
Return bIsJumping && !bIsFalling;

// Jump → 이전 상태 전이 조건
Return !bIsInAir && bIsFalling == false;

Part 4: Blend Spaces

방향에 따라 다른 이동 애니메이션을 어떻게 재생하나요?

Blend Space를 사용합니다. 1D는 단일 축(속도), 2D는 두 축(속도 + 방향)으로 여러 애니메이션을 블렌딩합니다.

Blend Space 2D: Locomotion (X = Direction, Y = Speed)
Speed \ Dir -180 0 180
400 BwdRun FwdRun FwdRun
200 BwdWalk FwdWalk FwdWalk
0 Idle Idle Idle

2D Blend Space로 8방향 이동 구현

Blend Space 사용
// Event Graph에서 변수 계산
void UMyAnimInstance::NativeUpdateAnimation(float DeltaSeconds)
{
    Super::NativeUpdateAnimation(DeltaSeconds);

    if (APawn* Pawn = TryGetPawnOwner())
    {
        // 속도 계산
        FVector Velocity = Pawn->GetVelocity();
        Speed = Velocity.Size2D();

        // 방향 계산 (캐릭터 기준)
        FRotator Rotation = Pawn->GetActorRotation();
        Direction = CalculateDirection(Velocity, Rotation);
    }
}

Part 5: Animation Montage

공격이나 스킬 같은 일회성 애니메이션은 어떻게 재생하나요?

Animation Montage를 사용합니다. 몽타주는 기본 애니메이션 위에 오버레이되어 재생됩니다.

Montage 재생
// 캐릭터 코드에서 몽타주 재생
void AMyCharacter::Attack()
{
    if (UAnimInstance* AnimInstance = GetMesh()->GetAnimInstance())
    {
        // 몽타주 재생
        AnimInstance->Montage_Play(AttackMontage, 1.0f);

        // 특정 섹션으로 점프
        AnimInstance->Montage_JumpToSection(FName("Combo2"), AttackMontage);
    }
}

// 몽타주 끝났을 때 콜백
AnimInstance->OnMontageEnded.AddDynamic(
    this, &AMyCharacter::OnAttackMontageEnded);

void AMyCharacter::OnAttackMontageEnded(
    UAnimMontage* Montage, bool bInterrupted)
{
    if (Montage == AttackMontage)
    {
        bIsAttacking = false;
    }
}
Montage에서 이벤트를 발생시키려면?

Anim Notify를 사용합니다. 몽타주 타임라인에 노티파이를 배치하면 해당 시점에 이벤트가 발생합니다.

Notify 유형 용도
AnimNotify 단일 시점 이벤트 (사운드, 히트 판정)
AnimNotifyState 구간 이벤트 (파티클, 무적 시간)

Part 6: Animation Layer와 Slot

상체는 공격, 하체는 이동 애니메이션을 동시에 재생하려면?

Animation Slot과 Layered Blend per Bone을 사용합니다. 신체 부위별로 다른 애니메이션을 블렌딩할 수 있습니다.

Base Pose (전신)

Locomotion (State Machine) - 걷기/뛰기 애니메이션

▼
Layered Blend per Bone

  • 상체: Slot Anim (공격 몽타주)
▼
Final Pose

📌 핵심 정리

  • Event Graph에서 데이터 수집, Anim Graph에서 포즈 계산
  • State Machine으로 상태별 애니메이션 관리 (Idle, Walk, Jump 등)
  • Blend Space로 속도/방향에 따른 부드러운 블렌딩
  • Montage로 일회성 애니메이션 (공격, 스킬) 재생
  • Layered Blend로 부위별 다른 애니메이션 합성
