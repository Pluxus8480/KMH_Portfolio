무기를 바꿔도 공격 코드는 그대로

전략 패턴(Strategy Pattern)으로 유연한 행동 교체하기

Part 1: 전략 패턴이란?

전략 패턴의 정의는?

전략 패턴은 인터페이스를 통해 특정 행동들을 묶고, 이를 런타임에 교환하면서 사용하는 패턴입니다.

🎯 비유로 이해하기

캐릭터가 "공격" 버튼을 누르면, 지금 들고 있는 무기에 따라 다른 공격이 나갑니다.

무기가 바뀌어도 "공격 버튼을 누른다"는 코드는 변하지 않습니다.

Part 2: 구현 예시

게임에서 전략 패턴을 어떻게 적용하나요?

캐릭터의 무기가 총과 칼이 있다고 할 때:

C++
// IWeapon.h - 전략 인터페이스
class IWeapon
{
public:
    virtual void Attack() = 0;
};

// Player.h - 전략을 사용하는 컨텍스트
class Player
{
    IWeapon* Weapon;

    // 공격 키가 눌리면
    void OnAttack() {
        Weapon->Attack();  // 어떤 무기든 동일한 호출
    }
};

// 구체적인 전략들
class Sword : public IWeapon
{
    void Attack() override
    {
        // 검을 휘두른다
    }
};

class Gun : public IWeapon
{
    void Attack() override
    {
        // 총을 쏜다
    }
};

플레이어가 무기를 변경해도 IWeapon 인터페이스만 알면 되므로, 플레이어 코드 변경 없이 무기 교체가 가능합니다.

Part 3: 핵심 가치

전략 패턴의 핵심 가치는 무엇인가요?

구체적인 클래스와 알고리즘 사이의 결합도를 낮추는 디자인 패턴입니다.

구체 클래스를 수정하는 일이 줄어들어 유지보수에 큰 도움이 됩니다.

면접에서는 정의를 바로 말하기보다, 본인의 프로젝트에서 전략 패턴을 활용한 경험을 예시로 드는 것이 좋습니다.

📌 핵심 정리

  • 전략 패턴은 인터페이스로 행동을 추상화하고 런타임에 교체하는 패턴
  • 구체 클래스와 알고리즘 간 결합도를 낮춤
  • 새로운 전략(무기, 행동 등) 추가 시 기존 코드 수정 불필요
  • 면접에서는 실제 프로젝트 적용 경험을 예시로 드는 것이 효과적
