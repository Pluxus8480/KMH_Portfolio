화장실 열쇠는 하나, 대기줄은 길다

뮤텍스(Mutex)와 세마포어(Semaphore)의 동기화 메커니즘

Part 1: 임계 구역과 동기화

임계 구역(Critical Section)이란?

임계 구역은 공유된 자원에 접근하는 코드 영역입니다. 여러 스레드/프로세스가 동시에 접근하면 데이터 무결성이 깨질 수 있습니다.

뮤텍스와 세마포어는 모두 이 임계 구역에 여러 스레드/프로세스가 한 번에 접근하는 것을 막는 방식입니다.

Part 2: 뮤텍스 vs 세마포어

뮤텍스와 세마포어의 가장 큰 차이는?

뮤텍스 객체는 한 개의 스레드/프로세스만 가질 수 있고, 세마포어는 카운터 변수를 공유하면서 관리합니다.

단계 Mutex Semaphore
접근 한 스레드가 뮤텍스 객체에 접근 스레드가 접근하면 카운터 -1
잠금 뮤텍스 객체를 Lock 카운터가 0이 되면 Lock
대기 Lock된 객체는 다른 스레드 접근 불가 카운터 0일 때 다른 스레드 접근 불가
해제 사용 완료 시 Unlock 사용 완료 시 카운터 +1
🎯 비유로 이해하기

Mutex: 화장실 열쇠가 1개 - 한 사람만 사용 가능, 나올 때 열쇠 반납

Semaphore: 놀이공원 놀이기구 좌석이 N개 - N명까지 탑승 가능, 좌석이 다 차면 대기

뮤텍스는 세마포어의 특수한 경우인가요?

네, 뮤텍스는 사실상 Binary Semaphore(카운터가 0 또는 1)라고 볼 수 있습니다. 카운터가 1인 세마포어와 동작이 같습니다.

📌 핵심 정리

  • Mutex: 한 번에 하나의 스레드만 접근 가능 (Lock/Unlock)
  • Semaphore: 카운터 기반으로 N개 스레드까지 접근 가능
  • Mutex는 Binary Semaphore(카운터 1)의 특수한 경우
  • 둘 다 임계 구역(Critical Section)의 동시 접근을 방지하는 동기화 도구
