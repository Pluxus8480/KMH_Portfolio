스테이트 패턴(State Pattern)과 FSM 구현

Part 1: 스테이트 패턴이란?

스테이트 패턴은 왜 필요한가요?

한 번에 한 상태만 가질 수 있는 FSM(유한 상태 기계)을 구현할 때 많이 사용하는 패턴입니다.

중첩 switch/case 문으로도 구현할 수 있지만, 상태가 많아질수록 확장성이 떨어지기 때문에 스테이트 패턴으로 구현합니다.

🎯 비유로 이해하기

좀비를 예로 들면 - 좀비는 순찰, 추격, 공격 상태를 가집니다.

각 상태에서 할 수 있는 행동이 다르고, 조건에 따라 상태가 전환됩니다.

어떻게 구현하나요?

상위 인터페이스(State)에 관련 기능들의 함수를 준비하고, 파생 State에서 해당 상태에서 할 수 있는 기능을 구현합니다.

IState 인터페이스
Enter() / Update() / Exit()

순찰 로직
↔

Part 2: 전략 패턴과의 차이

스테이트 패턴과 전략 패턴은 비슷해 보이는데, 차이점은?
구분 스테이트 패턴 전략 패턴
유래 조건문(switch/case)을 대체 상속을 대체
Context 참조 파생 클래스가 Context를 참조 Context 참조 없음
상태 전환 State가 스스로 다음 상태를 결정 외부에서 전략을 교체

스테이트 패턴은 전부 전략 패턴이 될 수 있지만, 전략 패턴이 전부 스테이트 패턴이 될 수는 없습니다.

📌 핵심 정리

  • 스테이트 패턴은 FSM 구현에 사용되는 패턴, switch/case의 확장성 문제를 해결
  • 상위 인터페이스(State)를 두고, 각 상태별 파생 클래스로 구현
  • 전략 패턴과 유사하지만, State가 Context를 참조하고 스스로 상태 전환 가능
  • 스테이트 패턴은 전략 패턴의 상위 호환이 아님 - 용도와 관계가 다름
