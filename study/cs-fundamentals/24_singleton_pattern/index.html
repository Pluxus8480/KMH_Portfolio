싱글톤 패턴(Singleton Pattern)의 구현과 멀티스레드 이슈

Part 1: 싱글톤 패턴이란?

싱글톤 패턴의 정의는?

싱글톤 패턴(Singleton Pattern)은 클래스의 인스턴스를 한 개만 가지게 하는 디자인 패턴입니다.

C#(유니티)에서는 어떻게 구현하나요?
C# (Unity)
public SingletonClass : MonoBehaviour
{
    public static SingletonClass instance;

    private void Awake()
    {
        if (instance == null)
        {
            instance = this;
        }
        else
        {
            Destroy(this);
        }
    }
}
C++에서는 어떻게 구현하나요?

생성자를 private에 넣고, 대입/복사 관련 연산을 삭제합니다.

C++
class SingletonClass
{
public:
    static Singleton& GetInstance() {
        static Singleton* singleton = new Singleton();
        return *singleton;
    }

private:
    SingletonClass() = default;

    SingletonClass(const SingletonClass&) = delete;
    SingletonClass& operator=(const SingletonClass&) = delete;
    SingletonClass(SingletonClass&&) = delete;
    SingletonClass& operator=(SingletonClass&&) = delete;
};

Part 2: 장점과 사용 이유

왜 싱글톤 패턴을 사용하나요?
  • 프로그램의 생명주기와 동일하게 메모리에 올라가고 사라짐
  • 메모리에 한 번만 올라가므로 여러 인스턴스 생성에 의한 메모리 낭비 방지
  • 전역으로 올라가기 때문에 다른 인스턴스들에서 자유롭게 접근 가능

Part 3: 단점과 주의사항

싱글톤 패턴의 문제점은?
  • 사용하지 않을 때도 계속 메모리를 점유
  • C++에서 소멸이 정의되지 않아 소멸된 객체의 역참조 문제 발생 가능
  • 인스턴스가 null인지 항상 체크해야 하는 비효율성

가장 큰 문제는 동시성 문제입니다. 멀티스레드 환경에서 인스턴스가 2개 생기는 경우도 있고, 여러 스레드에서 참조할 때 동기화 처리를 해줘야 하는 번거로움이 있습니다.

면접에서 싱글톤을 언급하면 "멀티스레드 환경에서 어떻게 활용했나요?"라는 질문이 높은 확률로 따라옵니다. 자신 있게 답할 수 없다면, 다른 디자인 패턴으로 질문을 유도하는 것이 좋습니다.

📌 핵심 정리

  • 싱글톤은 클래스 인스턴스를 하나만 생성하는 패턴
  • 전역 접근이 가능하고 메모리 효율적이지만, 사용하지 않을 때도 메모리 점유
  • C++에서는 생성자 private + 복사/대입 삭제로 구현
  • 멀티스레드 환경에서 동기화 처리가 핵심 이슈
  • 면접 시 멀티스레드 관련 경험 질문에 대비 필수
