누구를 먼저 실행시킬 것인가?

CPU 스케줄러 알고리즘의 종류와 특징

Part 1: FCFS - 먼저 온 순서대로

FCFS(First Come First Served)란?

먼저 온 프로세스를 먼저 처리하는 가장 단순한 알고리즘입니다.

Non-preemptive 방식이므로 현재 진행 중인 프로세스를 중간에 종료할 수 없습니다.

짧은 작업이 긴 작업 뒤에 오면 오래 기다려야 하는 Convoy Effect가 발생할 수 있습니다.

Part 2: SJF - 짧은 작업 우선

SJF(Shortest Job First)는 어떻게 동작하나요?

현재 시점에서 가장 짧은 작업을 먼저 처리합니다.

구분 Non-preemptive SJF Preemptive SJF (SRTF)
동작 현재 실행 중인 프로세스 유지 더 짧은 프로세스 도착 시 교체
프로세스의 실행 시간을 어떻게 미리 알 수 있나요?

정확히 알 수 없으므로 예측을 합니다.

예측 공식
T(n) = a * t(n-1) + (1-a) * T(n-1)

T(n) : N번째 CPU 사용 예측치
t(n-1) : N-1번째 실제 수행 시간
T(n-1) : N-1번째 예측 시간
a : 적응 속도 (0~1)
  - 0에 가까울수록 이전 예측에 가중치
  - 1에 가까울수록 실제 시간에 가중치

Part 3: Priority Scheduling

우선순위 스케줄링이란?

각 프로세스에 우선순위를 부여하고, 높은 우선순위의 프로세스를 먼저 실행합니다. SJF는 우선순위 스케줄링의 일종으로 "짧은 시간"에 우선순위를 부여한 것입니다.

기아(Starvation) 문제: 우선순위가 낮은 프로세스가 높은 프로세스에 계속 밀려 영원히 실행되지 못할 수 있습니다.

해결법 - Aging: 시간이 흐를수록 대기 중인 프로세스의 우선순위를 점진적으로 높여줍니다.

Part 4: Round Robin

Round Robin은 어떻게 동작하나요?

일정 시간(Time Quantum, q)을 정한 뒤, 이 시간이 지나면 프로세스 완료 여부와 관계없이 교체합니다.

N개의 프로세스가 있을 때, 각 프로세스는 최대 (N-1)q 만큼만 기다리면 됩니다.

P1 실행
시간 q만큼
→
P2 실행
시간 q만큼
→
P3 실행
시간 q만큼
→
다시 P1
순환 반복

Part 5: SRTF (Shortest Remaining Time First)

SRTF의 특징과 문제점은?

새로운 프로세스가 도착할 때마다 스케줄링을 다시 수행하는 선점형(Preemptive) 알고리즘입니다.

현재 수행 중인 프로세스의 남은 burst time보다 더 짧은 프로세스가 도착하면 CPU를 빼앗깁니다.

문제점: Starvation 발생 가능하며, 새 프로세스 도착 시마다 스케줄링하므로 CPU burst time 측정이 어렵습니다.

Part 6: 알고리즘 비교 총정리

전체 알고리즘을 한눈에 비교하면?
알고리즘 선점 여부 기아 문제 핵심 특징
FCFS Non-preemptive 없음 가장 단순, Convoy Effect
SJF 둘 다 가능 있음 짧은 작업 우선
Priority 둘 다 가능 있음 (Aging으로 해결) 우선순위 기반
RR Preemptive 없음 공정, Time Quantum
SRTF Preemptive 있음 남은 시간 최소 우선

📌 핵심 정리

  • FCFS: 선입선출, 단순하지만 Convoy Effect 문제
  • SJF: 짧은 작업 우선, 실행 시간 예측 필요
  • Priority: 우선순위 기반, Starvation은 Aging으로 해결
  • Round Robin: 공정한 시간 분배, Time Quantum이 핵심
  • SRTF: 선점형 SJF, 최적에 가까우나 오버헤드 큼
