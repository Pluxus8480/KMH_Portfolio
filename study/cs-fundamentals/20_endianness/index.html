바이트를 어디서부터 읽을 것인가?

리틀 앤디언(Little Endian)과 빅 앤디언(Big Endian)의 차이

Part 1: 앤디언이란?

앤디언(Endian)이 무엇인가요?

앤디언(엔디안)은 컴퓨터 메모리 같은 1차원 연속 배열에서 바이트를 어떤 순서로 배열하는지에 대한 방법입니다.

Part 2: 빅 앤디언 vs 리틀 앤디언

두 방식의 차이는 무엇인가요?
구분 빅 앤디언 (Big Endian) 리틀 앤디언 (Little Endian)
저장 순서 큰 단위 바이트부터 (사람 읽는 순서) 작은 단위 바이트부터
장점 사람이 읽기 편해 디버깅이 쉬움 아래 자리부터 보므로 사칙연산에 이점
사용처 네트워크 주소 (Network Byte Order) x86 프로세서 (대부분의 PC)

예를 들어 0x12345678을 메모리에 저장할 때:

메모리 저장 순서 비교
주소:      [0x00] [0x01] [0x02] [0x03]
빅 앤디언:    12     34     56     78   (사람 읽는 순서)
리틀 앤디언:  78     56     34     12   (뒤집어서 저장)

Part 3: 면접 포인트

면접에서 자주 나오는 질문은?

"x86 프로세서는 어떤 앤디언일까요?" - 리틀 앤디언입니다.

추가로 알아두면 좋은 부분:

  • ARM 아키텍처: 성능 향상을 위해 리틀/빅을 선택할 수 있음 (Bi-endian)
  • 네트워크 통신: 네트워크는 빅 앤디언(Network Byte Order)을 사용하므로, x86에서 네트워크 데이터를 보낼 때 바이트 순서 변환이 필요

게임 서버 개발 시 클라이언트-서버 간 데이터 전송에서 엔디언 차이를 고려해야 합니다. htonl(), ntohl() 등의 변환 함수를 사용합니다.

📌 핵심 정리

  • 앤디언은 바이트 배열 순서를 정의하는 방식
  • 빅 앤디언: 사람 읽는 순서, 네트워크 통신에 사용
  • 리틀 앤디언: 역순 저장, x86 프로세서에서 사용
  • ARM은 Bi-endian으로 선택 가능
  • 네트워크 프로그래밍 시 바이트 순서 변환 필수
