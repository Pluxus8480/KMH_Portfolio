메모리가 부족해도 프로그램이 돌아가는 마법

가상 메모리(Virtual Memory)와 페이지 교체 알고리즘

Part 1: 가상 메모리란?

가상 메모리가 왜 필요한가요?

가상 메모리 기법은 실제 메모리보다 메모리가 더 많아 보이게 하는 기법입니다.

프로그램의 현재 실행에 필요한 부분만 메모리에 올라가고, 나머지는 보조 기억장치(HDD, SSD 등)에 올려두고 사용합니다.

🎯 비유로 이해하기

책상이 작아서 모든 책을 올려놓을 수 없을 때, 지금 읽는 책만 책상에 올려놓고 나머지는 책장에 꽂아두는 것과 같습니다.

필요할 때 책장에서 꺼내오고, 안 쓰는 책은 다시 책장에 넣는 거죠.

MMU는 무엇인가요?

MMU(Memory Management Unit)는 논리 주소를 물리 주소로 변환해주는 장치입니다. 현대 CPU에는 모두 내장되어 있습니다.

Part 2: Page Fault와 Swap

Page Fault가 발생하면 어떻게 되나요?

필요한 페이지가 물리 메모리에 없는 상황을 Page Fault라고 합니다.

이 경우 디스크에서 페이지를 불러오고(Swap In), 필요 없는 페이지를 디스크로 보내는(Swap Out) 작업을 하게 됩니다.

Part 3: 페이지 교체 알고리즘

어떤 페이지를 교체해야 효율적인가요?

보조 기억장치는 메모리보다 느리기 때문에, Swap 횟수를 최소화하려면 사용 가능성이 높은 페이지는 메모리에, 낮은 페이지는 디스크에 두는 전략이 필요합니다.

알고리즘 방식 특징
FIFO 가장 먼저 들어온 페이지 교체 간단하지만 Belady's Anomaly 발생 가능
Optimal 가장 오래 사용하지 않을 페이지 교체 이상적이지만 미래 예측 불가 (성능 측정용)
LRU 가장 오래 사용하지 않은 페이지 교체 시간 기록 오버헤드 존재
LFU 가장 적게 사용한 페이지 교체 사용 빈도 기반
Belady's Anomaly가 뭔가요?

FIFO에서만 발생하는 현상으로, 페이지 프레임 수(사용 가능 메모리)가 커질수록 Page Fault가 오히려 늘어나는 역설적인 현상입니다.

Clock Algorithm(Second Chance)이 뭔가요?

LRU의 변형으로, 페이지별로 reference bit를 둡니다.

시계방향으로 순회하면서 bit가 1이면 0으로 바꾸고 넘어가고, 0이면 해당 페이지를 교체합니다.

Part 4: TLB (Translation Lookaside Buffer)

TLB는 어떤 역할을 하나요?

TLB는 최근에 사용된 페이지 테이블 엔트리를 캐싱하는 버퍼로, 주소 변환 속도를 높여줍니다.

주소를 찾을 때 먼저 TLB에 접근하고, TLB에 있으면(TLB Hit) 바로 물리 주소에 접근합니다. 없으면(TLB Miss) MMU를 통해 주소를 변환한 후 메모리에 접근합니다.

📌 핵심 정리

  • 가상 메모리는 필요한 부분만 물리 메모리에 올리고 나머지는 디스크에 두는 기법
  • Page Fault 발생 시 Swap In/Out을 통해 페이지를 교환
  • 페이지 교체 알고리즘: FIFO, Optimal, LRU, LFU - 각각 장단점이 다름
  • TLB는 주소 변환을 캐싱하여 메모리 접근 속도를 향상시킴
  • MMU가 논리 주소를 물리 주소로 변환하는 핵심 장치
