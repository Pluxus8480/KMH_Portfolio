메모리가 남는데 왜 할당이 안 될까?

메모리 단편화(Memory Fragmentation)의 원인과 해결법

Part 1: 메모리 단편화란?

메모리 단편화가 무엇인가요?

메모리 단편화는 사용 가능한 메모리는 충분하지만, 흩어져 있어서 할당이 불가능한 경우를 의미합니다.

종류는 외부 단편화와 내부 단편화 2가지가 있습니다.

내부 단편화와 외부 단편화의 차이는?
구분 내부 단편화 외부 단편화
정의 프로세스에 필요한 것보다 많은 메모리를 할당 전체 메모리는 충분하지만 연속 공간이 부족
원인 고정 크기 할당 할당/해제 반복
🎯 비유로 이해하기

내부 단편화: 1인실에 혼자 사는데 3인실을 배정받은 것 - 남는 공간이 낭비

외부 단편화: 주차장에 빈 자리가 여기저기 흩어져 있어서 대형 버스가 주차할 연속 공간이 없는 것

Part 2: 외부 단편화 해결법

외부 단편화는 어떻게 해결하나요?

1. 페이징(Paging)

보조 기억장치를 메모리처럼 구역을 나눈 뒤 필요한 부분을 메모리로 옮겨서 사용합니다.

페이지(Page)는 프로세스를 일정한 크기로 나눈 것을 의미합니다.

  • 첫 페이지를 우선 로드 후 이후 페이지를 차례로 로드
  • 프로세스가 같은 페이지를 사용한다면 공유하여 메모리 절약 가능
  • 페이지 테이블(PTBR)이 필요하여 시간/공간적 오버헤드 발생
  • 시간적 오버헤드는 TLB(Translation Lookaside Buffer)로 해결

2. 압축(Compaction)

삭제 공간들을 회수해 메모리를 정리하는 방식이지만, 비용이 많이 드는 작업입니다.

3. 통합(Coalescing)

인접한 빈 메모리 블록들을 합치는 작업으로, 압축보다 가벼운 방식입니다.

Part 3: 내부 단편화 해결법

내부 단편화는 어떻게 해결하나요?

세그멘테이션(Segmentation)

페이징이 일정 크기로 나누는 것이라면, 세그멘테이션은 논리 단위로 나누는 것입니다.

  • 논리 단위의 크기가 일정하지 않음
  • 논리 주소와 물리 주소를 매핑하는 테이블이 필요

세그멘테이션은 외부 단편화가 발생할 우려가 있고, 크기가 제멋대로이기 때문에 실무에서는 페이징을 더 많이 사용합니다.

Part 4: 둘 다 해결하는 방법 - 메모리 풀

내부/외부 단편화를 동시에 해결하는 방법이 있나요?

메모리 풀(Memory Pool)은 필요한 메모리를 미리 할당받아 두고, 필요할 때 가져다 쓰고 반납하는 기법입니다.

  • 풀링 없이 할당/해제를 반복하면 단편화 발생
  • 미리 할당받은 공간을 가져다 쓰고 반납하므로 외부 단편화 없음
  • "필요한 만큼" 가져다 쓰므로 내부 단편화도 없음

오브젝트 풀링과 비슷한 개념이며, 할당/해제가 빈번한 경우 매우 유용합니다. 게임 개발에서 총알, 이펙트 등에 자주 사용됩니다.

📌 핵심 정리

  • 내부 단편화: 필요보다 많은 메모리 할당으로 발생하는 낭비
  • 외부 단편화: 연속 공간 부족으로 발생하는 할당 실패
  • 외부 단편화 해결: 페이징, 압축, 통합
  • 내부 단편화 해결: 세그멘테이션 (그러나 외부 단편화 우려)
  • 메모리 풀은 내부/외부 단편화를 동시에 해결하는 효과적인 방법
