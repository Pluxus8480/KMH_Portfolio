물이 퍼지듯 영역을 채운다?

Flood Fill과 Flow Field를 활용한 경로 최적화

Part 1: Flood Fill이란?

Flood Fill은 어디에 사용되나요?

연결된 영역을 탐색하거나 채우는 알고리즘입니다.

  • 그래픽 에디터: 페인트 버킷 도구
  • 게임 맵: 이동 가능 영역 탐지
  • 미로 풀이: 도달 가능 영역 탐색
  • Flow Field: 대규모 유닛 길찾기
Flood Fill 예시
시작 전:
    █    
  ★ █    
    █    
→
Flood Fill 후:
● ● █ ○ ○
● ★ █ ○ ○
● ● █ ○ ○
★: 시작점   █: 벽   ●: 채워진 영역   벽으로 분리된 오른쪽(○)은 채워지지 않음
어떻게 구현하나요?

두 가지 방식:

  • 재귀 (DFS): 깊이 우선, 스택 오버플로우 위험
  • 큐 (BFS): 너비 우선, 물이 퍼지는 것처럼 동작

Part 2: Flood Fill 구현

BFS 기반 Flood Fill
#include <queue>
#include <vector>

class FloodFill {
private:
    int width, height;
    std::vector<int> grid;  // 0: 빈 공간, 1: 벽, 2+: 채워진 영역

    int dx[4] = {-1, 0, 1, 0};
    int dy[4] = {0, -1, 0, 1};

    bool isValid(int x, int y) const {
        return x >= 0 && x < width && y >= 0 && y < height;
    }

    int& cell(int x, int y) {
        return grid[y * width + x];
    }

public:
    FloodFill(int w, int h) : width(w), height(h), grid(w * h, 0) {}

    // BFS Flood Fill
    int fill(int startX, int startY, int fillValue) {
        if (!isValid(startX, startY)) return 0;
        if (cell(startX, startY) != 0) return 0;

        int filledCount = 0;
        std::queue<std::pair<int, int>> queue;
        queue.push({startX, startY});
        cell(startX, startY) = fillValue;

        while (!queue.empty()) {
            auto [x, y] = queue.front();
            queue.pop();
            filledCount++;

            for (int i = 0; i < 4; i++) {
                int nx = x + dx[i];
                int ny = y + dy[i];

                if (isValid(nx, ny) && cell(nx, ny) == 0) {
                    cell(nx, ny) = fillValue;
                    queue.push({nx, ny});
                }
            }
        }

        return filledCount;
    }

    // 스캔라인 최적화 (수평 연속 영역 한 번에 처리)
    int fillScanline(int startX, int startY, int fillValue) {
        if (!isValid(startX, startY)) return 0;
        if (cell(startX, startY) != 0) return 0;

        int filledCount = 0;
        std::queue<std::pair<int, int>> queue;
        queue.push({startX, startY});

        while (!queue.empty()) {
            auto [x, y] = queue.front();
            queue.pop();

            if (cell(x, y) != 0) continue;

            // 왼쪽 끝 찾기
            int left = x;
            while (left > 0 && cell(left - 1, y) == 0) left--;

            // 오른쪽으로 채우면서 위/아래 스캔
            bool checkAbove = true, checkBelow = true;

            for (int cx = left; cx < width && cell(cx, y) == 0; cx++) {
                cell(cx, y) = fillValue;
                filledCount++;

                // 위쪽 행 체크
                if (y > 0) {
                    if (cell(cx, y - 1) == 0) {
                        if (checkAbove) {
                            queue.push({cx, y - 1});
                            checkAbove = false;
                        }
                    } else {
                        checkAbove = true;
                    }
                }

                // 아래쪽 행 체크
                if (y < height - 1) {
                    if (cell(cx, y + 1) == 0) {
                        if (checkBelow) {
                            queue.push({cx, y + 1});
                            checkBelow = false;
                        }
                    } else {
                        checkBelow = true;
                    }
                }
            }
        }

        return filledCount;
    }
};

Part 3: 거리 필드 생성

목표까지의 거리를 어떻게 계산하나요?

BFS로 목표에서 시작하여 각 셀까지의 거리를 기록합니다.

거리 필드 (Distance Field) — 목표(G)에서 BFS:
4 3 █ 5 6
3 2 █ 4 5
2 1 G 1 2
3 2 1 2 3

숫자 = 목표까지의 최단 거리

거리 필드 생성
class DistanceField {
private:
    int width, height;
    std::vector<int> distances;  // -1: 벽/미방문, 0+: 거리
    std::vector<bool> obstacles;

public:
    void compute(int goalX, int goalY) {
        // 초기화
        std::fill(distances.begin(), distances.end(), -1);

        std::queue<std::pair<int, int>> queue;
        queue.push({goalX, goalY});
        distances[goalY * width + goalX] = 0;

        while (!queue.empty()) {
            auto [x, y] = queue.front();
            queue.pop();

            int currentDist = distances[y * width + x];

            // 4방향 (또는 8방향) 탐색
            for (int i = 0; i < 4; i++) {
                int nx = x + dx[i];
                int ny = y + dy[i];

                if (!isValid(nx, ny)) continue;
                if (obstacles[ny * width + nx]) continue;
                if (distances[ny * width + nx] >= 0) continue;

                distances[ny * width + nx] = currentDist + 1;
                queue.push({nx, ny});
            }
        }
    }

    int getDistance(int x, int y) const {
        if (!isValid(x, y)) return -1;
        return distances[y * width + x];
    }
};

Part 4: Flow Field 길찾기

Flow Field는 어떻게 동작하나요?

각 셀에서 목표로 향하는 방향을 저장하여, 수천 유닛이 동일한 필드를 공유합니다.

Flow Field (화살표 = 이동 방향)
↓ ↓ █ ↓ ↓
↓ ↓ █ ↙ ↓
→ → G ← ←
↗ ↑ ↑ ↑ ↖

모든 유닛은 화살표를 따라 이동하면 G에 도착!

Flow Field 생성 및 사용
struct Vec2i { int x, y; };

class FlowField {
private:
    int width, height;
    std::vector<int> distances;
    std::vector<Vec2i> flowDirections;

    // 8방향
    Vec2i dirs[8] = {
        {-1, 0}, {1, 0}, {0, -1}, {0, 1},  // 4방향
        {-1, -1}, {1, -1}, {-1, 1}, {1, 1}  // 대각선
    };

public:
    void compute(int goalX, int goalY) {
        // 1단계: 거리 필드 생성 (BFS)
        computeDistances(goalX, goalY);

        // 2단계: Flow 방향 계산
        computeFlowDirections();
    }

private:
    void computeDistances(int goalX, int goalY) {
        std::fill(distances.begin(), distances.end(), INT_MAX);
        std::queue<Vec2i> queue;

        distances[goalY * width + goalX] = 0;
        queue.push({goalX, goalY});

        while (!queue.empty()) {
            Vec2i current = queue.front();
            queue.pop();

            int currentDist = distances[current.y * width + current.x];

            for (int i = 0; i < 8; i++) {
                int nx = current.x + dirs[i].x;
                int ny = current.y + dirs[i].y;

                if (!isValid(nx, ny) || isObstacle(nx, ny)) continue;

                int newDist = currentDist + (i < 4 ? 10 : 14);  // 대각선 비용
                if (newDist < distances[ny * width + nx]) {
                    distances[ny * width + nx] = newDist;
                    queue.push({nx, ny});
                }
            }
        }
    }

    void computeFlowDirections() {
        flowDirections.resize(width * height, {0, 0});

        for (int y = 0; y < height; y++) {
            for (int x = 0; x < width; x++) {
                if (isObstacle(x, y)) continue;

                int minDist = distances[y * width + x];
                Vec2i bestDir = {0, 0};

                for (int i = 0; i < 8; i++) {
                    int nx = x + dirs[i].x;
                    int ny = y + dirs[i].y;

                    if (!isValid(nx, ny)) continue;

                    int dist = distances[ny * width + nx];
                    if (dist < minDist) {
                        minDist = dist;
                        bestDir = dirs[i];
                    }
                }

                flowDirections[y * width + x] = bestDir;
            }
        }
    }

public:
    Vec2i getFlowDirection(int x, int y) const {
        if (!isValid(x, y)) return {0, 0};
        return flowDirections[y * width + x];
    }

    // 유닛 이동
    void moveUnit(float& unitX, float& unitY, float speed, float dt) {
        int cellX = static_cast<int>(unitX);
        int cellY = static_cast<int>(unitY);

        Vec2i dir = getFlowDirection(cellX, cellY);

        // 보간을 위해 주변 셀도 고려 (선택적)
        float flowX = static_cast<float>(dir.x);
        float flowY = static_cast<float>(dir.y);

        // 정규화
        float len = std::sqrt(flowX * flowX + flowY * flowY);
        if (len > 0.01f) {
            flowX /= len;
            flowY /= len;
        }

        unitX += flowX * speed * dt;
        unitY += flowY * speed * dt;
    }
};

Part 5: 다중 목표와 가중치

여러 목표가 있으면 어떻게 하나요?

모든 목표를 시작점으로 BFS를 수행하면 "가장 가까운 목표"로 향하는 필드가 생성됩니다.

다중 목표 Flow Field
void computeMultiGoal(const std::vector<Vec2i>& goals) {
    std::fill(distances.begin(), distances.end(), INT_MAX);
    std::queue<Vec2i> queue;

    // 모든 목표를 거리 0으로 시작
    for (const Vec2i& goal : goals) {
        distances[goal.y * width + goal.x] = 0;
        queue.push(goal);
    }

    // 일반 BFS
    while (!queue.empty()) {
        // ... (동일한 BFS 로직)
    }

    computeFlowDirections();
}

// 지형 가중치 적용
void computeWithCosts(int goalX, int goalY,
                      const std::vector<int>& movementCosts) {
    // Dijkstra 사용 (BFS 대신)
    std::priority_queue<std::pair<int, Vec2i>,
                        std::vector<std::pair<int, Vec2i>>,
                        std::greater<>> pq;

    std::fill(distances.begin(), distances.end(), INT_MAX);
    distances[goalY * width + goalX] = 0;
    pq.push({0, {goalX, goalY}});

    while (!pq.empty()) {
        auto [dist, current] = pq.top();
        pq.pop();

        if (dist > distances[current.y * width + current.x]) continue;

        for (int i = 0; i < 8; i++) {
            int nx = current.x + dirs[i].x;
            int ny = current.y + dirs[i].y;

            if (!isValid(nx, ny) || isObstacle(nx, ny)) continue;

            // 이동 비용 (지형에 따라 다름)
            int baseCost = (i < 4) ? 10 : 14;
            int terrainCost = movementCosts[ny * width + nx];
            int newDist = dist + baseCost * terrainCost;

            if (newDist < distances[ny * width + nx]) {
                distances[ny * width + nx] = newDist;
                pq.push({newDist, {nx, ny}});
            }
        }
    }

    computeFlowDirections();
}

Part 6: Flow Field vs A*

언제 Flow Field를 사용해야 하나요?
상황 A* Flow Field
유닛 수 소수 (1~10) 다수 (100+)
목표 유닛마다 다름 공통 목표
계산 비용 유닛당 O(n log n) 맵당 O(n)
동적 장애물 빠른 재계산 전체 재계산 필요
사용 사례:
A* 적합
  • RPG 파티 이동
  • 개별 NPC 패스파인딩
  • 동적 목표 추적
Flow Field 적합
  • RTS 대규모 유닛 이동
  • 타워 디펜스 몬스터 웨이브
  • 군중 시뮬레이션
하이브리드
  • Flow Field로 대략적 방향
  • 로컬 A*로 장애물 회피

Supreme Commander, StarCraft 2 등 RTS 게임은 Flow Field를 사용합니다!

📌 핵심 정리

  • Flood Fill: BFS로 연결된 영역 탐색/채우기
  • 스캔라인: 수평 연속 영역 한 번에 처리로 최적화
  • 거리 필드: 목표에서 BFS로 각 셀까지 거리 계산
  • Flow Field: 거리 필드에서 방향 추출, 다수 유닛 공유
  • 활용: RTS 유닛 이동, 군중 시뮬레이션, 타워 디펜스
