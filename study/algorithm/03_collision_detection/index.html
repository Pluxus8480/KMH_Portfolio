SAT(분리축 정리)와 GJK 충돌 감지 알고리즘

Part 1: 충돌 감지의 단계

게임에서 충돌 감지는 어떤 단계로 이루어지나요?

효율성을 위해 두 단계로 나눕니다:

Broad Phase → Narrow Phase

입력
모든 물체 (1000개)
↓
Broad Phase
AABB 검사, 공간 분할O(n log n) ~ O(n)
↓ 후보 50쌍
Narrow Phase
SAT, GJK 알고리즘실제 충돌 여부 판정
↓ 실제 충돌 10쌍
Response
밀어내기, 반발, 데미지 등
왜 두 단계로 나누나요?

정밀 검사는 비용이 높기 때문입니다.

  • AABB 검사: 단순 비교 6번 (빠름)
  • SAT 검사: 축 투영 + 겹침 체크 (느림)
  • GJK 검사: 반복 계산 (더 느림)

Broad Phase로 99%를 걸러내고, 1%만 정밀 검사!

Part 2: SAT (Separating Axis Theorem)

분리축 정리(SAT)란 무엇인가요?

두 볼록 다각형이 분리되려면, 적어도 하나의 축에서 그림자가 겹치지 않아야 합니다.

🎯 비유로 이해하기

손전등으로 두 물체에 빛을 비췄을 때 그림자를 보세요.

모든 방향에서 그림자가 겹치면 충돌! 한 방향이라도 분리되면 충돌 아님!

축1 투영
A
B
겹침 ✓
축2 투영
A
B
겹침 ✓
축3 투영 (분리축)
A
B
분리 ✗

하나라도 분리 = 충돌 아님! / 모든 축에서 겹침 = 충돌!

어떤 축을 검사해야 하나요?

두 다각형의 모든 변에 수직인 축만 검사하면 됩니다.

  • 사각형 vs 사각형: 최대 4개 축
  • 삼각형 vs 삼각형: 최대 6개 축
  • n각형 vs m각형: 최대 n + m개 축

3D에서는 면의 법선 + 엣지 조합으로 축이 많아집니다!

Part 3: SAT 구현

C++ SAT 알고리즘
#include <vector>
#include <cmath>
#include <limits>

struct Vec2 {
    float x, y;

    Vec2 operator-(const Vec2& other) const {
        return {x - other.x, y - other.y};
    }

    Vec2 perpendicular() const {
        return {-y, x};  // 90도 회전 (법선 벡터)
    }

    float dot(const Vec2& other) const {
        return x * other.x + y * other.y;
    }

    Vec2 normalize() const {
        float len = std::sqrt(x * x + y * y);
        return {x / len, y / len};
    }
};

struct Polygon {
    std::vector<Vec2> vertices;

    // 축에 투영했을 때 min/max 값
    void project(const Vec2& axis, float& min, float& max) const {
        min = max = vertices[0].dot(axis);

        for (size_t i = 1; i < vertices.size(); i++) {
            float proj = vertices[i].dot(axis);
            if (proj < min) min = proj;
            if (proj > max) max = proj;
        }
    }

    // 모든 변의 법선 벡터 (축 후보)
    std::vector<Vec2> getAxes() const {
        std::vector<Vec2> axes;
        for (size_t i = 0; i < vertices.size(); i++) {
            Vec2 edge = vertices[(i + 1) % vertices.size()] - vertices[i];
            axes.push_back(edge.perpendicular().normalize());
        }
        return axes;
    }
};

class SAT {
public:
    struct CollisionResult {
        bool colliding;
        float penetration;  // 침투 깊이
        Vec2 normal;        // 충돌 법선 (밀어낼 방향)
    };

    static CollisionResult checkCollision(const Polygon& a, const Polygon& b) {
        CollisionResult result = {true, std::numeric_limits<float>::max(), {0, 0}};

        // A의 축들에서 검사
        auto axesA = a.getAxes();
        for (const auto& axis : axesA) {
            if (!checkAxis(a, b, axis, result)) {
                result.colliding = false;
                return result;  // 분리축 발견!
            }
        }

        // B의 축들에서 검사
        auto axesB = b.getAxes();
        for (const auto& axis : axesB) {
            if (!checkAxis(a, b, axis, result)) {
                result.colliding = false;
                return result;  // 분리축 발견!
            }
        }

        return result;  // 모든 축에서 겹침 = 충돌!
    }

private:
    static bool checkAxis(const Polygon& a, const Polygon& b,
                          const Vec2& axis, CollisionResult& result) {
        float minA, maxA, minB, maxB;
        a.project(axis, minA, maxA);
        b.project(axis, minB, maxB);

        // 겹침 체크
        if (maxA < minB || maxB < minA) {
            return false;  // 분리됨!
        }

        // 침투 깊이 계산 (가장 작은 겹침 저장)
        float overlap = std::min(maxA - minB, maxB - minA);
        if (overlap < result.penetration) {
            result.penetration = overlap;
            result.normal = axis;
        }

        return true;  // 이 축에서는 겹침
    }
};

Part 4: GJK 알고리즘

GJK는 SAT와 무엇이 다른가요?

GJK는 민코프스키 차(Minkowski Difference)를 이용합니다.

두 도형의 민코프스키 차가 원점을 포함하면 충돌!

도형 A
−
도형 B
=
A - B (민코프스키 차)
• 원점 포함 → 충돌!
• 원점 미포함 → 충돌 아님
GJK의 장점은 무엇인가요?
특성 SAT GJK
적용 범위 볼록 다각형만 모든 볼록 도형
곡선 도형 근사 필요 직접 처리 가능
조기 종료 분리축 발견 시 원점 미포함 확인 시
확장 - EPA로 침투 정보

GJK는 원, 타원 등 곡선 도형에도 사용 가능!

Part 5: GJK 구현

C++ GJK 알고리즘
struct Vec2 {
    float x, y;

    Vec2 operator+(const Vec2& o) const { return {x + o.x, y + o.y}; }
    Vec2 operator-(const Vec2& o) const { return {x - o.x, y - o.y}; }
    Vec2 operator*(float s) const { return {x * s, y * s}; }
    Vec2 operator-() const { return {-x, -y}; }
    float dot(const Vec2& o) const { return x * o.x + y * o.y; }
    float cross(const Vec2& o) const { return x * o.y - y * o.x; }
};

// Support 함수: 주어진 방향으로 가장 먼 점
Vec2 support(const Polygon& shape, const Vec2& direction) {
    float maxDot = -std::numeric_limits<float>::max();
    Vec2 result;

    for (const auto& v : shape.vertices) {
        float d = v.dot(direction);
        if (d > maxDot) {
            maxDot = d;
            result = v;
        }
    }
    return result;
}

// 민코프스키 차의 Support
Vec2 minkowskiSupport(const Polygon& a, const Polygon& b, const Vec2& dir) {
    return support(a, dir) - support(b, -dir);
}

class GJK {
public:
    static bool checkCollision(const Polygon& a, const Polygon& b) {
        Vec2 direction = {1, 0};  // 초기 방향
        std::vector<Vec2> simplex;

        // 첫 번째 점
        Vec2 pointA = minkowskiSupport(a, b, direction);
        simplex.push_back(pointA);
        direction = -pointA;  // 원점 방향으로

        const int MAX_ITERATIONS = 50;
        for (int i = 0; i < MAX_ITERATIONS; i++) {
            Vec2 newPoint = minkowskiSupport(a, b, direction);

            // 새 점이 원점을 지나지 못함 = 충돌 없음
            if (newPoint.dot(direction) < 0) {
                return false;
            }

            simplex.push_back(newPoint);

            // 심플렉스가 원점을 포함하는지 확인
            if (handleSimplex(simplex, direction)) {
                return true;  // 충돌!
            }
        }

        return false;
    }

private:
    static bool handleSimplex(std::vector<Vec2>& simplex, Vec2& direction) {
        if (simplex.size() == 2) {
            return handleLine(simplex, direction);
        }
        return handleTriangle(simplex, direction);
    }

    static bool handleLine(std::vector<Vec2>& simplex, Vec2& direction) {
        Vec2 a = simplex[1];
        Vec2 b = simplex[0];
        Vec2 ab = b - a;
        Vec2 ao = -a;  // 원점 방향

        // 원점이 AB 선분의 어느 쪽에 있는지
        direction = tripleProduct(ab, ao, ab);
        return false;
    }

    static bool handleTriangle(std::vector<Vec2>& simplex, Vec2& direction) {
        Vec2 a = simplex[2];
        Vec2 b = simplex[1];
        Vec2 c = simplex[0];

        Vec2 ab = b - a;
        Vec2 ac = c - a;
        Vec2 ao = -a;

        Vec2 abPerp = tripleProduct(ac, ab, ab);
        Vec2 acPerp = tripleProduct(ab, ac, ac);

        // 원점이 AB 외부
        if (abPerp.dot(ao) > 0) {
            simplex.erase(simplex.begin());  // C 제거
            direction = abPerp;
            return false;
        }

        // 원점이 AC 외부
        if (acPerp.dot(ao) > 0) {
            simplex.erase(simplex.begin() + 1);  // B 제거
            direction = acPerp;
            return false;
        }

        // 원점이 삼각형 내부!
        return true;
    }

    static Vec2 tripleProduct(const Vec2& a, const Vec2& b, const Vec2& c) {
        float ac = a.dot(c);
        float bc = b.dot(c);
        return {b.x * ac - a.x * bc, b.y * ac - a.y * bc};
    }
};

Part 6: EPA와 충돌 응답

충돌 후 어떻게 처리하나요?

GJK는 충돌 여부만 알려주므로, EPA(Expanding Polytope Algorithm)로 침투 정보를 얻습니다.

Step 1
GJK가 찾은 심플렉스(삼각형)에서 시작, 원점 포함
↓
Step 2
원점에서 가장 가까운 엣지 찾기
↓
Step 3
그 방향으로 새 Support 점 추가
↓
Step 4
반복하여 침투 벡터(MTV) 계산 — 물체를 밀어낼 최소 거리와 방향
실제 충돌 응답은 어떻게 구현하나요?
충돌 응답 (밀어내기)
void resolveCollision(GameObject& a, GameObject& b,
                      const Vec2& mtv, float penetration) {
    // 두 물체를 침투 깊이만큼 분리
    // 질량에 반비례하여 분배

    float totalMass = a.mass + b.mass;
    float ratioA = b.mass / totalMass;
    float ratioB = a.mass / totalMass;

    Vec2 correction = mtv * penetration;

    a.position = a.position + correction * ratioA;
    b.position = b.position - correction * ratioB;

    // 속도 반영 (반발 계수)
    float restitution = std::min(a.restitution, b.restitution);

    float relativeVel = (a.velocity - b.velocity).dot(mtv);
    if (relativeVel > 0) return;  // 이미 분리 중

    float impulse = -(1 + restitution) * relativeVel / totalMass;

    a.velocity = a.velocity + mtv * (impulse * b.mass);
    b.velocity = b.velocity - mtv * (impulse * a.mass);
}

📌 핵심 정리

  • Broad Phase → Narrow Phase: 계층적 필터링으로 검사량 감소
  • SAT: 분리축 정리, 모든 축에서 겹쳐야 충돌
  • GJK: 민코프스키 차가 원점 포함 시 충돌, 곡선 도형 지원
  • EPA: GJK 후 침투 깊이와 방향(MTV) 계산
  • 충돌 응답: MTV로 분리 + 반발 계수로 속도 조정
