캐릭터에 생명을 불어넣는 법

Animator와 Playables API로 애니메이션 시스템 심화

Part 1: Animator Controller 기초

Animator Controller가 정확히 뭔가요?

Animator Controller는 캐릭터의 애니메이션 상태와 전환 규칙을 정의하는 상태 머신(State Machine)입니다.

🎯 비유로 이해하기

신호등을 생각해보세요. 초록 → 노랑 → 빨강 순서로 바뀌고, 각 상태에서 다음 상태로 전환되는 조건이 정해져 있습니다.

Animator도 Idle → Walk → Run 처럼 상태를 정의하고, 파라미터 조건에 따라 전환합니다.

Animator는 상태(State), 전환(Transition), 파라미터(Parameter) 세 가지 요소로 구성됩니다.

파라미터 타입은 어떤 게 있나요?
타입용도예시
Float연속적 값 (블렌드 트리)Speed, Direction
Int정수 상태 구분WeaponType
BoolOn/Off 토글IsGrounded, IsCrouching
Trigger일회성 이벤트Jump, Attack, Die
C# - Animator 파라미터 제어
// 해시 캐싱 (문자열 비교보다 빠름)
private static readonly int SpeedHash = Animator.StringToHash("Speed");
private static readonly int JumpHash = Animator.StringToHash("Jump");
private static readonly int IsGroundedHash = Animator.StringToHash("IsGrounded");

void Update()
{
    float speed = _rigidbody.velocity.magnitude;
    _animator.SetFloat(SpeedHash, speed);
    _animator.SetBool(IsGroundedHash, _isGrounded);

    if (Input.GetButtonDown("Jump"))
        _animator.SetTrigger(JumpHash);
}

Animator.SetFloat("Speed", value)처럼 문자열을 직접 사용하면 매번 해시 변환이 발생합니다. 반드시 StringToHash로 캐싱하세요.

Part 2: Animation Layer와 Avatar Mask

하체는 걷고 상체는 총을 쏘려면 어떻게 해야 하나요?

Animation Layer를 사용합니다. 레이어를 분리하면 신체 부위별로 독립적인 애니메이션을 재생할 수 있습니다.

Base Layer
이동 애니메이션 (전신)
+
Upper Body Layer
상체 전투 애니메이션
=
최종 결과
하체 이동 + 상체 사격
Avatar Mask는 뭔가요?

Avatar Mask는 레이어가 영향을 줄 본(뼈대)을 선택하는 필터입니다.

Upper Body Mask를 만들면 Spine(척추) 이상의 본만 해당 레이어의 애니메이션이 적용되고, 하체는 Base Layer의 애니메이션을 유지합니다.

레이어 속성설명
Weight레이어 영향도 (0~1). 코드에서 동적으로 조절 가능
Blending: Override하위 레이어를 완전히 덮어씀
Blending: Additive하위 레이어에 추가 (숨쉬기 등)
C# - 레이어 Weight 동적 제어
// 전투 진입 시 상체 레이어 활성화
public void EnterCombat()
{
    // 1번 레이어(Upper Body)의 Weight를 1로
    _animator.SetLayerWeight(1, 1f);
}

// 전투 종료 시 부드럽게 비활성화
private IEnumerator FadeOutUpperLayer()
{
    float weight = 1f;
    while (weight > 0f)
    {
        weight -= Time.deltaTime * 2f;
        _animator.SetLayerWeight(1, Mathf.Max(0, weight));
        yield return null;
    }
}

Part 3: Blend Tree 활용

Idle, Walk, Run을 자연스럽게 전환하려면?

Blend Tree를 사용하면 파라미터 값에 따라 여러 애니메이션을 부드럽게 섞어줍니다.

Blend Tree는 상태 전환(Transition)과 다릅니다. Transition은 A에서 B로 전환되지만, Blend Tree는 A와 B를 동시에 섞습니다.

1D Blend와 2D Blend는 어떻게 다른가요?
타입파라미터용도
1D1개 (ex: Speed)Idle → Walk → Run 속도 블렌딩
2D Simple Directional2개 (ex: X, Y)8방향 이동 (각 방향 1개 클립)
2D Freeform Directional2개같은 방향에 여러 속도의 클립
2D Freeform Cartesian2개비방향성 블렌딩 (기울기+속도)
C# - 2D Blend Tree 파라미터 설정
// 이동 입력을 Blend Tree 파라미터로 전달
void Update()
{
    float h = Input.GetAxis("Horizontal");
    float v = Input.GetAxis("Vertical");

    // 부드러운 댐핑 적용 (급격한 전환 방지)
    _animator.SetFloat("MoveX", h, 0.1f, Time.deltaTime);
    _animator.SetFloat("MoveY", v, 0.1f, Time.deltaTime);
}

Part 4: Playables API

Animator Controller가 있는데 왜 Playables API가 필요한가요?

Animator Controller는 에디터에서 시각적으로 편리하지만, 런타임에서 동적으로 애니메이션을 조합하기 어렵습니다.

Playables API는 코드로 애니메이션 그래프를 생성하고 조작할 수 있어, 런타임 동적 애니메이션에 적합합니다.

Playables API의 구조는 어떻게 되나요?
PlayableGraph
전체 그래프 컨테이너
→
AnimationMixerPlayable
여러 입력을 혼합
→
AnimationPlayableOutput
Animator에 결과 출력
C# - Playables API 기본 사용법
using UnityEngine.Playables;
using UnityEngine.Animations;

public class PlayableAnimator : MonoBehaviour
{
    public AnimationClip walkClip;
    public AnimationClip runClip;

    private PlayableGraph _graph;
    private AnimationMixerPlayable _mixer;

    void Start()
    {
        // 그래프 생성
        _graph = PlayableGraph.Create("MyGraph");

        // 클립을 Playable로 변환
        var walkPlayable = AnimationClipPlayable.Create(_graph, walkClip);
        var runPlayable = AnimationClipPlayable.Create(_graph, runClip);

        // 믹서 생성 (2개 입력)
        _mixer = AnimationMixerPlayable.Create(_graph, 2);
        _graph.Connect(walkPlayable, 0, _mixer, 0);
        _graph.Connect(runPlayable, 0, _mixer, 1);

        // 출력 연결
        var output = AnimationPlayableOutput.Create(_graph, "Anim", GetComponent<Animator>());
        output.SetSourcePlayable(_mixer);

        _graph.Play();
    }

    void Update()
    {
        // 속도에 따라 Walk/Run 블렌딩
        float blend = Mathf.Clamp01(_speed / _maxSpeed);
        _mixer.SetInputWeight(0, 1f - blend);
        _mixer.SetInputWeight(1, blend);
    }

    void OnDestroy()
    {
        if (_graph.IsValid())
            _graph.Destroy();
    }
}

PlayableGraph는 반드시 OnDestroy에서 Destroy()를 호출해야 합니다. 그렇지 않으면 메모리 누수가 발생합니다.

Part 5: Inverse Kinematics (IK)

캐릭터가 경사면에서도 자연스럽게 발을 딛게 하려면?

Inverse Kinematics(IK)를 사용합니다. 기존 Forward Kinematics는 루트 → 말단 방향으로 본을 회전시키지만, IK는 반대로 말단 위치를 지정하면 나머지 본이 자동 계산됩니다.

🎯 비유로 이해하기

로봇 팔을 생각해보세요. FK는 각 관절의 각도를 일일이 설정해서 손 위치를 결정합니다.

IK는 "손이 여기 와야 해"라고 목표만 지정하면 각 관절 각도를 역으로 계산합니다.

C# - Foot IK 구현
// Animator에서 IK Pass 활성화 필요 (레이어 설정)
void OnAnimatorIK(int layerIndex)
{
    // 왼발 IK
    Ray ray = new Ray(
        _animator.GetIKPosition(AvatarIKGoal.LeftFoot) + Vector3.up,
        Vector3.down
    );

    if (Physics.Raycast(ray, out RaycastHit hit, 2f, _groundLayer))
    {
        _animator.SetIKPositionWeight(AvatarIKGoal.LeftFoot, 1f);
        _animator.SetIKRotationWeight(AvatarIKGoal.LeftFoot, 1f);

        _animator.SetIKPosition(AvatarIKGoal.LeftFoot, hit.point + Vector3.up * 0.05f);

        Quaternion footRotation = Quaternion.LookRotation(
            Vector3.ProjectOnPlane(transform.forward, hit.normal), hit.normal
        );
        _animator.SetIKRotation(AvatarIKGoal.LeftFoot, footRotation);
    }
}
IK는 어디에 주로 사용되나요?

Hand IK총기, 손잡이 등 오브젝트를 정확히 잡도록

Part 6: 애니메이션 성능 최적화

애니메이션이 많아지면 성능 문제는 없나요?

Animator Controller가 복잡해지면 상태 전환 평가, 블렌딩 계산 비용이 증가합니다.

Culling Mode 설정으로 화면 밖 캐릭터의 애니메이션 비용을 줄일 수 있습니다.

Culling Mode동작용도
Always Animate항상 업데이트카메라 밖에서도 상태가 중요한 경우
Cull Update Transforms본 변환만 생략대부분의 NPC
Cull Completely완전히 중단배경 캐릭터, 군중

Cull Completely를 사용하면 화면에 다시 들어왔을 때 마지막 상태에서 재개됩니다. 시간 기반 로직이 있다면 주의하세요.

📌 핵심 정리

  • Animator Controller: 상태 머신 기반 — 파라미터 해시를 캐싱하여 성능을 확보하라
  • Layer + Avatar Mask: 상/하체 분리 애니메이션은 레이어와 마스크로 구현한다
  • Blend Tree: 이동 애니메이션은 Transition이 아닌 Blend Tree로 자연스럽게 섞어라
  • Playables API: 런타임 동적 애니메이션이 필요하면 Playables로 그래프를 직접 구성하라
  • IK: Foot IK로 지형 적응, Hand IK로 오브젝트 상호작용을 자연스럽게 처리하라
