함수를 변수처럼 다룰 수 있다면?

delegate와 event - C#의 함수 타입화와 안전한 이벤트 시스템

Part 1: delegate(델리게이트)란?

delegate는 무엇인가요?

C#에서 델리게이트는 함수를 타입화한 것입니다. C++의 함수 포인터와 비슷한 개념입니다.

파라미터와 리턴 타입을 통해 정의하며, 동일한 시그니처를 가진 메서드들과 호환되어 참조를 가질 수 있습니다.

C#
public delegate void VoidAndIntEx(int i);

public class ExampleClass
{
    public void DoSomething(VoidAndIntEx exFunc)
    {
        // 인자로 받은 함수를 호출한다
        exFunc(1);
    }
}

델리게이트를 활용하면 메서드를 담아두거나, 함수 인자로 넘겨서 콜백 패턴을 구현하는 등 다양한 곳에 사용할 수 있습니다.

Part 2: event란?

event와 delegate의 차이점은?

event는 delegate와 비슷한 일을 하지만 한 가지 큰 차이점이 있습니다.

이벤트를 호출(Invoke)할 수 있는 것은 해당 이벤트를 소유한 클래스만 가능합니다. 외부에서는 구독(+=)과 구독 해제(-=)만 할 수 있습니다.

C#
class ExampleClass
{
    public event Action ExampleEvent;

    public void TriggerEvent()
    {
        if (ExampleEvent != null)
        {
            ExampleEvent(); // 내부에서만 호출 가능
        }
    }
}

// 외부 코드
var obj = new ExampleClass();
obj.ExampleEvent += MyHandler;  // 구독 O
// obj.ExampleEvent();          // 컴파일 에러! 외부 호출 X

Part 3: Action, Func, Predicate

매번 delegate를 선언해야 하나요?

자주 사용하는 패턴은 이미 제네릭 델리게이트로 제공됩니다.

키워드 파라미터 반환값 용도
Action<T> T void 반환 없는 함수
Func<T, TResult> T TResult 반환 있는 함수
Predicate<T> T bool 조건 판별 함수

Part 4: null 조건부 연산자 (?.)

delegate나 event가 null일 때 호출하면 어떻게 되나요?

null인 delegate를 호출하면 NullReferenceException이 발생합니다.

C# - 위험한 코드
if (ExampleEvent != null) // 여기서는 null이 아니었는데
{
    // 다른 스레드가 구독을 취소해서 null이 됨!
    ExampleEvent(); // NullReferenceException!
}
멀티스레드에서 안전하게 호출하는 방법은?

전통적으로는 "복사 후 실행" 패턴을 사용했습니다:

C# - 복사 후 실행
var copiedEvent = ExampleEvent;
if (copiedEvent != null)
{
    copiedEvent(); // ExampleEvent 구독 취소해도 안전
}

하지만 더 간결한 방법이 있습니다.

?. (null 조건부 연산자)를 사용하면 한 줄로 안전하게 호출할 수 있습니다. 이 연산자는 원자적으로 수행되므로 멀티스레드에서도 안전합니다.

C# - null 조건부 연산자
ExampleEvent?.Invoke(); // null이면 호출 안 함, 원자적 실행

?. 연산자는 원자적(atomic)으로 수행되어 연산 도중 다른 스레드가 개입할 여지가 없습니다.

📌 핵심 정리

  • delegate는 C#에서 함수를 타입화한 것으로 C++의 함수 포인터와 유사하다
  • event는 delegate에 접근 제한을 추가하여, 소유 클래스만 호출 가능하다
  • Action, Func, Predicate는 자주 사용하는 delegate 패턴의 제네릭 버전이다
  • ?. 연산자로 null 안전하고 스레드 안전한 호출이 가능하다
