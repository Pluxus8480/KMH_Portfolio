Part 1: URP와 렌더 파이프라인 이해

Unity에 렌더 파이프라인이 여러 개 있다는데, 왜 그런 건가요?

Unity는 다양한 플랫폼과 요구사항에 대응하기 위해 Scriptable Render Pipeline(SRP) 아키텍처를 도입했습니다.

파이프라인대상Shader Graph 지원
Built-in RP 레거시 프로젝트 제한적
URP 모바일~콘솔 (범용) 완전 지원
HDRP 고사양 PC/콘솔 완전 지원

Shader Graph는 URP와 HDRP에서만 완전히 지원됩니다. Built-in RP에서는 사용할 수 없으므로 프로젝트 시작 시 파이프라인 선택이 중요합니다.

URP가 게임 개발에서 가장 많이 선택되는 이유는?

성능과 품질의 균형이 뛰어나기 때문입니다. 모바일부터 콘솔까지 하나의 파이프라인으로 대응할 수 있고, 배칭과 SRP Batcher 덕분에 드로우 콜 최적화도 용이합니다.

URP에서 Shader Graph를 사용하려면 프로젝트에 Universal RP 패키지가 설치되어 있어야 하며, Graphics Settings에서 URP Asset이 할당되어야 합니다.

Part 2: Shader Graph 기본 노드 이해

Shader Graph에서 가장 먼저 알아야 할 노드는 무엇인가요?

Shader Graph는 노드(Node)를 연결하여 셰이더를 만드는 비주얼 도구입니다. 핵심 노드 카테고리를 이해하면 대부분의 이펙트를 구현할 수 있습니다.

Input 노드
텍스처, 색상, UV, 시간
→
Math / Utility
Add, Multiply, Lerp, Step
→
Master Stack
Base Color, Normal, Alpha 등

자주 사용하는 핵심 노드들:

노드역할사용 예
Sample Texture 2D텍스처 샘플링알베도, 노멀맵 읽기
UVUV 좌표 제공텍스처 매핑 기준
Time시간값 제공애니메이션, 스크롤
Lerp선형 보간두 값 사이 블렌딩
Step / Smoothstep임계값 기반 분리디졸브, 엣지 감지
Fresnel Effect시선 각도 기반 강도림 라이트, 홀로그램
Master Stack이 뭔가요?

Master Stack은 그림의 "최종 캔버스"입니다. 여러 물감(노드)을 혼합하여 최종 결과물을 캔버스에 올리는 것처럼, 모든 노드 계산의 결과가 Master Stack의 각 슬롯에 연결됩니다.

주요 슬롯: Base Color(기본 색), Normal(표면 법선), Metallic(금속성), Smoothness(매끄러움), Emission(발광), Alpha(투명도)

Part 3: 디졸브 이펙트 구현

적이 사라질 때 디졸브(용해) 이펙트를 Shader Graph로 어떻게 만드나요?

핵심 원리: 노이즈 텍스처의 값이 특정 임계값(Threshold) 미만이면 픽셀을 투명하게 만든다.

Simple Noise
노이즈 패턴 생성
→
Step (Threshold)
임계값 기준 0/1 분리
→
Alpha Clip
투명 영역 제거

노이즈 값이 임계값보다 작은 픽셀을 버리되, 경계 부분에 발광색을 입히면 타는 듯한 느낌을 줄 수 있습니다.

C# - 디졸브 제어 스크립트
public class DissolveController : MonoBehaviour
{
    [SerializeField] private float dissolveDuration = 2f;
    private Material material;
    private static readonly int DissolveAmount = Shader.PropertyToID("_DissolveAmount");
    private static readonly int EdgeColor = Shader.PropertyToID("_EdgeColor");

    void Start()
    {
        // 인스턴스 머티리얼 사용 (원본 변경 방지)
        material = GetComponent<Renderer>().material;
    }

    public void StartDissolve()
    {
        StartCoroutine(DissolveRoutine());
    }

    private IEnumerator DissolveRoutine()
    {
        float elapsed = 0f;
        while (elapsed < dissolveDuration)
        {
            elapsed += Time.deltaTime;
            float t = elapsed / dissolveDuration;
            material.SetFloat(DissolveAmount, t);
            yield return null;
        }
        gameObject.SetActive(false);
    }
}

Renderer.material은 머티리얼 인스턴스를 생성하므로 메모리 누수에 주의하세요. 오브젝트가 파괴되거나 풀로 반환될 때 명시적으로 Destroy(material)을 호출해야 합니다.

Part 4: 홀로그램 이펙트 구현

SF 게임에서 자주 보이는 홀로그램 효과는 어떤 원리인가요?

홀로그램은 세 가지 요소의 조합입니다:

  • Fresnel Effect: 가장자리가 밝게 빛나는 림 라이트
  • 스캔라인: 위아래로 움직이는 가로줄 패턴
  • 글리치: 불규칙한 왜곡/깜빡임
🎯 비유로 이해하기

Fresnel Effect는 비눗방울을 떠올리세요. 정면으로 보면 투명하지만, 가장자리(비스듬한 각도)에서는 무지개빛이 강하게 보입니다.

셰이더에서도 카메라와 표면의 각도(Normal vs View Direction)가 클수록 값이 강해집니다.

Shader Graph에서 Fresnel Effect 노드 하나로 림 라이트를 구현할 수 있으며, Power 값이 클수록 가장자리 발광이 얇아집니다.

스캔라인은 어떻게 만드나요?

오브젝트의 월드 Y 좌표에 Time을 더한 뒤 Sine/Frac 함수를 적용하면 스캔라인이 됩니다.

노드 연결 흐름:

Position (World)
Y 컴포넌트 추출
→
Add (Time)
스크롤 애니메이션
→
Multiply (Scale)
라인 밀도 조절
→
Sine → Remap
0~1 범위로 변환

결과를 Alpha나 Emission에 곱하면 스캔라인이 지나가는 느낌이 완성됩니다.

Part 5: 커스텀 라이팅과 Toon 셰이더

Shader Graph로 카툰 렌더링(Toon Shading)도 가능한가요?

가능합니다. 핵심은 라이트 방향과 노멀의 내적(Dot Product)을 구한 뒤, 이를 Smoothstep으로 몇 단계로 끊어서 셀 셰이딩 효과를 만드는 것입니다.

Shader Graph 노드 로직 (의사 코드)
// 1. 조명 방향과 표면 법선의 내적
float NdotL = dot(Normal, LightDirection);

// 2. 0~1로 정규화
float halfLambert = NdotL * 0.5 + 0.5;

// 3. Step으로 2단계 명암 분리 (Toon)
float toon = step(0.5, halfLambert);

// 4. 최종 색상
float3 finalColor = baseColor * lerp(shadowColor, lightColor, toon);

URP Shader Graph에서 메인 라이트 방향을 얻으려면 Custom Function 노드가 필요합니다. 기본 제공 노드만으로는 라이트 정보에 접근할 수 없습니다.

Custom Function 노드는 어떻게 사용하나요?

HLSL 파일을 작성하고 Custom Function 노드에서 참조합니다.

HLSL - GetMainLight.hlsl
#ifndef CUSTOM_LIGHTING_INCLUDED
#define CUSTOM_LIGHTING_INCLUDED

void GetMainLight_float(float3 WorldPos, out float3 Direction,
                        out float3 Color, out float Attenuation)
{
#ifdef SHADERGRAPH_PREVIEW
    Direction = float3(0.5, 0.5, 0);
    Color = 1;
    Attenuation = 1;
#else
    Light mainLight = GetMainLight();
    Direction = mainLight.direction;
    Color = mainLight.color;
    Attenuation = mainLight.distanceAttenuation * mainLight.shadowAttenuation;
#endif
}

#endif

SHADERGRAPH_PREVIEW 매크로는 Shader Graph 에디터의 미리보기에서 오류를 방지합니다. 프리뷰에서는 라이트 데이터가 없으므로 하드코딩된 값을 사용합니다.

Part 6: 성능 최적화와 주의사항

Shader Graph로 만든 셰이더는 성능이 괜찮나요?

Shader Graph가 생성하는 코드는 범용적이므로, 수작성 셰이더 대비 약간의 오버헤드가 있을 수 있습니다. 하지만 대부분의 경우 차이는 미미합니다.

항목Shader Graph수작성 ShaderLab
개발 속도빠름 (비주얼)느림 (코드)
디버깅노드별 미리보기RenderDoc 필요
성능 미세 최적화어려움가능
팀 협업아티스트 접근 용이프로그래머 전용

Shader Graph에서 Generated Code를 확인(Inspector > View Generated Shader)하여 불필요한 연산이 없는지 검증하세요.

📌 핵심 정리

  • Shader Graph는 URP/HDRP에서 코드 없이 셰이더를 만드는 비주얼 도구이다
  • 디졸브 이펙트는 노이즈 + Step + Alpha Clip으로 구현한다
  • 홀로그램은 Fresnel + 스캔라인(Sine) + 글리치의 조합이다
  • 커스텀 라이팅(Toon)은 Custom Function 노드로 라이트 정보를 가져와 구현한다
  • Renderer.material은 인스턴스를 생성하므로 메모리 관리에 주의해야 한다
