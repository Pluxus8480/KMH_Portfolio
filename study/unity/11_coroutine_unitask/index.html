yield return 너머의 세계

Coroutine과 UniTask — Unity 비동기 프로그래밍 완벽 비교

Part 1: Coroutine은 어떻게 동작하는가?

Coroutine의 yield return null이 내부적으로 어떻게 동작하나요?

Unity의 Coroutine은 C#의 IEnumerator 인터페이스를 활용한 일종의 트릭입니다. 실제 멀티스레드가 아니라, 메인 스레드에서 실행을 쪼개는 것입니다.

C# 컴파일러는 yield return이 포함된 메서드를 내부적으로 상태 머신(State Machine) 클래스로 변환합니다. 각 yield return이 상태 전환 지점이 됩니다.

C# - 우리가 작성하는 코드
IEnumerator FadeOut()
{
    float alpha = 1f;
    while (alpha > 0)
    {
        alpha -= Time.deltaTime;
        SetAlpha(alpha);
        yield return null; // 다음 프레임까지 대기
    }
}
C# - 컴파일러가 변환한 모습 (간략화)
// 컴파일러가 자동 생성하는 상태 머신
class FadeOut_StateMachine : IEnumerator
{
    private int state = 0;
    private float alpha;
    private MonoBehaviour self;

    public object Current { get; private set; }

    public bool MoveNext()
    {
        switch (state)
        {
            case 0:
                alpha = 1f;
                state = 1;
                goto case 1;

            case 1:
                if (alpha > 0)
                {
                    alpha -= Time.deltaTime;
                    self.SetAlpha(alpha);
                    Current = null; // yield return null
                    state = 1;
                    return true;    // 아직 끝나지 않음
                }
                return false;       // 코루틴 종료
        }
        return false;
    }
}

Unity 엔진은 매 프레임 MoveNext()를 호출합니다. true를 반환하면 계속 실행, false를 반환하면 코루틴 종료입니다.

Part 2: Coroutine의 한계

Coroutine은 왜 "레거시"라는 평가를 받나요?

여전히 유용하지만, 현대적 비동기 프로그래밍 관점에서 여러 한계가 있습니다:

1. 반환값이 없다 — IEnumerator는 값을 반환할 수 없어, 결과를 콜백이나 외부 변수로 전달해야 합니다.

C# - 코루틴의 불편한 결과 전달
// 결과를 받을 방법이 없어서 콜백 사용
IEnumerator LoadData(System.Action<string> onComplete)
{
    var request = UnityWebRequest.Get("https://api.example.com/data");
    yield return request.SendWebRequest();
    onComplete?.Invoke(request.downloadHandler.text);
}

// 호출 시 콜백 지옥 발생
StartCoroutine(LoadData(data => {
    StartCoroutine(ParseData(data, parsed => {
        StartCoroutine(ApplyData(parsed, () => {
            Debug.Log("완료... 읽기 힘들다");
        }));
    }));
}));

2. 에러 처리 불가 — try-catch로 yield return을 감쌀 수 없습니다.

3. MonoBehaviour 종속 — 게임오브젝트가 비활성화되거나 파괴되면 코루틴이 즉시 중단됩니다.

4. GC 할당 — new WaitForSeconds()는 매번 힙에 할당합니다. (WaitForSeconds를 캐싱하여 완화 가능)

WaitForSeconds를 캐싱하면 얼마나 도움이 되나요?
C# - WaitForSeconds 캐싱 패턴
// BAD: 매번 new 할당
IEnumerator SpawnLoop()
{
    while (true)
    {
        SpawnEnemy();
        yield return new WaitForSeconds(2f); // GC 부담
    }
}

// GOOD: 캐싱하여 재사용
private static readonly WaitForSeconds Wait2Sec = new WaitForSeconds(2f);

IEnumerator SpawnLoop()
{
    while (true)
    {
        SpawnEnemy();
        yield return Wait2Sec; // 할당 없음
    }
}

WaitForSeconds, WaitForFixedUpdate 등은 readonly static으로 캐싱하면 GC 할당을 제거할 수 있습니다. 하지만 WaitForSecondsRealtime은 내부 상태가 있어 캐싱할 수 없습니다.

Part 3: UniTask 소개

UniTask는 무엇이고, Coroutine과 무엇이 다른가요?

UniTask는 Unity에 최적화된 async/await 라이브러리입니다. C#의 Task와 달리 제로 GC 할당을 목표로 설계되었습니다.

🎯 비유로 이해하기

코루틴이 "전화로 한 줄씩 지시받는 요리"라면, UniTask는 "레시피 전체를 보고 자율적으로 요리하는 것"입니다.

await는 "이 단계가 끝날 때까지 기다린 후 다음을 진행"이라는 자연스러운 흐름을 코드에 직접 표현합니다.

C# - 동일한 작업: Coroutine vs UniTask
// Coroutine 방식
IEnumerator LoadAndApply(System.Action<bool> onComplete)
{
    var request = UnityWebRequest.Get(url);
    yield return request.SendWebRequest();

    if (request.result != UnityWebRequest.Result.Success)
    {
        Debug.LogError(request.error);
        onComplete?.Invoke(false);
        yield break;
    }

    var texture = DownloadHandlerTexture.GetContent(request);
    image.texture = texture;
    onComplete?.Invoke(true);
}

// UniTask 방식
async UniTask<bool> LoadAndApplyAsync()
{
    try
    {
        var request = await UnityWebRequest.Get(url).SendWebRequest();
        var texture = DownloadHandlerTexture.GetContent(request);
        image.texture = texture;
        return true;
    }
    catch (UnityWebRequestException e)
    {
        Debug.LogError(e.Message);
        return false;
    }
}

UniTask는 반환값, try-catch 에러 처리, MonoBehaviour 비종속이 모두 가능합니다.

Part 4: UniTask 실전 활용

UniTask의 대기(Delay) 기능은 어떻게 다른가요?
C# - UniTask 다양한 대기 방법
using Cysharp.Threading.Tasks;
using System.Threading;

public class UniTaskExamples : MonoBehaviour
{
    // 시간 대기
    async UniTaskVoid DelayExample()
    {
        await UniTask.Delay(1000);                    // 1초 (ms 단위)
        await UniTask.Delay(TimeSpan.FromSeconds(2)); // 2초
        await UniTask.DelayFrame(5);                  // 5프레임
        await UniTask.Yield(PlayerLoopTiming.FixedUpdate); // FixedUpdate 대기
    }

    // 조건 대기
    async UniTaskVoid WaitExample()
    {
        await UniTask.WaitUntil(() => hp <= 0);
        PlayDeathAnimation();
    }

    // 병렬 실행
    async UniTask LoadAllResources()
    {
        var (textures, sounds, configs) = await UniTask.WhenAll(
            LoadTexturesAsync(),
            LoadSoundsAsync(),
            LoadConfigsAsync()
        );
        // 세 작업이 모두 완료된 후 실행
        InitializeGame(textures, sounds, configs);
    }

    // 타임아웃
    async UniTask<string> FetchWithTimeout()
    {
        var cts = new CancellationTokenSource();
        cts.CancelAfterSlim(TimeSpan.FromSeconds(5));

        try
        {
            return await FetchDataAsync()
                .AttachExternalCancellation(cts.Token);
        }
        catch (OperationCanceledException)
        {
            Debug.Log("타임아웃!");
            return null;
        }
    }
}
CancellationToken이 왜 중요한가요?

CancellationToken은 UniTask의 생명줄입니다. 오브젝트가 파괴되었는데 비동기 작업이 계속 실행되면 NullReferenceException이 발생합니다.

C# - CancellationToken 올바른 사용법
public class Enemy : MonoBehaviour
{
    private CancellationTokenSource cts;

    async UniTaskVoid Start()
    {
        // 이 게임오브젝트의 Destroy와 연동
        var token = this.GetCancellationTokenOnDestroy();

        try
        {
            while (true)
            {
                await Patrol(token);
                await UniTask.Delay(1000, cancellationToken: token);
                await Attack(token);
                await UniTask.Delay(500, cancellationToken: token);
            }
        }
        catch (OperationCanceledException)
        {
            // 오브젝트 파괴 시 자연스럽게 종료
        }
    }
}

GetCancellationTokenOnDestroy()를 사용하지 않으면, Destroy 이후에도 비동기 코드가 실행되어 크래시가 발생할 수 있습니다. 모든 UniTask에 CancellationToken을 전달하는 습관을 들이세요.

Part 5: 성능 비교와 선택 기준

Coroutine과 UniTask의 성능 차이는 실제로 얼마나 큰가요?
항목CoroutineUniTaskC# Task
GC 할당 yield마다 Boxing 발생 가능 제로 할당 (구조체 기반) Task 객체 힙 할당
반환값 불가 UniTask<T> Task<T>
에러 처리 불가 (yield 내) try-catch 완전 지원 try-catch 지원
취소 StopCoroutine CancellationToken CancellationToken
스레드 메인 스레드만 메인 + 워커 스레드 ThreadPool
MonoBehaviour 의존 필수 불필요 불필요
그러면 항상 UniTask를 쓰면 되나요?

상황에 따라 다릅니다:

  • Coroutine이 적합: 간단한 시퀀스 (2~3줄), 프로토타이핑, 팀 학습 비용이 부담될 때
  • UniTask가 적합: 네트워크 요청, 리소스 로딩, 복잡한 비동기 흐름, 반환값/에러 처리가 필요할 때
  • C# Task는 주의: Unity 메인 스레드에서 벗어나므로, UnityEngine API 호출 시 오류 발생
🎯 비유로 이해하기

Coroutine은 일반 가위, UniTask는 전동 가위입니다.

종이 한 장 자를 때는 일반 가위가 빠르고 간편합니다. 하지만 100장을 정밀하게 잘라야 한다면 전동 가위가 압도적으로 효율적입니다.

UniTask의 await UniTask.SwitchToThreadPool()과 await UniTask.SwitchToMainThread()를 사용하면 무거운 계산을 워커 스레드에서 수행하고, 결과를 메인 스레드에서 적용할 수 있습니다.

C# - 스레드 전환 패턴
async UniTask<Mesh> GenerateMeshAsync(int[] data, CancellationToken token)
{
    // 워커 스레드에서 무거운 계산
    await UniTask.SwitchToThreadPool();

    var vertices = new Vector3[data.Length];
    var triangles = new int[data.Length * 3];
    // ... 복잡한 메시 계산 (메인 스레드 블로킹 없음)

    // 다시 메인 스레드로 복귀 (Unity API 사용)
    await UniTask.SwitchToMainThread(token);

    var mesh = new Mesh();
    mesh.vertices = vertices;
    mesh.triangles = triangles;
    mesh.RecalculateNormals();
    return mesh;
}

Part 6: UniTask 도입 시 주의사항

기존 프로젝트에 UniTask를 도입할 때 주의할 점은?
  • async void 금지: 반드시 async UniTaskVoid 또는 async UniTask를 사용하세요. async void는 예외가 삼켜집니다.
  • Forget() 호출: Fire-and-forget 패턴에서는 .Forget()을 호출하여 경고를 억제합니다.
  • 한 프레임 딜레이: await UniTask.Yield()는 yield return null과 동일합니다.
C# - 흔한 실수와 올바른 패턴
// BAD: async void — 예외가 삼켜짐
async void OnButtonClick()
{
    await LoadDataAsync(); // 예외 발생 시 추적 불가
}

// GOOD: UniTaskVoid + Forget
async UniTaskVoid OnButtonClickAsync()
{
    await LoadDataAsync(); // 예외가 정상적으로 전파됨
}

// 이벤트 핸들러에서 호출
button.onClick.AddListener(() => OnButtonClickAsync().Forget());

UniTask는 NuGet이 아닌 Unity Package Manager의 Git URL로 설치합니다: https://github.com/Cysharp/UniTask.git?path=src/UniTask/Assets/Plugins/UniTask

📌 핵심 정리

  • Coroutine은 IEnumerator 기반 상태 머신이며, 컴파일러가 yield return마다 상태를 분리한다
  • Coroutine의 한계: 반환값 없음, try-catch 불가, MonoBehaviour 종속, GC 할당
  • UniTask는 제로 GC 할당의 async/await 라이브러리로, 모든 Coroutine 한계를 해결한다
  • CancellationToken을 항상 전달하여 오브젝트 파괴 후 비동기 코드 실행을 방지하라
  • 간단한 대기에는 Coroutine, 복잡한 비동기 흐름에는 UniTask를 선택하라
