Part 1: Unity 생명주기 전체 흐름

Unity 생명주기의 전체 흐름을 설명해 주세요.

Unity의 MonoBehaviour는 정해진 순서에 따라 이벤트 함수들이 호출됩니다. 면접에서는 각 단계의 호출 순서와 용도를 아는 것이 중요합니다.

초기화 단계
Awake → OnEnable → Start
→
물리 단계
FixedUpdate → 물리 연산
→
게임 로직 단계
Update → LateUpdate
→
렌더링 단계
OnGUI → 렌더링

Part 2: 단계별 상세

초기화 단계에서 각 함수의 역할은?

Awake: 스크립트 인스턴스가 로드될 때 호출. 자기 자신의 컴포넌트 참조를 잡는 데 사용합니다.

OnEnable: 오브젝트가 활성화될 때마다 호출. 오브젝트 풀링 등에서 활용됩니다.

Start: 첫 번째 프레임 업데이트 전 한 번만 호출. 다른 오브젝트 참조를 통한 초기화에 사용합니다.

물리/로직 단계에서의 호출 순서는?

FixedUpdate: 고정 시간 간격으로 호출. 물리 연산(Rigidbody)은 반드시 여기서 처리합니다.

Update: 매 프레임 호출. 일반 게임 로직과 입력 처리에 사용합니다.

LateUpdate: Update 이후 호출. 카메라 추적 등 다른 오브젝트의 이동이 완료된 후 처리가 필요한 로직에 사용합니다.

한 프레임 내에서 FixedUpdate는 여러 번 호출될 수 있고, Update와 LateUpdate는 한 번씩 호출됩니다.

소멸 단계는 어떻게 되나요?

OnDisable: 오브젝트가 비활성화될 때 호출됩니다.

OnDestroy: 오브젝트가 파괴될 때 호출됩니다.

OnDisable은 비활성화/파괴 시 모두 호출되지만, OnDestroy는 파괴 시에만 호출됩니다.

Part 3: 면접 포인트

면접에서 자주 나오는 생명주기 관련 질문은?
  • Awake vs Start 차이: 호출 시점, 비활성 상태에서의 동작, 사용 용도
  • Update vs FixedUpdate: 호출 간격, 물리 연산과의 관계
  • 전체 호출 순서: Awake → OnEnable → Start → FixedUpdate → Update → LateUpdate

📌 핵심 정리

  • Unity 생명주기: 초기화 → 물리 → 게임 로직 → 렌더링 → 소멸

  • 매 프레임: FixedUpdate(여러 번 가능) → Update → LateUpdate
  • 소멸 순서: OnDisable → OnDestroy
